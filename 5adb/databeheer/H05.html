Leerdoelen

Na dit hoofdstuk kan je:

  1. Een SELECT-query schrijven om gegevens op te halen uit een tabel.
  2. Resultaten filteren met de WHERE-clausule en logische operatoren.
  3. Resultaten sorteren met ORDER BY.
  4. Het aantal resultaten beperken met LIMIT.
  5. Aggregaatfuncties toepassen: COUNT, SUM, AVG, MIN, MAX.
  6. Resultaten groeperen met GROUP BY en gefilterde groepen selecteren met HAVING.
  7. Kolomaliassen gebruiken met AS.

5.1 De SELECT-opdracht

📘 Definitie

SELECT is het SQL-sleutelwoord waarmee je gegevens ophaalt uit een of meer tabellen. Het is de meest gebruikte SQL-opdracht en vormt de basis van alle databevragingen.

5.1.1 Basisstructuur


SELECT kolom1, kolom2, ...
FROM tabelnaam;

Alle kolommen ophalen (gebruik met mate):


SELECT * FROM product;
⚠️ Let op

️ Let op

SELECT * haalt alle kolommen op, ook kolommen die je niet nodig hebt. Dit is traag bij grote tabellen en maakt code minder leesbaar. In productieomgevingen vermeld je altijd de kolomnamen expliciet.

5.1.2 Specifieke kolommen selecteren


-- Alleen naam en prijs ophalen
SELECT naam, prijs
FROM product;

5.1.3 Kolomaliassen met AS


-- Kolomnamen hernoemen in het resultaat
SELECT
    naam          AS productnaam,
    prijs         AS verkoopprijs,
    voorraad      AS aantal_in_stock
FROM product;

Resultaat:

productnaamverkoopprijsaantal_in_stock
Laptop Pro 15999.0010
Smartphone X12599.0025

5.2 Filteren met WHERE

📘 Definitie

De WHERE-clausule filtert rijen op basis van een of meer condities. Alleen rijen waarvoor de conditie WAAR (TRUE) is, worden opgenomen in het resultaat.

5.2.1 Vergelijkingsoperatoren

OperatorBetekenisVoorbeeld
=Gelijk aanWHERE stad = 'Gent'
<> of !=Niet gelijk aanWHERE status <> 'geannuleerd'
<Kleiner danWHERE prijs < 100
>Groter danWHERE prijs > 100
<=Kleiner dan of gelijkWHERE leeftijd <= 18
>=Groter dan of gelijkWHERE voorraad >= 10

-- Producten duurder dan €500
SELECT naam, prijs
FROM product
WHERE prijs > 500;

5.2.2 Logische operatoren: AND, OR, NOT


-- Elektronica duurder dan €500
SELECT naam, prijs, categorie_id
FROM product
WHERE categorie_id = 1 AND prijs > 500;

-- Producten duurder dan €500 OF met meer dan 20 in voorraad
SELECT naam, prijs, voorraad
FROM product
WHERE prijs > 500 OR voorraad > 20;

-- Producten die NIET in categorie 1 zitten
SELECT naam
FROM product
WHERE NOT categorie_id = 1;
💡 Tip

Gebruik haakjes bij complexe condities om de volgorde van uitwerking duidelijk te maken:

WHERE (prijs > 500 OR voorraad > 20) AND actief = 1

5.2.3 BETWEEN, IN, LIKE, IS NULL

BETWEEN — bereik van waarden:


-- Producten met prijs tussen €50 en €200 (beide grenzen inbegrepen)
SELECT naam, prijs
FROM product
WHERE prijs BETWEEN 50 AND 200;

IN — lijst van waarden:


-- Klanten uit Gent, Antwerpen of Brussel
SELECT voornaam, achternaam, stad
FROM klant
WHERE stad IN ('Gent', 'Antwerpen', 'Brussel');

LIKE — patroonvergelijking:


-- Klanten waarvan de achternaam begint met 'V'
SELECT voornaam, achternaam
FROM klant
WHERE achternaam LIKE 'V%';

-- Klanten waarvan de e-mail '@gmail.com' bevat
SELECT voornaam, email
FROM klant
WHERE email LIKE '%@gmail.com';
📘 Definitie

Bij LIKE gebruik je wildcards: % vervangt nul of meer tekens, _ vervangt precies één teken.

IS NULL — lege waarden:


-- Bestellingen zonder terugbrengdatum (nog niet teruggegeven)
SELECT lening_id, lid_id, boek_id, uitleendatum
FROM lening
WHERE terugbrengdatum IS NULL;

-- Het tegengestelde:
WHERE terugbrengdatum IS NOT NULL
⚠️ Let op

️ Let op

WHERE kolom = NULL werkt NIET in SQL. Gebruik altijd IS NULL of IS NOT NULL.

5.3 Sorteren met ORDER BY


-- Sorteren van goedkoopste naar duurste (ASC = standaard)
SELECT naam, prijs
FROM product
ORDER BY prijs ASC;

-- Sorteren van duurste naar goedkoopste
SELECT naam, prijs
FROM product
ORDER BY prijs DESC;

-- Sorteren op meerdere kolommen
SELECT achternaam, voornaam, stad
FROM klant
ORDER BY stad ASC, achternaam ASC;

5.4 Resultaten beperken met LIMIT


-- Eerste 5 producten ophalen
SELECT naam, prijs
FROM product
LIMIT 5;

-- Top 3 duurste producten
SELECT naam, prijs
FROM product
ORDER BY prijs DESC
LIMIT 3;

-- Paginering: sla 10 resultaten over, toon de volgende 10
SELECT naam, prijs
FROM product
LIMIT 10 OFFSET 10;

5.5 Aggregaatfuncties

📘 Definitie

Een aggregaatfunctie berekent een enkelvoudige waarde op basis van een reeks rijen. Aggregaatfuncties negeren NULL-waarden (behalve COUNT(*)).

FunctieBeschrijving
COUNT(*)Aantal rijen
COUNT(kolom)Aantal niet-NULL waarden in de kolom
SUM(kolom)Som van de waarden
AVG(kolom)Gemiddelde van de waarden
MIN(kolom)Minimumwaarde
MAX(kolom)Maximumwaarde

-- Hoeveel producten zijn er?
SELECT COUNT(*) AS aantal_producten FROM product;

-- Wat is de gemiddelde prijs?
SELECT AVG(prijs) AS gemiddelde_prijs FROM product;

-- Wat is de goedkoopste en duurste prijs?
SELECT MIN(prijs) AS goedkoopste, MAX(prijs) AS duurste FROM product;

-- Totale waarde van alle voorraad
SELECT SUM(prijs * voorraad) AS totale_voorraadwaarde FROM product;

5.6 Groeperen met GROUP BY

📘 Definitie

GROUP BY groepeert rijen met dezelfde waarde in een kolom samen. Op elke groep kan je vervolgens een aggregaatfunctie toepassen.


-- Aantal producten per categorie
SELECT categorie_id, COUNT(*) AS aantal
FROM product
GROUP BY categorie_id;

-- Gemiddelde prijs per categorie
SELECT categorie_id, AVG(prijs) AS gem_prijs, COUNT(*) AS aantal
FROM product
GROUP BY categorie_id
ORDER BY gem_prijs DESC;
💡 Tip

In een SELECT-query met GROUP BY mogen alleen kolommen voorkomen die (a) in de GROUP BY staan, of (b) binnen een aggregaatfunctie zitten.

5.6.1 Groepen filteren met HAVING

📘 Definitie

HAVING filtert groepen na de GROUP BY-operatie, zoals WHERE rijen filtert. Gebruik HAVING voor condities op aggregaatfuncties.


-- Categorieën met meer dan 2 producten
SELECT categorie_id, COUNT(*) AS aantal
FROM product
GROUP BY categorie_id
HAVING COUNT(*) > 2;

-- Categorieën waarvan de gemiddelde prijs boven €200 ligt
SELECT categorie_id, AVG(prijs) AS gem_prijs
FROM product
GROUP BY categorie_id
HAVING AVG(prijs) > 200;

Verschil WHERE vs. HAVING:

WHEREHAVING
WanneerVóór GROUP BYNá GROUP BY
FiltertIndividuele rijenGroepen
AggregaatfunctiesNiet toegelatenToegelaten

-- WHERE en HAVING samen:
-- Actieve producten per categorie, maar toon alleen categorieën met gem. prijs > 100
SELECT categorie_id, COUNT(*) AS aantal, AVG(prijs) AS gem_prijs
FROM product
WHERE actief = 1           -- filter EERST individuele rijen
GROUP BY categorie_id
HAVING AVG(prijs) > 100;   -- filter DAN de groepen

Huistaak H05

Rapport schrijven met SQL

Gebruik je eigen database (uit H04) om een "rapport" te genereren via SQL-queries.

Opdracht: Schrijf minstens 6 zinvolle SELECT-queries die samen een nuttig overzicht geven van de data in je database. Minimale vereisten:

  1. Eén query met WHERE en meerdere condities (AND/OR)
  2. Eén query met ORDER BY op meerdere kolommen
  3. Eén query met een aggregaatfunctie (COUNT, SUM, AVG, MIN of MAX)
  4. Eén query met GROUP BY
  5. Eén query met HAVING
  6. Één vrije query die iets interessants toont over je data

Voeg elke query toe aan het bestand queries/rapport.sql in je GitHub-repository. Voeg boven elke query een commentaarregel toe (-- Beschrijving van de query).

Samenvatting

📋 Samenvatting
  • SELECT haalt gegevens op; vermeld altijd de kolomnamen (geen SELECT * in productie).
  • WHERE filtert rijen: gebruik =, <>, <, >, BETWEEN, IN, LIKE, IS NULL.
  • ORDER BY sorteert: ASC (standaard, oplopend) of DESC (aflopend).
  • LIMIT beperkt het aantal resultaten; OFFSET gebruikt voor paginering.
  • Aggregaatfuncties (COUNT, SUM, AVG, MIN, MAX) berekenen een waarde over een groep rijen.
  • GROUP BY groepeert rijen; HAVING filtert groepen (zoals WHERE voor rijen).
  • AS hernoemt kolommen of tabellen in het resultaat (alias).
