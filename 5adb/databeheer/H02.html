2.1 De werkomgeving opzetten

2.1.1 MySQL installeren

Voor dit vak gebruiken we MySQL als databasesysteem. MySQL is gratis, breed ondersteund en eenvoudig te installeren.

Stap-voor-stap installatie (Windows):


# 1. Download MySQL Installer van https://dev.mysql.com/downloads/installer/
# 2. Kies "MySQL Server" + "MySQL Workbench" tijdens de installatie
# 3. Stel een root-wachtwoord in (onthoud dit!)
# 4. Kies "Standard Configuration"
💡 Tip

Noteer je root-wachtwoord direct in je projectdocumentatie (nooit online of in een publieke repository!). Vergeten root-wachtwoord herstellen kost tijd.

2.1.2 MySQL Workbench als grafische client

MySQL Workbench is een grafische tool waarmee je:

  • Verbinding maakt met je MySQL-server
  • SQL-queries schrijft en uitvoert
  • Tabellen visueel bekijkt en bewerkt
  • ERD-schema's tekent

Verbinding opzetten in MySQL Workbench:


Hostname:   127.0.0.1 (of localhost)
Port:       3306
Username:   root
Password:   [jouw root-wachtwoord]

2.1.3 Eerste verbinding via de command-line


# Start de MySQL command-line client
mysql -u root -p

# Voer je wachtwoord in wanneer gevraagd
# Je ziet nu de MySQL-prompt:
# mysql>
🟢 Voorbeeld

Eerste commando's in de MySQL-client:


-- Toon alle bestaande databases
SHOW DATABASES;

-- Maak een nieuwe database aan
CREATE DATABASE mijn_eerste_db;

-- Activeer de database
USE mijn_eerste_db;

-- Bevestig welke database actief is
SELECT DATABASE();

Resultaat van SHOW DATABASES:

| Database |

|---|

| information_schema |

| mijn_eerste_db |

| mysql |

| performance_schema |

| sys |

2.2 Mappenstructuur voor een databeheer-project

Een goede mappenstructuur is essentieel voor overzicht en samenwerking.

📘 Definitie

Een projectstructuur is de manier waarop bestanden en mappen worden georganiseerd binnen een project. Een consistente structuur maakt samenwerken, back-ups en versiebeheer eenvoudiger.

Aanbevolen structuur voor een databeheer-project:


mijn_project/
├── README.md            ← Beschrijving van het project
├── schema/
│   ├── create_tables.sql   ← DDL: tabellen aanmaken
│   └── erd.png             ← ERD-diagram
├── data/
│   ├── seed_data.sql    ← Testdata invoegen
│   └── import/          ← CSV-bestanden voor import
├── queries/
│   ├── select_queries.sql
│   └── rapport_queries.sql
└── docs/
    └── werkplanning.md
💡 Tip

Maak altijd een README.md in de root van je project. Beschrijf daarin: wat doet de databank, welke tabellen zijn er, en hoe importeer je de databank?

2.3 Versiebeheer met Git

2.3.1 Wat is versiebeheer?

📘 Definitie

Versiebeheer (version control) is een systeem dat wijzigingen in bestanden bijhoudt over tijd. Je kan op elk moment teruggaan naar een vorige versie, zien wie wat heeft gewijzigd, en samenwerken zonder elkaars werk te overschrijven.

Stel je voor: je werkt aan een SQL-script en na een uur merk je dat je iets kapot hebt gemaakt. Zonder versiebeheer moet je alles manueel herstellen. Met Git: git checkout en je bent terug naar de werkende versie.

2.3.2 Git basisbegrippen

BegripBetekenis
Repository (repo)Een map met versiebeheer — bevat alle bestanden + hun geschiedenis
CommitEen opgeslagen snapshot van de bestanden op een bepaald moment
BranchEen aparte ontwikkellijn (standaard: main)
PushLokale commits doorsturen naar GitHub
PullWijzigingen van GitHub ophalen naar je lokale machine
CloneEen bestaande GitHub-repository kopiëren naar je computer

2.3.3 Git installeren en configureren


# Installeer Git via https://git-scm.com/
# Configureer je naam en e-mail (eenmalig):
git config --global user.name "Jouw Naam"
git config --global user.email "jouw@email.be"

# Controleer de configuratie:
git config --list

2.3.4 Een project opzetten met Git


# Navigeer naar je projectmap
cd mijn_project

# Initialiseer een Git-repository
git init

# Voeg alle bestanden toe aan de staging area
git add .

# Maak je eerste commit
git commit -m "Initiële projectstructuur aangemaakt"

# Koppel je lokale repo aan GitHub (maak eerst een repo aan op GitHub.com)
git remote add origin https://github.com/jouwgebruikersnaam/mijn_project.git

# Stuur je commits naar GitHub
git push -u origin main
🟢 Voorbeeld

Je hebt een SQL-script gewijzigd en wil dit opslaan in Git:


# Controleer welke bestanden zijn gewijzigd
git status

# Voeg het gewijzigde bestand toe
git add schema/create_tables.sql

# Maak een commit met een beschrijvende boodschap
git commit -m "Tabel 'klant' uitgebreid met kolom 'stad'"

# Stuur naar GitHub
git push

Goede commit-boodschappen:

  • ✅ "Tabel 'bestelling' aangemaakt met foreign key naar klant"
  • ✅ "Bug opgelost: NOT NULL constraint ontbrak op email-kolom"
  • ❌ "update" (te vaag)
  • ❌ "asdfjkl" (totaal nutteloos)
⚠️ Let op

️ Let op

Sla nooit wachtwoorden, API-sleutels of andere gevoelige informatie op in een Git-repository, ook niet in een privé-repository. Gebruik een .gitignore-bestand om gevoelige bestanden uit te sluiten.

2.3.5 Het .gitignore-bestand


# Inhoud van .gitignore voor een databeheer-project:
*.log
*.tmp
config/database_credentials.env
data/persoonlijke_data/

2.4 Werkplanning opstellen

📘 Definitie

Een werkplanning is een overzicht van de taken die moeten worden uitgevoerd, de volgorde ervan, en de geschatte tijd per taak. Een goede werkplanning helpt je om deadlines te halen en niets te vergeten.

Voor elke database-opdracht stel je een werkplanning op met:

  1. Analyse — Wat moet de databank bijhouden? Welke entiteiten zijn er?
  2. Ontwerp — ERD tekenen, tabellen definiëren, relaties bepalen.
  3. Implementatie — Tabellen aanmaken, data invoegen.
  4. Testen — Queries schrijven, resultaten controleren.
  5. Documentatie — README, ERD toevoegen aan repo.
FaseTaakGeschatte tijd
AnalyseBehoeften in kaart brengen30 min
OntwerpERD tekenen45 min
ImplementatieCREATE TABLE statements30 min
ImplementatieTestdata invoegen20 min
TestenSELECT-queries schrijven45 min
DocumentatieREADME schrijven, Git push20 min

Voorbeelden

🟢 Voorbeeld

We maken een project aan voor een kleine bibliotheek.


# Stap 1: Projectmap aanmaken
mkdir bibliotheek_db
cd bibliotheek_db

# Stap 2: Submappen aanmaken
mkdir schema data queries docs

# Stap 3: README aanmaken
echo "# Bibliotheek Database" > README.md

# Stap 4: Git initialiseren
git init
git add .
git commit -m "Initiële structuur bibliotheek_db"


-- schema/create_tables.sql
CREATE DATABASE IF NOT EXISTS bibliotheek;
USE bibliotheek;

CREATE TABLE boek (
boek_id    INT          PRIMARY KEY AUTO_INCREMENT,
titel      VARCHAR(200) NOT NULL,
auteur     VARCHAR(100) NOT NULL,
isbn       VARCHAR(20)  UNIQUE,
jaar       YEAR
);
🟢 Voorbeeld

Twee leerlingen werken samen aan één project. Leerling A maakt de klant-tabel, leerling B de bestelling-tabel.


# Leerling A:
git pull origin main          # Haal laatste versie op
# ... werkt aan schema/klant.sql ...
git add schema/klant.sql
git commit -m "Tabel klant aangemaakt"
git push origin main

# Leerling B (daarna):
git pull origin main          # Haal commit van A op
# ... werkt aan schema/bestelling.sql ...
git add schema/bestelling.sql
git commit -m "Tabel bestelling aangemaakt met FK naar klant"
git push origin main

Samenvatting

📋 Samenvatting
  • MySQL is het databasesysteem dat we gebruiken; verbinding via MySQL Workbench of command-line client.
  • Een consistente mappenstructuur (schema/, data/, queries/, docs/, README.md) is de basis van elk project.
  • Git is een versiebeheersysteem dat alle wijzigingen bijhoudt; commando's: git init, git add, git commit, git push.
  • Sla nooit wachtwoorden op in een Git-repository; gebruik een .gitignore-bestand.
  • Een werkplanning verdeelt een opdracht in fases: analyse → ontwerp → implementatie → testen → documentatie.
