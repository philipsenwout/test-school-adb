3.1 Bouwstenen van een datamodel

3.1.1 Entiteit

📘 Definitie

Een entiteit is een "ding" of "persoon" waarover je gegevens wil bijhouden. In een database wordt een entiteit een tabel. Voorbeelden: Klant, Product, Leerling, Boek.

Entiteiten zijn altijd zelfstandige naamwoorden.

3.1.2 Attribuut

📘 Definitie

Een attribuut is een eigenschap van een entiteit. In een tabel is elk attribuut een kolom.

Voorbeeld — entiteit Leerling:

AttribuutUitleg
leerling_iduniek nummer
voornaamvoornaam van de leerling
achternaamachternaam
geboortedatumwanneer geboren
emaile-mailadres

3.1.3 Primaire sleutel

📘 Definitie

De primaire sleutel (PRIMARY KEY) is een kolom die elke rij in een tabel uniek identificeert. Geen twee rijen mogen dezelfde primaire sleutel hebben.

Meestal gebruik je een automatisch oplopend getal als primaire sleutel: id, klant_id, leerling_id ...


CREATE TABLE leerling (
    leerling_id  INT         PRIMARY KEY AUTO_INCREMENT,
    voornaam     VARCHAR(50) NOT NULL,
    achternaam   VARCHAR(50) NOT NULL,
    email        VARCHAR(150)
);
💡 Tip

Gebruik altijd AUTO_INCREMENT voor je primaire sleutel. Dan hoef je zelf nooit een nummer in te vullen — de database doet dat automatisch.

3.2 Waarom meerdere tabellen?

Stel je voor dat je een database maakt voor bestellingen. Je zou alles in één tabel kunnen stoppen:

bestelling_idklant_naamklant_emailproductprijs
1Jan Peetersjan@email.beLaptop999
2Jan Peetersjan@email.beMuis25
3Sara Claessara@email.beToetsenbord45

Problemen:

  • jan@email.be staat twee keer → als Jan zijn e-mail verandert, moet je meerdere rijen aanpassen
  • Je kan een klant niet opslaan zonder bestelling
  • Als je een bestelling verwijdert, verdwijnt ook de klantinfo

Oplossing: verdeel over twee tabellen:


Tabel klant:       klant_id | naam | email
Tabel bestelling:  bestelling_id | klant_id | product | prijs

Door klant_id in de bestelling-tabel te bewaren, koppel je de twee tabellen aan elkaar.

3.3 Types relaties

3.3.1 Één-op-één (1:1)

Eén record in tabel A hoort bij precies één record in tabel B.

Voorbeeld: Een leerling heeft precies één leerlingenkaart.


LEERLING  ←———→  LEERLINGENKAART
   1                    1

Dit type is zeldzaam.

3.3.2 Één-op-veel (1:N)

Eén record in tabel A hoort bij meerdere records in tabel B.

Voorbeeld: Eén klant kan meerdere bestellingen plaatsen, maar elke bestelling hoort bij slechts één klant.


KLANT  ←———→  BESTELLING
  1             meerdere

Dit is het meest voorkomende type relatie.

3.3.3 Veel-op-veel (M:N)

Records in tabel A kunnen bij meerdere records in tabel B horen, en omgekeerd.

Voorbeeld: Een leerling kan meerdere vakken volgen. Een vak kan meerdere leerlingen hebben.


LEERLING  ←———→  VAK
meerdere        meerdere

Probleem: Een database kan dit niet rechtstreeks opslaan. Oplossing: een tussentabel.


LEERLING ←→ INSCHRIJVING ←→ VAK

3.4 Een datamodel tekenen

Gebruik een eenvoudig schema met dozen en pijlen:


┌─────────────┐         ┌──────────────────┐
│   KLANT     │         │   BESTELLING     │
├─────────────┤         ├──────────────────┤
│ klant_id PK │ ———————>│ bestelling_id PK │
│ voornaam    │    1:N  │ klant_id         │
│ achternaam  │         │ datum            │
│ email       │         │ totaal           │
└─────────────┘         └──────────────────┘

De pijl loopt van de kant "1" naar de kant "veel".

Voorbeelden

🟢 Voorbeeld

Situatie: een bibliotheek wil bijhouden welke boeken er zijn en welk lid welk boek leent.

Entiteiten: lid, boek, lening

Relaties:

  • Een lid kan meerdere boeken lenen (1:N tussen lid en lening)
  • Een boek kan meerdere keren uitgeleend worden (1:N tussen boek en lening)

LID ←→ LENING ←→ BOEK


CREATE TABLE lid (
lid_id       INT         PRIMARY KEY AUTO_INCREMENT,
voornaam     VARCHAR(50) NOT NULL,
achternaam   VARCHAR(50) NOT NULL
);

CREATE TABLE boek (
boek_id  INT          PRIMARY KEY AUTO_INCREMENT,
titel    VARCHAR(200) NOT NULL,
auteur   VARCHAR(100) NOT NULL
);

CREATE TABLE lening (
lening_id    INT  PRIMARY KEY AUTO_INCREMENT,
lid_id       INT  NOT NULL,
boek_id      INT  NOT NULL,
uitleendatum DATE NOT NULL
);

Samenvatting

📋 Samenvatting
  • Een entiteit is een "ding" waarover je data bijhoudt (= tabel). Een attribuut is een eigenschap van de entiteit (= kolom).
  • De primaire sleutel (PRIMARY KEY) identificeert elke rij uniek. Gebruik AUTO_INCREMENT.
  • Een goede database heeft meerdere tabellen die via een gedeeld ID-veld aan elkaar gekoppeld zijn.
  • Drie types relaties: 1:1 (zeldzaam), 1:N (meest voorkomend), M:N (vereist een tussentabel).
