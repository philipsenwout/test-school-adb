Leerdoelen

Na dit hoofdstuk kan je:

  1. Bestaande gegevens wijzigen met UPDATE … SET … WHERE.
  2. Gegevens verwijderen met DELETE … WHERE.
  3. Het belang van de WHERE-clausule bij UPDATE en DELETE uitleggen.
  4. Transacties gebruiken om destructieve operaties veilig te maken.
  5. Het verschil uitleggen tussen een harde verwijdering (DELETE) en een zachte verwijdering (soft delete).

6.1 Gegevens wijzigen met UPDATE

📘 Definitie

UPDATE wijzigt de waarden van een of meer kolommen in een of meer bestaande rijen. De WHERE-clausule bepaalt welke rijen worden aangepast.

6.1.1 Basisstructuur


UPDATE tabelnaam
SET kolom1 = nieuwe_waarde1,
    kolom2 = nieuwe_waarde2
WHERE conditie;

6.1.2 Voorbeelden


-- Prijs van één specifiek product aanpassen
UPDATE product
SET prijs = 899.00
WHERE product_id = 1;

-- Meerdere kolommen tegelijk aanpassen
UPDATE product
SET prijs    = 549.00,
    voorraad = 30
WHERE product_id = 2;

-- E-mailadres van een klant bijwerken
UPDATE klant
SET email = 'lena.claes@nieuw.be'
WHERE klant_id = 1;

-- Meerdere rijen tegelijk aanpassen
-- Deactiveer alle producten die niet meer op voorraad zijn
UPDATE product
SET actief = 0
WHERE voorraad = 0;
⚠️ Let op

️ Let op

Een UPDATE zonder WHERE-clausule past alle rijen in de tabel aan. Dit is vrijwel altijd een fout. Controleer altijd eerst met een SELECT welke rijen worden geraakt.

Veilige werkwijze:


-- Stap 1: Controleer welke rijen worden geraakt
SELECT product_id, naam, prijs
FROM product
WHERE product_id = 1;

-- Stap 2: Pas aan
UPDATE product
SET prijs = 899.00
WHERE product_id = 1;

-- Stap 3: Verifieer het resultaat
SELECT product_id, naam, prijs
FROM product
WHERE product_id = 1;

6.1.3 UPDATE met berekeningen


-- Verhoog de prijs van alle elektronica met 10%
UPDATE product
SET prijs = prijs * 1.10
WHERE categorie_id = 1;

-- Verminder de voorraad met 1 (na verkoop)
UPDATE product
SET voorraad = voorraad - 1
WHERE product_id = 3;

6.2 Gegevens verwijderen met DELETE

📘 Definitie

DELETE verwijdert een of meer rijen uit een tabel. De WHERE-clausule bepaalt welke rijen worden verwijderd. Verwijderde rijen kunnen niet worden teruggehaald (tenzij via een back-up of transactie).

6.2.1 Basisstructuur


DELETE FROM tabelnaam
WHERE conditie;

6.2.2 Voorbeelden


-- Eén specifiek product verwijderen
DELETE FROM product
WHERE product_id = 5;

-- Alle inactieve producten verwijderen
DELETE FROM product
WHERE actief = 0;

-- Verwijder een lening op basis van ID
DELETE FROM lening
WHERE lening_id = 3;
⚠️ Let op

️ Let op

DELETE FROM product; (zonder WHERE) verwijdert alle producten. Dit is vrijwel altijd onbedoeld. Gebruik dezelfde "veilige werkwijze" als bij UPDATE: controleer eerst met SELECT.

6.2.3 Verwijdering en referentiële integriteit

Als je een rij probeert te verwijderen die nog wordt gerefereerd door een FOREIGN KEY, geeft MySQL een foutmelding.


-- Probeer klant_id = 1 te verwijderen
-- Maar bestelling heeft FK naar klant!
DELETE FROM klant WHERE klant_id = 1;
-- ERROR 1451: Cannot delete or update a parent row: a foreign key constraint fails

Oplossingen:

  1. Verwijder eerst de afhankelijke rijen (bestellingen van de klant).
  2. Gebruik ON DELETE CASCADE in de FOREIGN KEY-definitie.
  3. Gebruik een soft delete (zie 6.3).

-- ON DELETE CASCADE: als klant verwijderd wordt, worden zijn bestellingen ook verwijderd
CREATE TABLE bestelling (
    bestelling_id  INT  PRIMARY KEY AUTO_INCREMENT,
    klant_id       INT  NOT NULL,
    datum          DATETIME NOT NULL,
    FOREIGN KEY (klant_id) REFERENCES klant(klant_id)
        ON DELETE CASCADE
);

6.3 Transacties

📘 Definitie

Een transactie is een reeks SQL-operaties die als één geheel worden uitgevoerd. Als één operatie mislukt, worden alle wijzigingen teruggedraaid (ROLLBACK). Als alles slaagt, worden de wijzigingen permanent opgeslagen (COMMIT).

6.3.1 ACID-eigenschappen

Transacties garanderen vier eigenschappen, aangeduid met het acroniem ACID:

EigenschapBeschrijving
AtomicityAlles of niets: alle operaties slagen of geen enkele
ConsistencyDe database blijft in een consistente toestand
IsolationGelijktijdige transacties storen elkaar niet
DurabilityEenmaal gecommit zijn wijzigingen permanent

6.3.2 Transacties gebruiken


-- Start een transactie
START TRANSACTION;

-- Voer operaties uit
UPDATE rekening SET saldo = saldo - 100 WHERE rekening_id = 1;
UPDATE rekening SET saldo = saldo + 100 WHERE rekening_id = 2;

-- Als alles correct: sla op
COMMIT;

-- Als er iets misloopt: draai alles terug
ROLLBACK;

6.3.3 Transactie met foutcontrole


START TRANSACTION;

-- Verwijder bestelregels van een bestelling
DELETE FROM bestelregel WHERE bestelling_id = 1;

-- Verwijder de bestelling zelf
DELETE FROM bestelling WHERE bestelling_id = 1;

-- Controleer: zijn er nog andere bestellingen van deze klant?
-- SELECT * FROM bestelling WHERE klant_id = 1;

-- Als alles klopt:
COMMIT;
-- Als er twijfel is:
-- ROLLBACK;
💡 Tip

In MySQL Workbench kan je "Auto-Commit" uitschakelen. Dan worden alle wijzigingen pas opgeslagen als je expliciet COMMIT uitvoert. Dit is veiliger bij destructieve operaties.

6.4 Soft delete

📘 Definitie

Bij een soft delete worden rijen niet fysiek verwijderd maar gemarkeerd als "verwijderd" via een statuskolom (bv. verwijderd_op of actief). De data blijft beschikbaar voor rapportage of herstel.

Voordelen van soft delete:

  • Data kan worden hersteld.
  • Historische data blijft beschikbaar voor analyses.
  • Voldoet beter aan GDPR (audit trail).

-- Tabel met soft delete kolom
ALTER TABLE klant ADD COLUMN verwijderd_op DATETIME DEFAULT NULL;

-- Soft delete: markeer als verwijderd
UPDATE klant
SET verwijderd_op = NOW()
WHERE klant_id = 3;

-- Query die soft-deleted rijen uitsluit
SELECT klant_id, voornaam, achternaam
FROM klant
WHERE verwijderd_op IS NULL;

-- Herstel: verwijder de markering
UPDATE klant
SET verwijderd_op = NULL
WHERE klant_id = 3;

Voorbeelden

🟢 Voorbeeld

Scenario: de naam van een product moet worden verbeterd, en een uitverkocht product moet worden verwijderd.


-- 1. Controleer welke rijen geraakt worden
SELECT product_id, naam, prijs, voorraad
FROM product
WHERE product_id IN (1, 4);

-- 2. UPDATE: verbeter de naam
START TRANSACTION;
UPDATE product
SET naam = 'Laptop Pro 15" (2026 editie)'
WHERE product_id = 1;

-- 3. Controleer het resultaat
SELECT product_id, naam FROM product WHERE product_id = 1;

-- 4. Bevestig de wijziging
COMMIT;

-- 5. Soft delete voor het uitverkochte product
UPDATE product
SET actief = 0
WHERE product_id = 4 AND voorraad = 0;
🟢 Voorbeeld

Illustratie van waarom transacties essentieel zijn.


CREATE TABLE rekening (
rekening_id  INT          PRIMARY KEY AUTO_INCREMENT,
eigenaar     VARCHAR(100) NOT NULL,
saldo        DECIMAL(10,2) NOT NULL CHECK (saldo >= 0)
);

INSERT INTO rekening (eigenaar, saldo) VALUES
('Lena Claes',  1500.00),
('Raf Dubois',   300.00);

-- Overschrijving: Lena betaalt Raf €200
START TRANSACTION;

UPDATE rekening SET saldo = saldo - 200 WHERE rekening_id = 1;
UPDATE rekening SET saldo = saldo + 200 WHERE rekening_id = 2;

-- Controleer: totaalsom moet gelijk blijven (1500 + 300 = 1800)
SELECT SUM(saldo) AS totaal FROM rekening;

COMMIT;

Zonder transactie: Als de server crasht na de eerste UPDATE, verliest Lena €200 maar Raf krijgt niets. Met transactie: bij crash wordt de eerste UPDATE ook teruggedraaid.

Samenvatting

📋 Samenvatting
  • UPDATE wijzigt bestaande rijen: UPDATE tabel SET kolom = waarde WHERE conditie;
  • DELETE verwijdert rijen: DELETE FROM tabel WHERE conditie;
  • Gebruik altijd een WHERE-clausule bij UPDATE en DELETE; controleer eerst met SELECT.
  • Een transactie (START TRANSACTION → COMMIT / ROLLBACK) zorgt voor atomaire uitvoering.
  • Soft delete markeert rijen als verwijderd (via verwijderd_op of actief) zonder ze echt te verwijderen — beter voor GDPR en herstelbaarheid.
  • ON DELETE CASCADE verwijdert automatisch afhankelijke rijen bij het verwijderen van een parent.
