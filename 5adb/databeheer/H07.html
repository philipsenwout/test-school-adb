7.1 Waarom JOIN?

In een goed opgebouwde database staan gegevens verspreid over meerdere tabellen. Om een volledig beeld te krijgen, moeten we die tabellen samenvoegen.

Probleem: Je hebt een tabel bestelling met een klant_id, maar de naam van de klant staat in de tabel klant.


-- Dit toont enkel het klant_id, niet de naam
SELECT bestelling_id, klant_id, datum
FROM bestelling;

Resultaat:

bestelling_idklant_iddatum
132026-02-10
212026-02-11

Met JOIN kunnen we de naam erbij halen:


SELECT bestelling_id, voornaam, achternaam, datum
FROM bestelling
INNER JOIN klant ON bestelling.klant_id = klant.klant_id;

Resultaat:

bestelling_idvoornaamachternaamdatum
1SaraClaes2026-02-10
2JanPeeters2026-02-11
📘 Definitie

Een JOIN combineert rijen uit twee tabellen op basis van een gemeenschappelijk veld. Je verbindt de tabellen via het ID-veld dat ze delen.

7.2 INNER JOIN

📘 Definitie

Een INNER JOIN geeft enkel de rijen terug waarvoor er een overeenkomst is in beide tabellen.

Basisstructuur:


SELECT kolommen
FROM tabel_A
INNER JOIN tabel_B ON tabel_A.gedeeld_veld = tabel_B.gedeeld_veld;

Voorbeeld — bibliotheek:


-- Leningen met naam van het lid en titel van het boek
SELECT
    lening.lening_id,
    lid.voornaam,
    lid.achternaam,
    boek.titel,
    lening.uitleendatum
FROM lening
INNER JOIN lid  ON lening.lid_id  = lid.lid_id
INNER JOIN boek ON lening.boek_id = boek.boek_id;

7.3 Tabelaliassen

Bij lange queries is het handig om kortere namen te geven aan tabellen:


-- Zonder aliassen (lang en herhaling)
SELECT bestelling.bestelling_id, klant.voornaam, klant.achternaam
FROM bestelling
INNER JOIN klant ON bestelling.klant_id = klant.klant_id;

-- Met aliassen (korter en leesbaarder)
SELECT b.bestelling_id, k.voornaam, k.achternaam
FROM bestelling AS b
INNER JOIN klant AS k ON b.klant_id = k.klant_id;
💡 Tip

Gebruik de eerste letter van de tabelnaam als alias: b voor bestelling, k voor klant, l voor lening ...

Voorbeelden

🟢 Voorbeeld

SELECT
b.bestelling_id,
k.voornaam,
k.achternaam,
b.datum
FROM bestelling AS b
INNER JOIN klant AS k ON b.klant_id = k.klant_id
ORDER BY b.datum DESC;
🟢 Voorbeeld

SELECT
l.lening_id,
lid.voornaam,
lid.achternaam,
boek.titel,
l.uitleendatum
FROM lening AS l
INNER JOIN lid  ON l.lid_id  = lid.lid_id
INNER JOIN boek ON l.boek_id = boek.boek_id
WHERE l.terugbrengdatum IS NULL;

Samenvatting

📋 Samenvatting
  • JOIN combineert rijen uit twee (of meer) tabellen via een gedeeld veld.
  • INNER JOIN geeft enkel rijen met overeenkomst in beide tabellen.
  • Gebruik tabelaliassen (AS b, AS k) om queries korter en leesbaarder te maken.
  • Meerdere JOINs achter elkaar zijn mogelijk (bv. lening → lid → boek).
