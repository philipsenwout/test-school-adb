Leerdoelen

Na dit hoofdstuk kan je:

  1. Een database aanmaken en verwijderen met DDL-commando's (CREATE DATABASE, DROP DATABASE).
  2. Tabellen aanmaken met correcte datatypes en constraints.
  3. De meest gebruikte SQL-datatypes correct inzetten.
  4. Constraints toepassen: PRIMARY KEY, NOT NULL, UNIQUE, DEFAULT, CHECK.
  5. Gegevens invoegen met INSERT INTO.
  6. Data importeren vanuit een CSV-bestand via MySQL Workbench of LOAD DATA INFILE.

4.1 DDL — Data Definition Language

📘 Definitie

DDL (Data Definition Language) is de categorie van SQL-commando's waarmee je de structuur van een database beheert: databases en tabellen aanmaken, wijzigen en verwijderen.

De drie hoofdcommando's van DDL zijn:

CommandoGebruik
CREATEDatabase of tabel aanmaken
ALTERStructuur van een tabel wijzigen
DROPDatabase of tabel verwijderen

4.1.1 Database aanmaken en selecteren


-- Maak een nieuwe database aan
CREATE DATABASE IF NOT EXISTS webshop;

-- Activeer de database voor gebruik
USE webshop;

-- Controleer welke database actief is
SELECT DATABASE();

-- Toon alle databases
SHOW DATABASES;

-- Verwijder een database (voorzichtig!)
DROP DATABASE IF EXISTS webshop;
⚠️ Let op

️ Let op

DROP DATABASE verwijdert de database én alle gegevens erin, permanent en zonder bevestigingsvraag. Gebruik dit commando nooit zonder een recente back-up.

4.2 Tabellen aanmaken — CREATE TABLE

4.2.1 Basisstructuur


CREATE TABLE tabelnaam (
    kolomnaam1  DATATYPE    [constraints],
    kolomnaam2  DATATYPE    [constraints],
    ...
);

4.2.2 SQL-datatypes

📘 Definitie

Een datatype bepaalt welk soort waarde een kolom kan bevatten: een getal, een tekst, een datum, ... Het kiezen van het juiste datatype zorgt voor efficiënte opslag en voorkomt ongeldige gegevens.

Tekst:

DatatypeBeschrijvingGebruik
VARCHAR(n)Variabele tekst, max. n tekensnamen, e-mail, beschrijvingen
CHAR(n)Vaste lengte, altijd n tekenspostcodes, landcodes
TEXTLange tekst (max. 65.535 tekens)beschrijvingen, opmerkingen
ENUM(...)Vaste lijst van toegestane waardenstatus, type, geslacht

Getallen:

DatatypeBeschrijvingGebruik
INTGeheel getal (-2,1 miljard tot +2,1 miljard)id's, aantallen
TINYINTKlein getal (0–255 of -128–127)booleaanse waarden, kleine codes
BIGINTGroot geheel getaltransactie-id's, grote aantallen
DECIMAL(p,s)Exact decimaal getalgeldbedragen (bv. DECIMAL(8,2))
FLOAT / DOUBLEBenaderend decimaalwetenschappelijke waarden

Datum en tijd:

DatatypeBeschrijvingFormaat
DATEAlleen datumYYYY-MM-DD
TIMEAlleen tijdHH:MM:SS
DATETIMEDatum + tijdYYYY-MM-DD HH:MM:SS
TIMESTAMPDatum + tijd (auto-update mogelijk)YYYY-MM-DD HH:MM:SS
YEARAlleen jaarYYYY
💡 Tip

Gebruik voor geldbedragen altijd DECIMAL(8,2), nooit FLOAT. Floating-point getallen hebben afrondingsfouten die bij financiële berekeningen tot grote problemen leiden.

4.2.3 Constraints

📘 Definitie

Een constraint is een regel die MySQL afdwingt op de gegevens in een kolom. Constraints voorkomen ongeldige of inconsistente data.

Overzicht constraints:

ConstraintBetekenis
PRIMARY KEYUniek en niet NULL; identificeert elke rij
NOT NULLWaarde mag niet leeg zijn
UNIQUEWaarde moet uniek zijn in de kolom
DEFAULT waardeStandaardwaarde als geen waarde wordt opgegeven
CHECK (conditie)Waarde moet aan een conditie voldoen
FOREIGN KEYVerwijzing naar primaire sleutel van andere tabel
AUTO_INCREMENTWaarde wordt automatisch verhoogd (voor id-kolommen)

CREATE TABLE product (
    product_id    INT            PRIMARY KEY AUTO_INCREMENT,
    naam          VARCHAR(200)   NOT NULL,
    prijs         DECIMAL(8,2)   NOT NULL CHECK (prijs >= 0),
    voorraad      INT            NOT NULL DEFAULT 0 CHECK (voorraad >= 0),
    categorie     VARCHAR(50),
    actief        TINYINT(1)     NOT NULL DEFAULT 1,
    aangemaakt_op TIMESTAMP      DEFAULT CURRENT_TIMESTAMP
);

4.2.4 Tabel wijzigen — ALTER TABLE


-- Kolom toevoegen
ALTER TABLE product ADD COLUMN gewicht DECIMAL(5,2);

-- Kolom verwijderen
ALTER TABLE product DROP COLUMN gewicht;

-- Kolom hernoemen/wijzigen
ALTER TABLE product MODIFY COLUMN naam VARCHAR(300) NOT NULL;

-- Constraint toevoegen
ALTER TABLE product ADD CONSTRAINT chk_prijs CHECK (prijs >= 0);

4.2.5 Tabel verwijderen


-- Tabel verwijderen (inclusief alle data)
DROP TABLE IF EXISTS product;

-- Tabel leegmaken maar structuur behouden
TRUNCATE TABLE product;
⚠️ Let op

️ Let op

TRUNCATE kan niet worden teruggedraaid en omzeilt triggers. Gebruik het enkel in een testomgeving.

4.3 DML — Data invoegen met INSERT INTO

📘 Definitie

DML (Data Manipulation Language) bevat SQL-commando's voor het invoegen, wijzigen en verwijderen van data in tabellen: INSERT, UPDATE, DELETE.

4.3.1 INSERT INTO — één rij


-- Syntaxis met kolomnamen (aanbevolen)
INSERT INTO tabelnaam (kolom1, kolom2, kolom3)
VALUES (waarde1, waarde2, waarde3);

-- Voorbeeld:
INSERT INTO product (naam, prijs, voorraad, categorie)
VALUES ('Laptop', 799.99, 15, 'Elektronica');
💡 Tip

Vermeld altijd de kolomnamen bij INSERT INTO. Als je dit weglaat, is de volgorde van waarden afhankelijk van de tabelstructuur — dit breekt als je later een kolom toevoegt.

4.3.2 INSERT INTO — meerdere rijen tegelijk


INSERT INTO product (naam, prijs, voorraad, categorie)
VALUES
    ('Smartphone',    599.00, 25, 'Elektronica'),
    ('Koptelefoon',    89.99, 50, 'Elektronica'),
    ('Bureau',        249.00,  8, 'Meubelen'),
    ('Bureaulamp',     34.99, 30, 'Meubelen');

4.3.3 Waarden voor speciale datatypes


-- DATUM: gebruik altijd 'YYYY-MM-DD'
INSERT INTO bestelling (klant_id, datum, totaal)
VALUES (1, '2026-01-15', 89.99);

-- NULL invoegen (als de kolom dit toelaat)
INSERT INTO klant (voornaam, achternaam, email, geboortedatum)
VALUES ('Lena', 'Claes', 'lena@mail.be', NULL);

-- DEFAULT gebruiken (kolom weglaten of expliciet DEFAULT meegeven)
INSERT INTO product (naam, prijs)
VALUES ('Muis', 29.99);
-- voorraad krijgt standaard 0, actief krijgt standaard 1

4.4 Data importeren uit CSV

4.4.1 Wat is een CSV-bestand?

📘 Definitie

Een CSV-bestand (Comma-Separated Values) is een tekstbestand waarbij gegevens worden gescheiden door komma's (of puntkomma's). CSV is een veelgebruikt uitwisselingsformaat voor tabellaire data.


product_id,naam,prijs,voorraad,categorie
1,Laptop,799.99,15,Elektronica
2,Smartphone,599.00,25,Elektronica
3,Bureau,249.00,8,Meubelen

4.4.2 Importeren via MySQL Workbench

  1. Klik met rechtermuisknop op de tabel in de navigator.
  2. Kies "Table Data Import Wizard".
  3. Selecteer het CSV-bestand.
  4. Wijs de CSV-kolommen toe aan de tabelkolommen.
  5. Voer de import uit.

4.4.3 Importeren via LOAD DATA INFILE


-- CSV-bestand importeren via SQL-commando
LOAD DATA INFILE '/pad/naar/producten.csv'
INTO TABLE product
FIELDS TERMINATED BY ','
ENCLOSED BY '"'
LINES TERMINATED BY '\n'
IGNORE 1 ROWS   -- sla de koptekstrij over
(naam, prijs, voorraad, categorie);
⚠️ Let op

️ Let op

LOAD DATA INFILE vereist specifieke serverrechten. In schoolomgevingen gebruik je vaak de Import Wizard van MySQL Workbench, die geen extra rechten nodig heeft.

Voorbeelden

🟢 Voorbeeld

We bouwen de databank voor een kleine webshop met klanten, producten en bestellingen.


CREATE DATABASE IF NOT EXISTS webshop;
USE webshop;

CREATE TABLE klant (
klant_id    INT          PRIMARY KEY AUTO_INCREMENT,
voornaam    VARCHAR(50)  NOT NULL,
achternaam  VARCHAR(50)  NOT NULL,
email       VARCHAR(150) NOT NULL UNIQUE,
stad        VARCHAR(100),
aangemaakt  TIMESTAMP    DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE categorie (
categorie_id  INT         PRIMARY KEY AUTO_INCREMENT,
naam          VARCHAR(50) NOT NULL UNIQUE
);

CREATE TABLE product (
product_id    INT            PRIMARY KEY AUTO_INCREMENT,
naam          VARCHAR(200)   NOT NULL,
prijs         DECIMAL(8,2)   NOT NULL CHECK (prijs >= 0),
voorraad      INT            NOT NULL DEFAULT 0,
categorie_id  INT,
FOREIGN KEY (categorie_id) REFERENCES categorie(categorie_id)
);

CREATE TABLE bestelling (
bestelling_id  INT          PRIMARY KEY AUTO_INCREMENT,
klant_id       INT          NOT NULL,
datum          DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
status         ENUM('nieuw', 'verwerkt', 'verzonden', 'geleverd') DEFAULT 'nieuw',
FOREIGN KEY (klant_id) REFERENCES klant(klant_id)
);

CREATE TABLE bestelregel (
regel_id       INT          PRIMARY KEY AUTO_INCREMENT,
bestelling_id  INT          NOT NULL,
product_id     INT          NOT NULL,
aantal         INT          NOT NULL CHECK (aantal > 0),
eenheidsprijs  DECIMAL(8,2) NOT NULL,
FOREIGN KEY (bestelling_id) REFERENCES bestelling(bestelling_id),
FOREIGN KEY (product_id)    REFERENCES product(product_id)
);

Oefeningen

Oefening 4.1 SQL-datatypes kiezen

Het kiezen van het juiste datatype is essentieel voor efficiënte opslag en correcte data. Test je kennis van de meest gebruikte SQL-datatypes.

Vraag 1
Welk datatype gebruik je voor de prijs van een product (bv. €9,99 of €1.299,99)?
Vraag 2
Welk datatype gebruik je voor de naam van een persoon (variabele lengte, max. 100 tekens)?
Vraag 3
Je wil een "actief/inactief" vlag opslaan (0 of 1). Welk datatype is het meest geschikt?
Vraag 4
Welk datatype gebruik je voor een aanmaaktijdstip dat automatisch moet worden ingevuld bij het aanmaken van een rij?
Oefening 4.2 Constraints en CREATE TABLE

Constraints zorgen voor data-integriteit. Test je kennis van PRIMARY KEY, NOT NULL, UNIQUE, DEFAULT en CHECK.

Vraag 1
Welke constraint zorgt ervoor dat geen twee rijen dezelfde waarde in de kolom email mogen hebben?
Vraag 2
Welke MySQL-eigenschap zorg je ervoor dat de waarde van de primaire sleutel automatisch wordt verhoogd bij elke nieuwe rij?
Vraag 3
Welke constraint gebruik je om te valideren dat de prijs van een product niet negatief mag zijn?
Vraag 4
Wat is het verschil tussen DROP TABLE en TRUNCATE TABLE?
Oefening 4.3 INSERT INTO en DML

DML-commando's zoals INSERT INTO worden gebruikt om data in tabellen te plaatsen. Test je kennis van de correcte syntax.

Huistaak H04

Database bouwen voor je miniproject

In H09 (miniproject) bouw je een volledige database over een zelfgekozen thema. Start nu met de implementatie van je eigen datamodel (uit Huistaak H03 of een nieuw thema).

Opdracht:

  1. Maak een nieuwe database aan met een betekenisvolle naam.
  2. Implementeer minstens 3 tabellen met correcte datatypes, PRIMARY KEY, FOREIGN KEY en minstens 3 constraints.
  3. Voeg testdata in: minstens 5 rijen per tabel.
  4. Exporteer je database als een .sql-bestand (schema + data).
  5. Voeg het bestand toe aan je GitHub-repository met een zinvolle commit-boodschap.

Verbetersleutel: zie output/databeheer/huistaken/H04-huistaak-verbetersleutel.md

Samenvatting

📋 Samenvatting
  • DDL = structuurcommando's: CREATE, ALTER, DROP voor databases en tabellen.
  • Datatypes: gebruik VARCHAR voor tekst, DECIMAL(p,s) voor geldbedragen, DATE/DATETIME voor datum/tijd, INT voor gehele getallen.
  • Constraints: PRIMARY KEY, NOT NULL, UNIQUE, DEFAULT, CHECK, FOREIGN KEY zorgen voor data-integriteit.
  • DML voor invoegen: INSERT INTO tabelnaam (kolommen) VALUES (waarden); — vermeld altijd de kolomnamen.
  • CSV-data importeer je via de Import Wizard van MySQL Workbench of LOAD DATA INFILE.
