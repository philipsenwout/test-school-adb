2. Theorie

2.1 Het Von Neumann-model

Vrijwel alle moderne computers zijn gebaseerd op het Von Neumann-model, voorgesteld door wiskundige John von Neumann in 1945. Het model beschrijft een computersysteem met vier basiscomponenten:

📘 Definitie

Het Von Neumann-model is de architectuur waarop bijna alle moderne computers zijn gebouwd. Het beschrijft een systeem met een centrale verwerkingseenheid (CPU), geheugen (RAM), invoer/uitvoer-eenheden en een bus die alles verbindt.

  1. CPU (Central Processing Unit) — voert instructies uit
  2. Geheugen (RAM) — slaat tijdelijk data en instructies op
  3. Opslag (HDD, SSD) — slaat data permanent op
  4. I/O-apparaten — invoer (toetsenbord, muis) en uitvoer (scherm, printer)
  5. Bus — verbindt alle componenten met elkaar

|      CPU       |<----->|    RAM    |
| (ALU + CU)     |       +-----------+

   [Systeembus]

2.2 De CPU (processor)

📘 Definitie

De Central Processing Unit (CPU) is het "brein" van de computer. De CPU haalt instructies op uit het geheugen, interpreteert ze en voert ze uit. De snelheid van een CPU wordt uitgedrukt in GHz (gigahertz) en het aantal cores.

De CPU bestaat intern uit twee kerndelen:

ComponentNaamFunctie
ALUArithmetic Logic UnitVoert berekeningen en logische operaties uit
CUControl UnitHaalt instructies op, decodeert ze en coördineert de uitvoering

Kernbegrippen bij processors:

BegriffOmschrijving
KloksnelheidAantal bewerkingen per seconde (GHz). Hogere klok = sneller (maar niet altijd!)
CoresAparte verwerkingseenheden in één chip. Meer cores = beter voor parallelle taken
CacheSupersnel klein geheugen direct op de CPU (L1, L2, L3)
TDPThermal Design Power — hoe veel warmte (en stroom) de CPU verbruikt
64-bitKan 64-bit data per cyclus verwerken; alle moderne OS'en vereisen dit

2.3 Geheugen (RAM)

📘 Definitie

Random Access Memory (RAM) is het werkgeheugen van de computer. Het slaat tijdelijk de data op waarmee de CPU actief bezig is. RAM is vluchtig: bij uitschakeling gaat alle inhoud verloren.

RAM-types:

TypeBeschrijving
DDR4Huidig standaard in desktops en laptops
DDR5Nieuwste generatie — hogere snelheid en bandbreedte
LPDDR5Low-Power DDR5 — voor laptops en mobiele apparaten

Vuistregel voor geheugen:

  • 8 GB: minimum voor schoolgebruik
  • 16 GB: comfortabel voor kantoor en ontwikkeling
  • 32 GB+: dataverwerking, virtualisatie, gaming
⚠️ Let op

️ Let op

Meer RAM is niet altijd sneller. Een CPU kan alleen snel werken als er voldoende RAM is voor de actieve processen. Als het RAM volzit, gebruikt het OS swap (pagina-uitwisseling met de harde schijf) — dit is véél trager.

2.4 Opslag: HDD vs SSD

📘 Definitie

Een Hard Disk Drive (HDD) is een mechanisch opslagmedium met roterende schijven en een beweegbare lees/schrijfkop. Data wordt magnetisch opgeslagen.

📘 Definitie

Een Solid State Drive (SSD) is een elektronisch opslagmedium zonder bewegende onderdelen. Data wordt opgeslagen in flash-geheugen (NAND). Veel sneller en stiller dan een HDD.

EigenschapHDDSSD
Snelheid lezen~100–200 MB/s~500–7000 MB/s
Snelheid schrijven~100–150 MB/s~400–6000 MB/s
Prijs per GBGoedkoopDuurder (maar dalend)
CapaciteitTot 20 TB (consument)Tot 8 TB (consument)
SlagvastheidKwetsbaar (mechanisch)Robuust (geen bewegende delen)
GeluidHoorbaarStil
LevensduurSleet door bewegingBeperkt aantal schrijfcycli (TBW)
GebruikBulk-opslag, archief, NASOS-schijf, snelle werkopslag

SSD-formaten:

  • 2,5" SATA — oud formaat, langzamer
  • M.2 SATA — compact, zelfde snelheid als 2,5" SATA
  • M.2 NVMe (PCIe) — huidig standaard voor snelle SSD's

2.5 Processorarchitecturen: x86 vs ARM

📘 Definitie

De instructieset-architectuur bepaalt welke machinecodetaal de CPU begrijpt. De twee dominante architecturen zijn x86 (voor desktops en servers) en ARM (voor mobiel en steeds meer laptops).

Eigenschapx86 (AMD/Intel)ARM
Primair gebruikDesktop, laptop, serverSmartphone, tablet, embedded
VoorbeeldenIntel Core i7, AMD Ryzen 9Apple M4, Qualcomm Snapdragon, Raspberry Pi
StroomverbruikHogerLager — efficiënter
PrestatiesHoog (brute kracht)Hoog-efficiënt (prestatie/watt)
OS-ondersteuningWindows, Linux, macOS (pre-M1)Android, iOS, macOS (M1+), Windows ARM
💡 Tip

Apple schakelde in 2020 over van x86 (Intel) naar eigen ARM-chips (Apple Silicon M1/M2/M3/M4). Dit leverde enorm efficiëntere laptops op met betere batterijduur. Microsoft heeft een ARM-versie van Windows die draait op Qualcomm Snapdragon-processors.

2.6 Gegevenstransport: de bus

📘 Definitie

Een bus is een set van elektrische verbindingen (draden of sporen op een moederbord) waarover data, adressen en controleinfo tussen CPU, geheugen en randapparatuur worden uitgewisseld.

Drie soorten bussen:

  1. Databus — transporteert de eigenlijke data (8, 16, 32, 64 bit breed)
  2. Adresbus — geeft aan welk geheugenadres aangesproken wordt
  3. Controlebus — stuurt de timing en richting van het datatransport

Moderne bussen:

  • PCIe (PCI Express) — snelle interne bus voor GPU, SSD (NVMe), netwerkkaarten
  • USB — universele externe bus (USB 3.2, USB4, Thunderbolt)
  • SATA — voor HDD en oudere SSD's

2. Voorbeelden

🟢 Voorbeeld

Een netwerkbeheerder heeft een computer nodig voor:

  • Beheer van netwerkapparatuur via SSH en webinterface
  • Draaien van virtuele machines (VM's) voor testomgevingen
  • Verwerking van grote logbestanden

Geschikte configuratie:

  • CPU: Intel Core i7-13700 (16 cores) of AMD Ryzen 7 7700X
  • RAM: 32 GB DDR5 (voor VM's)
  • Opslag: 512 GB NVMe SSD (OS) + 2 TB HDD (archief/logs)
  • Architectuur: x86-64
🟢 Voorbeeld

Een UniFi Express 7 (de router die we later in het praktijkproject gebruiken) draait op een ARM-processor. Dit is bewust: ARM verbruikt minder stroom, genereert minder warmte, en is goedkoper voor apparaten die 24/7 aan staan. De UniFi Express 7 heeft geen ventilator nodig dankzij de lage TDP van de ARM-chip.

4. Samenvatting

📋 Samenvatting
  • Een computer bestaat uit CPU, RAM, opslag, I/O-apparaten en bussen.
  • De CPU (met ALU en CU) voert instructies uit; snelheid wordt bepaald door klok en cores.
  • RAM is vluchtig werkgeheugen; SSD/HDD is permanente opslag.
  • SSD is veel sneller dan HDD maar duurder per GB — gebruik SSD voor OS en actieve data, HDD voor bulk-archivering.
  • x86 (Intel/AMD) is dominant in desktops en servers; ARM is energiezuiniger en overheerst in mobiel en embedded systemen.
  • Bussen (PCIe, USB, SATA) verbinden de componenten intern en extern.
