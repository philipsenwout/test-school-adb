13. Theorie en aanpak

13.1 Het ontwerpproces

Een goed netwerkontwerp volgt een gestructureerd proces:


1. Analyse van de eisen
        ↓
2. Logisch ontwerp (IP-schema, topologie)
        ↓
3. Fysiek ontwerp (apparatuur, bekabeling)
        ↓
4. Implementatie (configuratie)
        ↓
5. Testen en verificatie
        ↓
6. Documentatie

13.2 Stap 1: Analyse van de eisen

Stel jezelf de volgende vragen:

  • Hoeveel apparaten worden verbonden?
  • Zijn er afzonderlijke afdelingen die netwerk-isolatie vereisen?
  • Is draadloze toegang nodig?
  • Wat is de verwachte groei (toekomstige schaalbaarheid)?
  • Welke diensten zijn nodig (DHCP, DNS, file sharing, internet)?
  • Zijn er beveiligingseisen (firewallregels, VLAN's)?

13.3 Stap 2: Logisch ontwerp

Het logisch ontwerp beschrijft:

  1. Adresseringsschema: welke IP-adressen gebruik je, hoe verdeel je de subnetten?
  2. Topologieschema: hoe zijn de apparaten logisch verbonden?
  3. Protocol- en dienstenoverzicht: welke protocollen/diensten (DHCP, DNS, statische routing) gebruik je?

13.4 Cisco Packet Tracer — werkwijze

Cisco Packet Tracer is een gratis netwerksimulator van Cisco. Je kunt er routers, switches, pc's, servers en wireless access points in toevoegen en configureren.

13.4.1 Apparaten toevoegen

  1. Open Packet Tracer.
  2. Kies in het onderpaneel: Routers → sleep een Cisco 2911 naar het canvas.
  3. Kies Switches → sleep een Cisco 2960.
  4. Kies End Devices → sleep PC's.
  5. Kies het juiste kabeltype (let op!):

- Straight-through (zwart): PC/server → switch

- Cross-over (blauw): switch → switch, router → router (of gebruik Auto voor moderne apparaten)

- Serieel (rood): WAN-verbindingen tussen routers (DCE/DTE)

- Console (lichtblauw): beheerconnectie van PC naar router/switch

13.4.2 Configuratie via CLI

Dubbelklik op een router of switch → tabblad CLI.


! Router configureren
Router> enable
Router# configure terminal
Router(config)# hostname R1
R1(config)# interface GigabitEthernet0/0
R1(config-if)# ip address 192.168.1.1 255.255.255.0
R1(config-if)# no shutdown
R1(config-if)# exit

13.4.3 PC configureren

Dubbelklik op PC → Desktop → IP Configuration:

  • IP Address: 192.168.1.10
  • Subnet Mask: 255.255.255.0
  • Default Gateway: 192.168.1.1
  • DNS Server: 192.168.2.5 (als je een DNS-server hebt)

13.4.4 Testen

  • Desktop → Command Prompt → ping 192.168.1.1 (router testen)
  • Desktop → Command Prompt → ping 192.168.2.5 (cross-network testen)

13.5 Documentatiestandaarden

Professionele documentatie bevat:

  1. Netwerkdiagram (fysiek + logisch)
  2. Adresseringstabel (alle apparaten, interfaces, IP's)
  3. Apparatuuroplijsting (wat, hoeveel, merk/model)
  4. Configuratiebestanden (IOS-configs van routers/switches)
  5. Testresultaten (screenshots van pings, show-commando's)
💡 Tip

Gebruik een naamconventie voor je apparaten: R[nummer] voor routers, SW[nummer] voor switches, SRV[naam] voor servers. Consistentie maakt documentatie leesbaar.

13. Simulatieproject

Projectopdracht: Kantoornetwerk "DataBrussels"

Scenario:

Het bedrijf DataBrussels heeft twee afdelingen: Administratie (10 pc's) en Ontwikkeling (8 pc's). Er is ook een serverruimte (2 servers: DNS/DHCP en fileserver).

Je hebt het adresblok 172.16.0.0/24 ter beschikking. Ontwerp en implementeer dit netwerk in Cisco Packet Tracer.

Eisen:

  • Eén router verbindt de drie subnetten.
  • Elke afdeling heeft een eigen subnet (VLSM).
  • De DHCP-server deelt automatisch IP-adressen uit aan de pc's.
  • De servers hebben vaste (statische) IP-adressen.
  • Communicatie tussen alle afdelingen moet mogelijk zijn via de router.
  • Elk apparaat heeft een correcte default gateway.

Deliverables:

  1. Adresseringsschema (tabel)
  2. Netwerkdiagram (Packet Tracer screenshot)
  3. Configuratie-excerpts van de router (show running-config)
  4. Testrapport: screenshots van pings tussen alle drie subnetten

3. Oefeningen

Oefening 13.1 Adresseringsschema voor DataBrussels

Het bedrijf DataBrussels heeft het adresblok 172.16.0.0/24. Je moet drie subnetten ontwerpen voor Administratie (10 hosts), Ontwikkeling (8 hosts) en Servers (2 hosts).

Vraag 1

Vraag 2

Vraag 3

4. Samenvatting

📋 Samenvatting
  • Netwerkontwerp volgt een proces: analyse → logisch ontwerp → fysiek ontwerp → implementatie → test → documentatie.
  • Een adresseringsschema legt vast welke IP-adressen en subnetten gebruikt worden voor elk apparaat.
  • Cisco Packet Tracer simuleert netwerken zonder echte hardware — ideaal om te oefenen.
  • Documentatie is essentieel in professionele netwerkomgevingen: adresseringstabel, diagrammen, configs, testresultaten.
  • Teamwerk en planning zijn sleutelcompetenties voor een netwerkbeheerder.

Huistaak H13

Ontwerp een netwerk voor jouw thuis- of schoolsituatie:

  3. Maak een eenvoudig logisch diagram (met de hand of digitaal — bv. draw.io).
  4. Beschrijf welke apparaten nodig zijn (router, switch, access point, ...).
  5. Optioneel: bouw het na in Cisco Packet Tracer.
