7.1 Waarom subnetten?

Een groot netwerk (192.168.1.0/24) met 254 toestellen heeft nadelen:

  • Alle broadcasts gaan naar alle 254 apparaten
  • Als er een probleem is, heeft het invloed op iedereen
  • Geen scheiding tussen afdelingen

Subnetting laat toe een groot netwerk op te splitsen in kleinere stukken.

📘 Definitie

Een subnet is een logisch afgebakend deel van een groter netwerk. Elk subnet heeft zijn eigen netwerkadres, broadcastadres en hostbereik. Apparaten in verschillende subnetten communiceren via een router.

💡 Tip

De stapgrootte vertelt je hoe ver elk subnet van het volgende begint. Bij /26 is de stapgrootte 64: de subnetten beginnen op 0, 64, 128, 192.

7.3 Stappenplan gelijke subnetten

Gegeven: 192.168.1.0/24, opdelen in 4 gelijke subnetten.

Stap 1: Zoek in de tabel een prefix die minstens 4 subnetten geeft.

  • /26 geeft 4 subnetten van elk 62 hosts ✓

Stap 2: Bepaal de stapgrootte.

  • /26 → stapgrootte = 64

Stap 3: Stel de subnettabel op (begin bij 0, tel telkens de stapgrootte op):

SubnetNetwerkadresEerste hostLaatste hostBroadcast
1192.168.1.0/26192.168.1.1192.168.1.62192.168.1.63
2192.168.1.64/26192.168.1.65192.168.1.126192.168.1.127
3192.168.1.128/26192.168.1.129192.168.1.190192.168.1.191
4192.168.1.192/26192.168.1.193192.168.1.254192.168.1.255
💡 Tip
  • Netwerkadres = eerste adres van het subnet (niet bruikbaar)
  • Broadcast = laatste adres van het subnet (niet bruikbaar)
  • Eerste bruikbare host = netwerkadres + 1
  • Laatste bruikbare host = broadcast - 1

7.4 Voorbeelden

🟢 Voorbeeld

Een school heeft het netwerk 10.0.0.0/24 en wil het opdelen in 2 gelijke stukken.

  • /25 geeft 2 subnetten van elk 126 hosts
  • Stapgrootte = 128

| Subnet | Netwerkadres | Eerste host | Laatste host | Broadcast |

|--------|-------------|-------------|--------------|-----------|

| Leerlingen | 10.0.0.0/25 | 10.0.0.1 | 10.0.0.126 | 10.0.0.127 |

| Leerkrachten | 10.0.0.128/25 | 10.0.0.129 | 10.0.0.254 | 10.0.0.255 |

🟢 Voorbeeld

Gegeven: 192.168.5.100/26

  • /26 → stapgrootte = 64
  • Subnetten: 0–63, 64–127, 128–191 ← maar .100 zit in 64–127!
  • Subnet 1: 192.168.5.0/26 (hosts .1 t/m .62)
  • Subnet 2: 192.168.5.64/26 (hosts .65 t/m .126) ← .100 zit hier

Netwerkadres: 192.168.5.64, Broadcast: 192.168.5.127

Samenvatting

📋 Samenvatting
  • Subnetting verdeelt een groot netwerk in kleinere segmenten.
  • Gebruik de referentietabel: kies het prefix dat genoeg hosts geeft.
  • Stapgrootte = de afstand tussen twee opeenvolgende netwerkadresssen.
  • Netwerkadres = eerste adres (niet bruikbaar). Broadcast = laatste adres (niet bruikbaar).
  • Eerste bruikbare host = netwerkadres + 1. Laatste bruikbare host = broadcast - 1.
