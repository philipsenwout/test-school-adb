8.1 De transportlaag

De transportlaag (OSI Laag 4) zorgt ervoor dat data bij de juiste toepassing aankomt. Het IP-adres zegt welke computer het is; het poortnummer zegt welk programma op die computer.

Voorbeeld: Als je tegelijk een website bezoekt en een Teams-gesprek hebt, weet je computer welke data bij je browser hoort en welke bij Teams — dankzij de poortnummers.

8.2 Poortnummers

📘 Definitie

Een poortnummer is een getal dat een specifieke toepassing op een computer identificeert. Samen met het IP-adres weet het netwerk precies waar data naartoe moet.

Meest gebruikte poortnummers (handig om te kennen):

PoortDienstProtocol
80HTTP (websites)TCP
443HTTPS (veilige websites)TCP
22SSH (veilige verbinding op afstand)TCP
53DNS (domeinnamen opzoeken)TCP/UDP
67/68DHCP (automatisch IP-adres)UDP

8.3 TCP — betrouwbaar maar iets trager

📘 Definitie

TCP (Transmission Control Protocol) is een verbindingsgericht protocol. Het zorgt ervoor dat alle data aankomt, in de juiste volgorde, zonder fouten.

Hoe TCP een verbinding opstart — de three-way handshake:

Voordat TCP data verstuurt, geven client en server elkaar een "teken":


Client                    Server
  |                          |
  |------ SYN ------------> |   "Mag ik verbinden?"
  |                          |
  |<----- SYN-ACK --------- |   "Ja, mag. Ik ook?"
  |                          |
  |------ ACK ------------> |   "Akkoord!"
  |                          |
  |<====== Data ============>|
  • SYN = "Ik wil verbinden"
  • SYN-ACK = "Begrepen, ik ook"
  • ACK = "OK, we zijn verbonden"

Figuur 8.1 — De drie stappen van een TCP-verbinding

Wanneer kies je TCP?

  • Websites laden (HTTP/HTTPS)
  • E-mail versturen
  • Bestanden kopiëren
  • SSH-verbinding

8.4 UDP — snel maar zonder garanties

📘 Definitie

UDP (User Datagram Protocol) is een verbindingsloos protocol. Het verstuurt data zo snel mogelijk, zonder te controleren of alles aankomt.

UDP stuurt data gewoon weg — geen handshake, geen bevestiging. Als een pakket verloren gaat, wordt het niet opnieuw verstuurd.

Wanneer kies je UDP?

  • Videobellen (Teams, Zoom)
  • Muziek/video streamen
  • Online games
  • DNS-opzoekingen
💡 Tip

Bij een Teams-gesprek merk je een hapering als een pakket verloren gaat. Dat is UDP in actie: het gaat gewoon verder. Bij TCP zou je wachten tot het pakket opnieuw verstuurd is — te laat voor een live gesprek.

8.5 TCP vs UDP — vergelijking

EigenschapTCPUDP
Verbinding opzettenJa (handshake)Nee
BetrouwbaarheidJaNee
SnelheidIets tragerSnel
GebruikWeb, e-mail, SSH, bestandenStreaming, gaming, VoIP, DNS

Voorbeelden

🟢 Voorbeeld

Je laadt een webpagina via HTTPS. Eén gemist pakket = de pagina laadt niet correct. Dus: TCP — betrouwbaarheid is hier het belangrijkste.

Je speelt een online game. Een gemist pakket = een lichte hapering die vanzelf herstelt. Latency (vertraging) is hier het grootste probleem. Dus: UDP — snelheid primeert.

🟢 Voorbeeld

Je bezoekt https://www.school.be. Je browser maakt een verbinding naar poort 443 (HTTPS) op de server van de school. Tegelijk doet je computer een DNS-opzoeking op poort 53 (UDP) om het IP-adres van www.school.be te vinden.

Samenvatting

📋 Samenvatting
  • De transportlaag gebruikt poortnummers om data bij de juiste toepassing te brengen.
  • TCP: verbindingsgericht, betrouwbaar, three-way handshake (SYN → SYN-ACK → ACK). Gebruikt voor web, e-mail, bestanden.
  • UDP: verbindingsloos, snel, geen garanties. Gebruikt voor streaming, gaming, VoIP, DNS.
  • Belangrijkste poorten: HTTP=80, HTTPS=443, SSH=22, DNS=53, DHCP=67/68.
