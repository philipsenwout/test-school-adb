9.1 DNS — domeinnamen vertalen

📘 Definitie

DNS (Domain Name System) is een systeem dat domeinnamen (bv. www.google.be) vertaalt naar IP-adressen (bv. 142.250.74.46). Zonder DNS zou je elk IP-adres uit het hoofd moeten kennen.

Poort: 53 (TCP/UDP)

Hoe werkt DNS?

  1. Je typt www.school.be in je browser.
  2. Je computer vraagt aan de DNS-server: "Wat is het IP-adres van www.school.be?"
  3. De DNS-server antwoordt: "Het IP-adres is 203.0.113.10."
  4. Je browser maakt verbinding met dat IP-adres.

💡 Tip

Als je een webpagina niet kan openen maar het IP-adres wel kunt pingen, is er waarschijnlijk een DNS-probleem. DNS-problemen herken je aan de foutmelding "Kan de server niet vinden" of "ERR_NAME_NOT_RESOLVED".

9.2 DHCP — automatisch IP-adres toewijzen

📘 Definitie

DHCP (Dynamic Host Configuration Protocol) is een protocol waarmee een server automatisch een IP-adres, subnetmasker, gateway en DNS-server uitdeelt aan nieuwe apparaten op het netwerk.

Poorten: 67 (server) / 68 (client) — UDP

Het DORA-proces:

Wanneer je een apparaat aansluit op het netwerk, verlopen er 4 stappen:


Client                          DHCP-Server
  |                               |
  |--- DISCOVER (broadcast) ----->|   "Is er een DHCP-server?"
  |                               |
  |<-- OFFER --------------------|   "Hier is een aanbieding: 192.168.1.50"
  |                               |
  |--- REQUEST ------------------>|   "Ik accepteer 192.168.1.50"
  |                               |
  |<-- ACK ----------------------|   "OK! Het adres is voor jou."
  • Discover — het apparaat zoekt een DHCP-server
  • Offer — de server biedt een IP-adres aan
  • Request — het apparaat accepteert het aanbod
  • ACK — de server bevestigt

!DHCP DORA-proces

Figuur 9.2 — De vier stappen van het DORA-proces

DHCP geeft ook mee:

  • Subnetmasker
  • Default gateway (adres van de router)
  • DNS-serveradressen
⚠️ Let op

️ Let op

Als een apparaat geen DHCP-antwoord krijgt, wijst het zichzelf een adres toe dat begint met 169.254.x.x. Dit is een teken dat er een probleem is met de DHCP-server of de verbinding.

9.3 HTTP en HTTPS — websites bezoeken

📘 Definitie

HTTP (HyperText Transfer Protocol) is het protocol voor het uitwisselen van webpagina's. Een browser verstuurt een aanvraag; de server stuurt de webpagina terug.

Poort: 80 (HTTP), 443 (HTTPS)

📘 Definitie

HTTPS (HTTP Secure) is HTTP met beveiliging. Alle data tussen browser en server is versleuteld via TLS. Niemand die het netwerkverkeer onderschept kan de inhoud lezen.

Verschil HTTP vs HTTPS:

HTTPHTTPS
VersleutelingGeenJa (TLS)
Poort80443
VeiligheidOnveiligVeilig
GebruikVerouderdStandaard vandaag
⚠️ Let op

️ Let op

Gebruik nooit HTTP op sites waar je inlogt of betaalt. Je wachtwoord en gegevens zijn leesbaar voor iedereen op hetzelfde netwerk. Moderne browsers tonen "Niet veilig" bij HTTP-sites.

Voorbeelden

🟢 Voorbeeld

Open een opdrachtprompt en typ:


nslookup www.vrt.be

Je ziet het IP-adres van de VRT-website. Dit is precies wat DNS doet: de naam omzetten naar een adres.

🟢 Voorbeeld

Open een opdrachtprompt en typ:


ipconfig /all

Je ziet bij je netwerkadapter: "DHCP ingeschakeld: Ja" en het IP-adres van de DHCP-server. Dit adres heeft jouw computer automatisch gekregen via het DORA-proces.

Samenvatting

📋 Samenvatting
  • DNS (poort 53): vertaalt domeinnamen naar IP-adressen.
  • DHCP (poort 67/68): deelt automatisch IP-adressen uit via het DORA-proces (Discover → Offer → Request → ACK).
  • HTTP (poort 80): webprotocol, onveilig. HTTPS (poort 443): webprotocol met versleuteling — gebruik altijd HTTPS.
  • Andere protocollen: SSH (22), FTP/SFTP (21/22), SMTP (587), IMAP (993), NTP (123).
