10.1 Wat doet een router?

📘 Definitie

Een router is een netwerkapparaat dat pakketten doorstuurt tussen verschillende netwerken. Hij werkt op basis van IP-adressen.

Vergelijking switch vs router:

SwitchRouter
VerbindtApparaten in hetzelfde netwerkVerschillende netwerken
Werkt opMAC-adres (laag 2)IP-adres (laag 3)
VoorbeeldComputers in één klas verbindenSchoolnetwerk verbinden met het internet

!Router vs Switch

Figuur 10.1 — Een router verbindt twee verschillende netwerken

Voorbeeld:


Netwerk A: 192.168.1.0/24          Netwerk B: 192.168.2.0/24
     |                                   |
PC-A (192.168.1.10)               PC-B (192.168.2.10)
     |                                   |
     +------ Switch A --- ROUTER --- Switch B -----+
                       192.168.1.1   192.168.2.1

PC-A en PC-B zitten in verschillende netwerken. Alleen de router kan pakketten van het ene naar het andere netwerk sturen.

10.2 Default gateway

📘 Definitie

De default gateway is het IP-adres van de router op jouw lokale netwerk. Als je iets wil bereiken buiten jouw subnet, stuur je het naar de default gateway.

Voorbeeld:

  • Jouw PC: 192.168.1.10/24, gateway: 192.168.1.1
  • Je wil 8.8.8.8 (Google DNS) bereiken
  • 8.8.8.8 zit niet in 192.168.1.0/24 → stuur naar gateway 192.168.1.1
  • De router stuurt het pakket verder naar het internet
⚠️ Let op

️ Veelgemaakte fout

Als de default gateway verkeerd is ingesteld, kan je lokale apparaten bereiken maar niets buiten je netwerk (geen internet, geen andere subnetten). Controleer de gateway altijd als eerste bij verbindingsproblemen.

10.3 De routeringstabel

📘 Definitie

Een routeringstabel is een lijst in de router met alle bekende bestemmingen en hoe je er komt (via welke interface of next-hop).

Hoe werkt het?

Wanneer een pakket aankomt, kijkt de router in zijn tabel:

  1. Zit de bestemming in een rechtstreeks verbonden netwerk? → stuur er naartoe
  2. Is er een specifieke route? → volg die route
  3. Geen match? → gebruik de default route (richting internet)

!Routeringstabel werking

Figuur 10.2 — De router kijkt in zijn tabel om te beslissen waar een pakket naartoe gaat

Eenvoudig voorbeeld van een routeringstabel:

BestemmingVia
192.168.1.0/24Directe verbinding (interface links)
192.168.2.0/24Directe verbinding (interface rechts)
Alles andere (0.0.0.0/0)10.0.0.1 (internet-router van ISP)

De laatste regel (0.0.0.0/0) is de default route — alles waarvoor de router geen specifieke route heeft, stuur je daarheen.

10.4 Windows Server vs Linux als server

In bedrijven draaien netwerkdiensten (DNS, DHCP, webserver) op een serverbesturingssysteem. De twee meest gebruikte zijn:

EigenschapWindows ServerLinux (Ubuntu, Debian …)
KostBetaalde licentieGratis
BeheerGrafische interface + PowerShellVooral via terminal
GebruikBedrijfsomgevingen, Active DirectoryWebservers, cloud, databases
CLIPowerShellBash
💡 Tip

Als data- en applicatiebeheerder zal je beide tegenkomen. Windows Server in kantooromgevingen; Linux op webservers en in de cloud.

Voorbeelden

🟢 Voorbeeld

Op een Windows-pc kan je de default gateway instellen via:

  • Configuratiescherm → Netwerkverbindingen → Eigenschappen → IPv4
  • Of via de opdrachtprompt: ipconfig toont de huidige gateway

Als de gateway verkeerd is ingesteld, zie je: ping naar lokale IP werkt, ping naar 8.8.8.8 mislukt.

🟢 Voorbeeld

PC-A (192.168.1.10) kan het internet niet bereiken.

Stap 1: ping 192.168.1.1 → is de gateway bereikbaar?

  • OK → probleem zit verder in het netwerk (bij de ISP of de router zelf)
  • Mislukt → probleem is lokaal (verkeerde gateway, kabel, switch)

Oefeningen

Oefening 10.1 Router of switch?

Kies voor elk scenario het juiste netwerkapparaat: een router of een switch. Denk aan het verschil: een switch verbindt apparaten in hetzelfde netwerk, een router verbindt verschillende netwerken.

Vraag 1
Je wil 5 computers in hetzelfde klaslokaal met elkaar verbinden. Wat heb je nodig?
Vraag 2
Je wil het schoolnetwerk (192.168.1.x) verbinden met het internet. Wat heb je nodig?
Vraag 3
Op welke laag van het OSI-model werkt een switch?
Oefening 10.2 Default gateway en routering

PC-A heeft IP-adres 10.5.20.15 met subnetmasker /24 en default gateway 10.5.20.1. Beantwoord de vragen over hoe het verkeer wordt gerouteerd.

Vraag 1
PC-A wil 10.5.20.30 bereiken. Gaat dit via de default gateway?
Vraag 2
PC-A wil 10.5.30.5 bereiken. Gaat dit via de default gateway?
Vraag 3
PC-A kan 10.5.20.30 pingen maar niet 8.8.8.8. Wat is de meest waarschijnlijke oorzaak?

Samenvatting

📋 Samenvatting
  • Een router verbindt verschillende netwerken op basis van IP-adressen. Een switch verbindt apparaten in hetzelfde netwerk.
  • De default gateway is de router op jouw subnet. Verkeer buiten jouw netwerk gaat altijd via de gateway.
  • De routeringstabel bevat alle bekende routes. De default route (0.0.0.0/0) is de vangnet voor al het andere verkeer.
  • Windows Server is sterk in bedrijfsomgevingen met Active Directory. Linux domineert op webservers en in de cloud.

Huistaak

Huistaak H10

Controleer de netwerkconfiguratie van je eigen toestel thuis.

  1. Open een opdrachtprompt (Windows) of terminal (Mac/Linux) en typ ipconfig (Windows) of ip addr (Linux/Mac).
  2. Noteer: jouw IP-adres, subnetmasker en default gateway.
  3. Ping de default gateway: ping [gateway-adres]. Werkt dit?
  4. Ping 8.8.8.8. Werkt dit ook?
  5. Leg in eigen woorden uit wat het verschil is tussen de twee pings.
