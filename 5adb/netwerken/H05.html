5. Theorie

5.1 De datalinklaag (OSI Laag 2)

De datalinklaag is verantwoordelijk voor de betrouwbare overdracht van data tussen twee rechtstreeks verbonden apparaten op hetzelfde netwerksegment. Ze werkt op het niveau van frames en gebruikt MAC-adressen voor adressering.

De datalinklaag is verdeeld in twee sublagen:

  • MAC (Media Access Control): adressering en toegangscontrole
  • LLC (Logical Link Control): interface met de netwerklaag (laag 3)

5.2 MAC-adressen

📘 Definitie

Een MAC-adres (Media Access Control-adres) is een 48-bit hardwareadres dat uniek is per netwerkinterface (NIC). Het wordt door de fabrikant in de hardware ingebrand en is (normaal gezien) wereldwijd uniek.

Notatie: AA:BB:CC:DD:EE:FF (6 bytes, hexadecimaal, gescheiden door dubbele punten of koppeltekens)

Opbouw van een MAC-adres:


AA:BB:CC | DD:EE:FF
   OUI   |  NIC-specific
(Fabrikant) (Apparaatspecifiek)
  • OUI (Organizationally Unique Identifier): de eerste 3 bytes — identificeert de fabrikant

- 00:50:56 → VMware

- 00:1A:2B → Cisco

- DC:A6:32 → Raspberry Pi Foundation

  • NIC-specifiek deel: de laatste 3 bytes — uniek per apparaat van die fabrikant

Speciale MAC-adressen:

  • FF:FF:FF:FF:FF:FF — broadcastadres (stuur naar alle apparaten op het segment)
  • 01:00:5E:xx:xx:xx — multicast-prefix
💡 Tip

Je eigen MAC-adressen opvragen:


# Windows
ipconfig /all
# Linux/Mac
ip link show
ifconfig -a

5.3 Ethernet-frames

📘 Definitie

Een frame is de PDU (Protocol Data Unit) van de datalinklaag. Het omvat de eigenlijke data (payload) van de bovenliggende lagen, aangevuld met een header en trailer met adresseringsinfo en foutdetectie.

Structuur van een Ethernet II frame (meest gebruikt):

| Preamble | SFD      | Doel | Bron | EtherType| Payload  | FCS |
| 7 bytes  | 1 byte   | MAC  | MAC  | 2 bytes  | 46-1500B |4 B  |
|          |          | 6 B  | 6 B  |          |          |     |
+----------+----------+------+------+----------+----------+-----+
VeldGrootteFunctie
Preamble7 bytesSynchronisatie (10101010 patroon)
SFD (Start Frame Delimiter)1 byteGeeft begin van frame aan (10101011)

⚠️ Let op

️ Let op

De maximale payload van een standaard Ethernet-frame is 1500 bytes (MTU = Maximum Transmission Unit). Grotere pakketten moeten gefragmenteerd worden.

5.4 Hub vs switch

5.4.1 Hub (verouderd)

📘 Definitie

Een hub is een laag-1-apparaat dat een binnenkomend signaal naar alle poorten doorstuurt. Er is geen intelligentie — de hub "weet" niets over MAC-adressen.

Problemen met hubs:

  • Collisions (botsingen): als twee apparaten tegelijk zenden, botsing → data verloren
  • Alle data zichtbaar voor alle apparaten → beveiligingsrisico
  • Laag efficiënt bij meer apparaten
  • Hubs zijn verouderd en worden niet meer geïnstalleerd

CSMA/CD (Carrier Sense Multiple Access with Collision Detection) is het protocol dat Ethernet-apparaten gebruikten om botsingen te detecteren en op te lossen op gedeelde media (hubs):

  1. Carrier Sense: luister eerst of het medium vrij is
  2. Multiple Access: meerdere apparaten mogen het medium gebruiken
  3. Collision Detection: detecteer botsing → stop zenden → wacht willekeurige tijd → probeer opnieuw

Moderne switches maken CSMA/CD overbodig (full-duplex).

5.4.2 Switch

📘 Definitie

Een switch is een laag-2-apparaat dat frames doorsturen op basis van MAC-adressen. De switch leert welk MAC-adres op welke poort zit (MAC-adrestabel) en stuurt frames enkel door naar de juiste poort.

Hoe een switch leert (MAC-adrestabel):

  1. Leren: wanneer een frame binnenkomt op poort X met bron-MAC AA:BB:CC:DD:EE:FF, wordt het adres geregistreerd als "op poort X te bereiken".
  2. Doorsturen (Unicast): als het doel-MAC bekend is → stuur enkel naar de juiste poort.
  3. Flooding (Unknown Unicast): als het doel-MAC onbekend is → stuur naar alle poorten behalve de bronpoort (net als een hub, maar tijdelijk).
  4. Filtering: als bron- en doel-MAC op dezelfde poort zitten → frame niet doorsturen.

Voordelen van een switch ten opzichte van een hub:

EigenschapHubSwitch
OSI-laagLaag 1Laag 2
IntelligentieGeenMAC-adrestabel
BotsingenJa (collision domain)Nee (full-duplex, elk poort eigen collision domain)
BeveiligingGeenBetere isolatie per poort
BandbreedteGedeeldDedicated per poort

5.5 ARP — Address Resolution Protocol

📘 Definitie

ARP (Address Resolution Protocol) is een protocol op de grens van laag 2 en laag 3. Het vertaalt een IP-adres naar het corresponderende MAC-adres op hetzelfde netwerksegment.

Waarom is ARP nodig?

Een IP-pakket weet het doel-IP, maar om het via Ethernet te versturen, heeft het apparaat het doel-MAC nodig. ARP vraagt dit op.

ARP-werkwijze:

  1. PC-A wil communiceren met PC-B (192.168.1.20), maar kent het MAC-adres niet.
  2. PC-A stuurt een ARP Request (broadcast FF:FF:FF:FF:FF:FF): "Wie heeft IP 192.168.1.20? Geef me je MAC-adres."
  3. PC-B ontvangt de broadcast, herkent zijn eigen IP en antwoordt met een ARP Reply (unicast): "IP 192.168.1.20 heeft MAC BB:CC:DD:EE:FF:AA."
  4. PC-A slaat dit op in zijn ARP-cache (tijdelijk geheugen).

# ARP-cache bekijken
arp -a          # Windows en Linux
⚠️ Let op

️ Let op — ARP Spoofing

ARP heeft geen authenticatie. Een aanvaller kan valse ARP Reply-berichten sturen om het verkeer van andere apparaten om te leiden naar zijn eigen machine. Dit noemen we een ARP Spoofing of ARP Poisoning-aanval — een veelgebruikte techniek voor man-in-the-middle-aanvallen. We bespreken dit verder in het cyberveiligheidshoofdstuk.

5. Voorbeelden

🟢 Voorbeeld

In een netwerk met een switch zijn drie pc's verbonden:

  • PC-A (MAC: AA:AA:AA:AA:AA:AA) op poort 1
  • PC-B (MAC: BB:BB:BB:BB:BB:BB) op poort 2
  • PC-C (MAC: CC:CC:CC:CC:CC:CC) op poort 3

Scenario: PC-A verstuurt een frame naar PC-B.

  1. Switch ontvangt frame op poort 1 met bron-MAC AA:AA:AA:AA:AA:AA → leert: poort 1 = AA:AA:AA:AA:AA:AA
  2. Doel-MAC is BB:BB:BB:BB:BB:BB → bekend (poort 2) → stuur frame enkel naar poort 2
  3. PC-C ontvangt niets → efficiënt en veilig
🟢 Voorbeeld

In Packet Tracer:

  1. Voeg twee pc's en een switch toe.
  2. Configureer IP-adressen: PC0 = 192.168.1.1/24, PC1 = 192.168.1.2/24.
  3. Open de command prompt van PC0 en voer arp -a in → leeg (nog geen communicatie).
  4. Ping PC1 (ping 192.168.1.2).
  5. Voer opnieuw arp -a in → je ziet nu het MAC-adres van PC1 in de ARP-cache.

4. Samenvatting

📋 Samenvatting
  • MAC-adressen zijn 48-bit hardwareadressen (6 bytes, hexadecimaal). De eerste 3 bytes = OUI (fabrikant).
  • Een Ethernet-frame bevat: preamble, doel-MAC, bron-MAC, EtherType, payload (max. 1500 bytes) en FCS.
  • Een hub stuurt alles naar alle poorten (laag 1) — inefficiënt en onveilig. Verouderd.
  • Een switch leert MAC-adressen per poort (MAC-adrestabel) en stuurt frames gericht door (laag 2).
  • ARP vertaalt IP-adressen naar MAC-adressen via een broadcastvraag en unicast-antwoord.
  • ARP heeft geen authenticatie → kwetsbaar voor ARP Spoofing (man-in-the-middle-aanval).

5. Huistaak

Huistaak H05

Gebruik Cisco Packet Tracer (gratis te downloaden op de Cisco Netacad-website).

  1. Maak een eenvoudig netwerk: 3 pc's aangesloten op een switch.
  2. Wijs IP-adressen toe: 192.168.1.1, 192.168.1.2, 192.168.1.3 (subnetmasker 255.255.255.0).
  3. Ping van PC0 naar PC1 en naar PC2.
  4. Bekijk in Packet Tracer de MAC-adrestabel van de switch (via CLI: show mac address-table).
  5. Maak een screenshot en breng mee naar de les. Leg uit welke MAC-adressen je ziet en op welke poorten.
