H16 — App-ontwikkeling 5ADB

Typografie

ADB10De leerlingen werken een applicatie uit binnen de opgelegde standaardsoftware of het IT-platform.
🎯 Leerdoelen
  • Je kan uitleggen hoe mensen tekst op een scherm lezen en waarom typografie belangrijk is
  • Je kan de juiste CSS-eenheden kiezen voor lettergrootte (rem) en regellengte (ch)
  • Je kan font-size, line-height, font-weight en letter-spacing correct instellen
  • Je kan een extern lettertype van Google Fonts koppelen aan een webpagina
  • Je kan een passend lettertypepaar kiezen (heading + body)
  • Je kan de ideale regellengte instellen met max-width

1. Waarom typografie?

Veel websites bestaan voor een groot deel uit tekst. Als die tekst moeilijk te lezen is — te klein, te dicht op elkaar, regels te lang — haken bezoekers af. Goede typografie is onzichtbaar: ze maakt het lezen moeiteloos.

Als front-end developer ben je automatisch ook typograaf. Je maakt keuzes over lettertype, grootte, regelafstand en regellengte — of je het weet of niet.

2. Hoe lezen mensen tekst?

Je denkt misschien dat je letter per letter leest. Dat is niet zo. Onze ogen lezen woordbeelden, geen losse letters. Ze bewegen in kleine sprongetjes over de tekst, met korte pauzepunten.

De rol van typografie: zorg ervoor dat die sprongetjes zo moeiteloos mogelijk verlopen.

3. Regellengte

Een te lange tekstlijn is vermoeiend: je ogen moeten een grote sprong maken van het einde van een regel naar het begin van de volgende. Precies daarom werken kranten met smalle kolommen.

Richtlijn: 50–90 tekens per regel, ideaal rond de 70. In CSS gebruik je max-width op de tekstcontainer:

CSS
/* Met ch-eenheid: 1ch ≈ breedte van één "0"-karakter */
p {
    max-width: 70ch;
}

/* Of met em */
.artikel {
    max-width: 45em;
}

4. Lettergrootte (font-size)

Gebruik altijd rem voor lettergroottes op het web — nooit px.

EenheidRelatief ten opzichte vanGebruik
pxNiets — vaste grootteVermijden voor tekst
emHet huidige elementMargins, padding
remHet root-element (<html>)Lettergroottes
CSS
html {
    font-size: 100%; /* = de browservoorkeur van de gebruiker */
}

body { font-size: 1rem; }   /* = 16px */
h1   { font-size: 2.5rem; } /* = 40px */
h2   { font-size: 1.75rem; }/* = 28px */
h3   { font-size: 1.25rem; }/* = 20px */
p    { font-size: 1rem; }   /* = 16px */

5. Regelafstand (line-height)

CSS
/* Gebruik een eenheidsloos getal — vermenigvuldigingsfactor */
p {
    line-height: 1.6;
}

/* NIET dit: */
p {
    line-height: 24px; /* vast — schaalt niet mee */
}

Richtlijn:

  • Lopende tekst (p): 1.5–1.7
  • Hoofdingen (h1, h2): 1.1–1.3 (korter, want grotere letters)

6. Lettertypefamilies en Google Fonts

6.1 Websafe lettertypes

CSS
body {
    font-family: Arial, Helvetica, sans-serif;
}

code {
    font-family: "Courier New", Courier, monospace;
}

6.2 Google Fonts

HTML
<head>
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  <link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;700&family=Playfair+Display:wght@700&display=swap" rel="stylesheet">
  <link rel="stylesheet" href="stijl.css">
</head>
CSS
body {
    font-family: "Inter", sans-serif;
}

h1, h2, h3 {
    font-family: "Playfair Display", serif;
}

7. Lettertypeparen kiezen

Een goede webpagina gebruikt twee lettertypes: één voor hoofdingen, één voor lopende tekst.

Gebruik nooit meer dan twee lettertypes op één pagina.

8. Uitlijning en overige teksteffecten

CSS
/* Uitlijning */
p    { text-align: left; }    /* standaard — altijd voor lopende tekst */
h1   { text-align: center; }  /* voor titels */

/* Hoofdletters */
h1 { text-transform: uppercase; }   /* ALLES IN HOOFDLETTERS */
h2 { text-transform: capitalize; }  /* Eerste Letter Elk Woord */

/* Letterafstand */
h1 { letter-spacing: 0.05em; }

/* Onderlijn */
a  { text-decoration: none; }       /* verwijdert standaard onderlijning */
❌ Vermijd

Gebruik text-align: justify nooit op het web — het creëert lelijke, ongelijke spaties tussen woorden.

Oefeningen

ADB10 Oefening 1 — Lettertypes koppelen

Maak een HTML-pagina artikel.html:

  1. Koppel twee lettertypes van Google Fonts (heading + body naar keuze)
  2. Schrijf een kort artikel (hoofding h1, twee h2-subkoppen, drie paragrafen)
  3. Stel een typografisch systeem in via CSS: lettergrootte in rem, line-height, max-width op de tekstcontainer
ADB10 Oefening 2 — Typografische hiërarchie

Maak een pagina menu.html voor een fictief restaurant:

  • Restaurantnaam als h1 (groot, uppercase, letter-spacing)
  • Secties als h2 (Starters, Hoofdgerechten, Desserts)
  • Gerechten als h3, prijs rechts uitgelijnd
  • Beschrijving als p (kleiner, lichtgrijs)

Huistaak

📋 Huistaak — Typografie

Opdracht: Typografisch systeem voor een blogpagina

  1. Deel 1 — Lettertypes (3 punten): Kies een lettertypepaar van Google Fonts. Koppel ze correct aan je HTML-bestand met de preconnect-hints. Gebruik het ene lettertype voor headings en het andere voor de bodytekst.
  2. Deel 2 — Typografisch systeem (3 punten): Stel in je CSS een volledig typografisch systeem in: font-size in rem voor alle niveaus, line-height voor tekst en headings, max-width: 70ch op de tekstcontainer.
  3. Deel 3 — Blogpagina (4 punten): Maak een pagina met een hoofdartikel. Gebruik de juiste hiërarchie (h1, h2, h3, p). Pas letter-spacing toe op de paginatitel en zorg voor een professioneel visueel resultaat.
