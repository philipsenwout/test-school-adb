1.1 Wat is HTML?

HTML staat voor HyperText Markup Language. Het is een markeertaal, geen programmeertaal. Het verschil is belangrijk:

  • Een programmeertaal geeft instructies die de computer uitvoert (zoals berekeningen of beslissingen).
  • Een markeertaal geeft betekenis en structuur aan tekst. HTML zegt "dit is een titel", "dit is een alinea", "dit is een link".

HTML houdt zich niet bezig met hoe iets eruitziet. Dat is het domein van CSS (dat leer je in een later hoofdstuk). HTML gaat over wat iets is, niet over hoe het eruitziet.

Stel je een krantenartikel voor. Daarin is er een grote krantenkop (de titel), kleinere tussenkopjes, alinea's met tekst, en foto's met bijschriften. HTML geeft al die onderdelen een naam en structuur.

1.2 De basisstructuur van een HTML-document

Elke HTML-pagina ziet er als volgt uit:

HTML
<!DOCTYPE html>
<html lang="nl">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mijn eerste pagina</title>
</head>
<body>
    <h1>Hallo wereld!</h1>
    <p>Dit is mijn eerste webpagina.</p>
</body>
</html>

Laten we elk onderdeel stap voor stap bekijken:

<!DOCTYPE html>

Dit is de documenttype-declaratie. Het is geen HTML-element, maar een instructie aan de browser: "dit document is HTML5". Je schrijft dit altijd op de allereerste regel.

<html lang="nl">

Dit is het wortelelement (root element). Alle andere elementen staan erin. Het attribuut lang="nl" vertelt de browser (en screenreaders, zoekmachines) dat de pagina in het Nederlands is geschreven. Gebruik lang="en" voor Engelse pagina's.

<head>

De <head> bevat verborgen informatie over de pagina — informatie die de browser nodig heeft, maar die de bezoeker niet direct ziet op het scherm. Hierin vind je onder andere:

  • <meta charset="UTF-8"> — Zorgt dat speciale tekens (zoals é, ü, ñ, €) correct worden getoond. UTF-8 is de meest gebruikte tekencodering ter wereld.
  • <meta name="viewport" content="width=device-width, initial-scale=1.0"> — Zorgt dat de pagina correct wordt weergegeven op smartphones en tablets (responsive design).
  • <title>Mijn eerste pagina</title> — De titel die bovenaan het browsertabblad verschijnt. Dit is ook wat zoekmachines tonen als zoekresultaat.

<body>

De <body> bevat alle inhoud die de bezoeker effectief ziet op de pagina: tekst, afbeeldingen, links, enzovoort.

1.3 Elementen, tags en attributen

Elementen en tags

Een HTML-element bestaat uit een openingstag, inhoud en een sluitingstag:

HTML
<p>Dit is een alinea.</p>
  • <p> = openingstag
  • Dit is een alinea. = inhoud
  • </p> = sluitingstag

Sommige elementen zijn leeg (void elements): ze hebben geen inhoud en ook geen sluitingstag. Ze bevatten alle informatie in hun attributen:

HTML
<meta charset="UTF-8">
<img src="foto.jpg" alt="Een foto">

Attributen

Attributen geven extra informatie over een element. Ze staan altijd in de openingstag, na de elementnaam:

HTML
<html lang="nl">

Hier is lang het attribuut en "nl" de waarde. Attribuutwaarden staan altijd tussen aanhalingstekens.

1.4 Blok- vs. inline-elementen

Dit is een fundamenteel concept in HTML. Elementen gedragen zich op twee manieren:

Blok-elementen (block elements)

  • Nemen de volledige breedte van hun container in
  • Beginnen altijd op een nieuwe regel
  • Voorbeelden: <p>, <h1>, <h2>, <ul>, <div>
HTML
<p>Dit is de eerste alinea.</p>
<p>Dit is de tweede alinea. Ze staan onder elkaar.</p>

Inline-elementen

  • Nemen slechts de breedte in die ze nodig hebben
  • Staan naast andere tekst of elementen op dezelfde regel
  • Voorbeelden: <em>, <strong>, <a>, <img>, <span>
HTML
<p>Dit is een zin met een <em>beklemtoond woord</em> en een <strong>sterk woord</strong>.</p>

Een handig ezelsbruggetje: stel je voor dat blok-elementen dozen zijn (ze nemen de volledige breedte in) en inline-elementen letters zijn (ze passen naast elkaar op een regel).

1.5 Nesting — elementen in elkaar zetten

HTML-elementen mogen in andere elementen staan. Dit noem je nesting (nestelen). De volgorde van openen en sluiten moet echter kloppen:

Correct:

HTML
<p>Dit is een zin met een <em>beklemtoond</em> woord.</p>

Fout (overlapping):

HTML
<p>Dit is een <em>fout geneste</p></em>

Bij fout geneste elementen gedraagt de browser zich onvoorspelbaar. Controleer altijd of elke openingstag een bijpassende sluitingstag heeft, op de juiste plek.

Een goede gewoonte is inspringen (indentation): voor elke genest niveau gebruik je een extra tab of spaties. Dit maakt de code veel leesbaarder:

HTML
<body>
    <header>
        <h1>Welkom</h1>
    </header>
    <main>
        <p>Inhoud hier.</p>
    </main>
</body>

1.6 Commentaar

Commentaar in HTML is tekst die de browser negeert. Het is voor de programmeur zelf (of andere lezers van de code) en verschijnt niet op de pagina:

HTML
<!-- Dit is commentaar. De browser toont dit niet. -->
<p>Dit zie je wel op de pagina.</p>

<!--
    Commentaar kan ook
    meerdere regels bevatten.
-->

Gebruik commentaar om:

  • Secties van je code te markeren ("begin navigatie", "einde navigatie")
  • Tijdelijk code uit te schakelen zonder te verwijderen
  • Uitleg te geven bij ingewikkelde code

1.7 Emmet — sneller HTML schrijven

Emmet is een plugin die standaard ingebouwd is in VS Code (en andere code-editors). Met Emmet schrijf je HTML-afkortingen die automatisch worden uitgebreid tot volledige HTML-code.

De basisstructuur in één keer

Typ in VS Code in een leeg .html-bestand het uitroepteken ! en druk op Tab.

Veelgebruikte Emmet-afkortingen

AfkortingResultaat
!Volledige HTML-basisstructuur
h1<h1></h1>
p<p></p>
ul>li*3<ul> met 3 <li> items
nav>ul>li*5>aNavigatie met 5 links
loremVuloptekst (lorem ipsum, 30 woorden)
lorem10Vuloptekst van 10 woorden

Nesting en vermenigvuldiging

HTML
<!-- ul>li*3 + Tab -->
<ul>
    <li></li>
    <li></li>
    <li></li>
</ul>

Uitleg van de operatoren:

  • > — Kind-element (child)
  • + — Broer/zus-element (sibling), op hetzelfde niveau
  • *3 — Herhaal 3 keer
  • .klassenaam — Voeg class attribuut toe

1.8 Jouw eerste webpagina — volledig voorbeeld

HTML
<!DOCTYPE html>
<html lang="nl">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Over mij</title>
</head>
<body>
    <!-- Hoofding van de pagina -->
    <header>
        <h1>Over mij</h1>
    </header>

    <!-- Hoofdinhoud -->
    <main>
        <h2>Wie ben ik?</h2>
        <p>
            Mijn naam is Lena. Ik ben 16 jaar en zit in het 5e jaar Data- en Applicatiebeheer.
            Ik leer graag programmeren en ben geïnteresseerd in webdesign.
        </p>

        <h2>Mijn hobby's</h2>
        <p>
            In mijn vrije tijd speel ik gitaar en lees ik graag <em>sciencefiction</em>.
            Ik vind het ook <strong>heel leuk</strong> om nieuwe programmeertalen te leren.
        </p>
    </main>

    <!-- Voettekst -->
    <footer>
        <p>Gemaakt door Lena &copy; 2025</p>
    </footer>
</body>
</html>

Oefeningen

ADB09 / ADB10 Oefening 1 — Jouw eerste pagina

Maak een nieuw bestand aan in VS Code: index.html. Gebruik de Emmet-afkorting ! om de basisstructuur te genereren. Pas daarna de volgende zaken aan:

  1. Verander de lang naar "nl".
  2. Geef de pagina een zinvolle <title>.
  3. Voeg in de <body> toe:
    • Een <h1> met jouw naam
    • Een <p> met een korte beschrijving over jezelf
    • Een <p> met je drie favoriete vakken op school (gebruik hiervoor <em> voor jouw lievelingsvak)

Sla op en open in je browser. Controleer of alles correct wordt getoond.

ADB10 Oefening 2 — Emmet-training

Typ onderstaande Emmet-afkortingen in VS Code en controleer het resultaat door op Tab te drukken:

  1. h2
  2. ul>li*4
  3. section>h2+p*2
  4. nav>ul>li*3>a
  5. p>lorem15

Schrijf bij elke afkorting op wat het resultaat was (in commentaar in je HTML-bestand).

ADB09 Oefening 3 — Blok vs. inline herkennen

Geef bij elk van de volgende elementen aan of het een blok-element of een inline-element is:

  • <p>
  • <em>
  • <h1>
  • <strong>
  • <ul>
  • <a>
  • <div>
  • <span>

Huistaak

📋 Huistaak — H01 HTML Basisstructuur

Opdracht: Mijn eerste webpagina

Je bouwt je allereerste eigen webpagina van nul af aan. De pagina gaat over jezelf of over iets wat jij interessant vindt (jouw hobby, favoriete game, sport, muziek, …).

Deel 1 — Basisstructuur (4 punten)

Maak een nieuw bestand aan met de naam index.html en zorg dat het volgende aanwezig is:

  1. De correcte <!DOCTYPE html>-declaratie
  2. Het <html>-element met het attribuut lang="nl"
  3. Een volledig <head>-blok met:
    • <meta charset="UTF-8">
    • <meta name="viewport" content="width=device-width, initial-scale=1.0">
    • Een zinvolle <title> (niet gewoon "Untitled")
  4. Een <body>-element dat de zichtbare inhoud zal bevatten

Deel 2 — Emmet gebruiken (3 punten)

Gebruik Emmet (in VS Code) om de structuur sneller te typen:

  1. Genereer de volledige HTML5-boilerplate via de Emmet-afkorting ! + Tab. Maak een screenshot van het resultaat en voeg dit toe aan je indiening (of beschrijf welke afkorting je hebt gebruikt).
  2. Gebruik de Emmet-afkorting h1{Welkom op mijn pagina} om snel een h1-element aan te maken en voeg dit toe aan je <body>.
  3. Gebruik de Emmet-afkorting p{Dit is mijn eerste alinea.} om een alinea toe te voegen.

Deel 3 — Inhoud toevoegen (3 punten)

Voeg de volgende inhoud toe aan de <body> van je pagina:

  1. Een <h1> met je naam of de naam van je onderwerp
  2. Minstens twee alinea's (<p>) met zinvolle tekst over jezelf of je onderwerp
  3. Een HTML-commentaar (<!-- -->) die uitlegt waar een toekomstige sectie zal komen, bv. <!-- Hier komt later een navigatiebalk -->
