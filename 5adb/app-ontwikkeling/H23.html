Logische Operatoren

ADB09De leerlingen ontwikkelen oplossingen voor concrete problemen.
🎯 Leerdoelen

  • Waarheidstabellen lezen en opstellen voor && en ||

  • Logische operatoren combineren met vergelijkingsoperatoren in if-statements

In H22 leerde je if-statements met één voorwaarde. Maar in de praktijk wil je vaak meerdere voorwaarden tegelijk controleren. Daarvoor gebruik je de drie logische operatoren:

OperatorNaamBetekenis
&&ENBeide zijden moeten true zijn
||OFMinstens één zijde moet true zijn
!NIETKeert de waarde om

23.1 De && operator — EN

&& geeft true als beide zijden true zijn.

Waarheidstabel voor &&

LinksRechtsResultaat
truetruetrue
truefalsefalse
falsetruefalse
falsefalsefalse
JavaScript
let leeftijd = 20;
let heeftTicket = true;

if (leeftijd >= 18 && heeftTicket) {
    console.log("Welkom bij het evenement!");
} else {
    console.log("Je mag er helaas niet in.");
}
// → "Welkom bij het evenement!" (beide voorwaarden zijn true)

Meerdere && aaneenkoppelen

JavaScript
let leeftijd = 22;
let heeftTicket = true;
let staatOpLijst = true;

if (leeftijd >= 18 && heeftTicket && staatOpLijst) {
    console.log("Toegang verleend.");
}
// Alle drie de voorwaarden moeten true zijn. Eén false is genoeg!

23.2 De || operator — OF

|| geeft true als minstens één van de zijden true is.

Waarheidstabel voor ||

LinksRechtsResultaat
truetruetrue
truefalsetrue
falsetruetrue
falsefalsefalse
JavaScript
let isStudent = false;
let heeftLidkaart = true;

if (isStudent || heeftLidkaart) {
    console.log("Je krijgt 20% korting.");
} else {
    console.log("Geen korting van toepassing.");
}
// → "Je krijgt 20% korting." (heeftLidkaart is true)

23.3 De ! operator — NIET

! keert een booleaanse waarde om: true wordt false, false wordt true.

JavaScript
let isIngelogd = false;

if (!isIngelogd) {
    console.log("Gelieve in te loggen.");
}
// isIngelogd is false → !isIngelogd is true → boodschap wordt getoond

Dubbele negatie !!

JavaScript
let naam = "Marie";
console.log(!!naam);  // → true  (naam bestaat en is niet leeg)

let leeg = "";
console.log(!!leeg);  // → false (lege string is falsy)

23.4 Kortsluiting (short-circuit evaluation)

JavaScript evalueert logische uitdrukkingen zo efficiënt mogelijk: zodra het resultaat vaststaat, stopt de evaluatie.

Kortsluiting bij &&

&& stopt bij de eerste false.

JavaScript
let gebruiker = null;

// Met kortsluiting is het veilig — tweede deel wordt nooit geëvalueerd:
if (gebruiker !== null && gebruiker.naam === "Marie") {
    console.log("Hallo Marie!");
}
// gebruiker !== null is false → JavaScript evalueert gebruiker.naam nooit
// Zonder kortsluiting zou dit een TypeError geven!

Kortsluiting bij ||

|| stopt bij de eerste true.

JavaScript
let naam = "";
let weergaveNaam = naam || "Anoniem";
console.log(weergaveNaam);  // → "Anoniem"
// naam is lege string (falsy) → || kijkt naar rechterkant → "Anoniem"

Truthy en falsy waarden

In JavaScript gedragen sommige waarden zich als false in een booleaanse context:

Falsy waarden
false
0
"" (lege string)
null
undefined
NaN

Alles wat niet falsy is, is truthy: "hallo", 1, [], {} zijn allemaal truthy.

23.5 Combinatie met vergelijkingsoperatoren

Vergelijkingen (>, <, ===, ...) worden voor logische operatoren berekend. Gebruik haakjes om de volgorde duidelijk te maken bij complexe combinaties.

JavaScript
let leeftijd = 19;
let isLid = false;
let heeftVipPas = true;

// Haakjes bepalen dat isLid || heeftVipPas eerst berekend wordt:
if (leeftijd >= 18 && (isLid || heeftVipPas)) {
    console.log("Toegang verleend.");
} else {
    console.log("Toegang geweigerd.");
}
// → "Toegang verleend." (18+ EN (geen lid maar wél vip))
⚠️ Haakjes zijn belangrijk

Zonder haakjes zou JavaScript dit lezen als (leeftijd >= 18 && isLid) || heeftVipPas — een heel andere logica!

23.6 Nullish coalescing ?? — voor gevorderde leerlingen

De ?? operator lijkt op ||, maar is specifieker: hij geeft de rechterkant terug alleen als de linkerkant null of undefined is. Andere falsy waarden (zoals 0 of "") worden niet als "leeg" beschouwd.

JavaScript
let aantalPogingen = 0;

// Met || : 0 is falsy, dus resultaat is 5 (waarschijnlijk niet bedoeld!)
let pogingen1 = aantalPogingen || 5;
console.log(pogingen1);  // → 5

// Met ?? : 0 is niet null/undefined, dus 0 blijft
let pogingen2 = aantalPogingen ?? 5;
console.log(pogingen2);  // → 0
