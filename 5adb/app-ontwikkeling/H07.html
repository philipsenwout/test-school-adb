Formulieren

7.1 Wat is een formulier?

Een formulier (<form>) is de manier waarop een HTML-pagina gegevens van de gebruiker ophaalt en verstuurt. Denk aan:

  • Een inlogpagina (gebruikersnaam + wachtwoord)
  • Een contactformulier (naam, e-mail, bericht)
  • Een bestelformulier (producten, aantal, adres)
  • Een zoekbalk

Formulieren zijn het belangrijkste interactiemiddel op het web. Een goed ontworpen formulier maakt het verschil tussen een fijne en een frustrerende gebruikerservaring.

7.2 Het <form> element

De <form> tag omsluit alle velden van het formulier. De twee belangrijkste attributen zijn:

AttribuutBetekenis
actionNaar welk adres worden de gegevens gestuurd?
methodHoe worden de gegevens gestuurd? (GET of POST)
HTML
<form action="verwerking.html" method="GET">
  <!-- velden komen hier -->
</form>

Verschil GET en POST:

  • GET — de gegevens worden zichtbaar in de URL (goed voor zoekopdrachten)
  • POST — de gegevens worden verborgen meegestuurd (goed voor wachtwoorden, persoonlijke info)

7.3 Het <label> element en de for/id koppeling

Elke invoer heeft een label nodig. Het <label> element koppel je aan een inputveld via twee bijpassende attributen:

  • Het for-attribuut op het <label> — bevat de naam
  • Het id-attribuut op het <input> — dezelfde naam
HTML
<label for="voornaam">Voornaam</label>
<input type="text" id="voornaam" name="voornaam">

Waarom is dit belangrijk?

  1. Als je op het label klikt, springt de cursor naar het veld — gebruiksvriendelijk
  2. Screenreaders koppelen het label aan het veld — toegankelijk voor blinden
  3. Op smartphones is het label een groter klikgebied
💡 Tip

Nooit een inputveld zonder label! Placeholdertekst (placeholder="...") is geen vervanging voor een label — die verdwijnt zodra de gebruiker begint te typen.

7.4 Invoervelden met <input>

Het <input> element is het meest veelzijdige element in formulieren. Het type-attribuut bepaalt het soort invoer.

type="text" — tekstveld

HTML
<div>
  <label for="naam">Naam</label>
  <input type="text" id="naam" name="naam" required>
</div>

type="email" — e-mailadres

HTML
<div>
  <label for="email">E-mailadres</label>
  <input type="email" id="email" name="email" required>
</div>

type="number" — getal

HTML
<div>
  <label for="leeftijd">Leeftijd</label>
  <input type="number" id="leeftijd" name="leeftijd" min="1" max="120" step="1">
</div>

type="password" — wachtwoord

HTML
<div>
  <label for="wachtwoord">Wachtwoord</label>
  <input type="password" id="wachtwoord" name="wachtwoord" required>
</div>

type="checkbox" — aanvinkvakje

HTML
<fieldset>
  <legend>Interesses</legend>
  <div>
    <input type="checkbox" id="netwerken" name="interesse" value="netwerken">
    <label for="netwerken">Netwerken</label>
  </div>
  <div>
    <input type="checkbox" id="programmeren" name="interesse" value="programmeren">
    <label for="programmeren">Programmeren</label>
  </div>
</fieldset>

type="radio" — keuzerondje

HTML
<fieldset>
  <legend>Klas</legend>
  <div>
    <input type="radio" id="klas5" name="klas" value="5ADB">
    <label for="klas5">5ADB</label>
  </div>
  <div>
    <input type="radio" id="klas6" name="klas" value="6ADB">
    <label for="klas6">6ADB</label>
  </div>
</fieldset>

7.5 Keuzemenu met <select> en <option>

HTML
<div>
  <label for="school">School</label>
  <select id="school" name="school">
    <option value="">-- Kies je school --</option>
    <option value="kta">KTA</option>
    <option value="atheneum">Atheneum</option>
    <option value="college">College</option>
  </select>
</div>
💡 Tip

De eerste lege <option> is een goede gewoonte: ze dwingt de gebruiker bewust een keuze te maken in plaats van dat de eerste optie automatisch geselecteerd staat.

7.6 Groot tekstveld met <textarea>

HTML
<div>
  <label for="bericht">Bericht</label>
  <textarea id="bericht" name="bericht" rows="5" cols="40"></textarea>
</div>
💡 Tip

Laat <textarea> nooit zelfsluitend schrijven: <textarea /> werkt NIET. Schrijf altijd de afsluitende tag: </textarea>.

7.7 Knoppen met <button>

HTML
<button type="submit">Verstuur formulier</button>
WaardeGedrag
submitVerzendt het formulier
buttonDoet niets van zichzelf (voor JavaScript-knoppen)
resetWis alle velden — gebruik dit NOOIT

Waarom geen resetknop? Stel: een gebruiker heeft vijf minuten lang een formulier ingevuld. Per ongeluk klikt hij op "Reset" in plaats van "Verstuur". Alles is weg. Er is geen enkel scenario waarbij een resetknop nuttig is voor de gebruiker.

7.8 Een volledig formulier — stap voor stap

HTML
<form action="bedankt.html" method="GET">

  <div>
    <label for="naam">Naam *</label>
    <input type="text" id="naam" name="naam" required>
  </div>

  <div>
    <label for="email">E-mailadres *</label>
    <input type="email" id="email" name="email" required>
  </div>

  <div>
    <label for="onderwerp">Onderwerp</label>
    <select id="onderwerp" name="onderwerp">
      <option value="">-- Kies een onderwerp --</option>
      <option value="info">Algemene informatie</option>
      <option value="technisch">Technische vraag</option>
    </select>
  </div>

  <div>
    <label for="bericht">Bericht *</label>
    <textarea id="bericht" name="bericht" rows="6"></textarea>
  </div>

  <div>
    <input type="checkbox" id="akkoord" name="akkoord" value="ja" required>
    <label for="akkoord">Ik ga akkoord met de privacyverklaring *</label>
  </div>

  <p>* Verplicht veld</p>
  <button type="submit">Verstuur bericht</button>

</form>

7.9 Het <fieldset> en <legend> element

Wanneer een formulier uit meerdere secties bestaat, gebruik je <fieldset> om velden te groeperen en <legend> als titel van die groep:

HTML
<form action="registreer.html" method="POST">
  <fieldset>
    <legend>Persoonlijke gegevens</legend>
    <div>
      <label for="voornaam">Voornaam</label>
      <input type="text" id="voornaam" name="voornaam" required>
    </div>
    <div>
      <label for="achternaam">Achternaam</label>
      <input type="text" id="achternaam" name="achternaam" required>
    </div>
  </fieldset>
  <button type="submit">Registreer</button>
</form>

7.10 Form usability — 10 richtlijnen

Dit zijn de 10 richtlijnen van Kathryn Whitenton voor bruikbare formulieren:

  1. Hou het kort. Een formulier met 3 velden is beter dan één met 10.
  2. Groepeer bij elkaar horende labels en inputvelden. Gebruik <fieldset> en <legend>.
  3. Zet je velden in één kolom. Formulieren verspreid over meerdere kolommen zijn verwarrender.
  4. Zorg voor een logische volgorde. Eerst naam, dan e-mail, dan bericht.
  5. Gebruik geen placeholdertekst als label. Placeholder verdwijnt bij het typen.
  6. Gebruik de lengte van een inputveld om te tonen hoeveel input je verwacht.
  7. Maak duidelijk welke velden verplicht zijn. Gebruik een asterisk *.
  8. Communiceer voorwaarden op voorhand. Schrijf boven het wachtwoordveld: "Minimaal 8 tekens".
  9. Vermijd een resetknop. Dit is nooit nuttig voor de gebruiker.
  10. Voorzie duidelijke foutmeldingen. Zeg precies wat er fout is en hoe het te herstellen.

Oefeningen

ADB09 / ADB10 Oefening 1 — Registratieformulier

Maak een registratieformulier voor een fictieve school-app. Het formulier bevat:

  • Voornaam (tekst, verplicht)
  • Achternaam (tekst, verplicht)
  • E-mailadres (verplicht)
  • Wachtwoord (verplicht)
  • Klas (radio buttons: 5ADB of 6ADB)
  • Favoriete vak (checkboxes: Netwerken, App-ontwikkeling, Databeheer, E-STEM)
  • Verzendknop (geen resetknop!)

Pas de 10 usability-richtlijnen toe: één kolom, duidelijke labels, verplichte velden markeren.

ADB09 / ADB10 Oefening 2 — Zoekformulier

Maak een eenvoudig zoekformulier voor een bibliotheek-website.

  • Zoekveld (tekst)
  • Categorie (dropdown: Boeken, Tijdschriften, Films, Muziek)
  • Sortering (radio buttons: Meest relevant, Nieuwste eerst, Alfabetisch)
  • Zoekknop

Huistaak

📋 Huistaak — H07 Formulieren

Opdracht: Inschrijvingsformulier voor een schoolevenement

Je maakt een HTML-formulier voor de inschrijving van leerlingen voor een fictief schoolevenement (bv. schoolreisje, sportdag, debatclub, filmavond, …). Je kiest zelf het evenement.

Maak een nieuw bestand: inschrijving.html

Deel 1 — Basisstructuur formulier (2 punten)

  1. Maak een <form>-element aan met:
    • action="#" (we sturen het formulier nog nergens naartoe)
    • method="post"
  2. Voeg een <h1> toe boven het formulier met de naam van het evenement.
  3. Voeg een <fieldset> toe met een <legend> "Persoonlijke gegevens".

Deel 2 — Tekstvelden en labels (4 punten)

Voeg binnen het fieldset de volgende velden toe, elk voorzien van een correct <label>:

  1. Voornaam (type="text", required)
  2. Achternaam (type="text", required)
  3. E-mailadres (type="email", required)
  4. Geboortedatum (type="date")
  5. Telefoonnummer (type="tel", placeholder="bv. 0412 34 56 78")

Elk label moet via for gekoppeld zijn aan het bijhorende input-veld via id.

Deel 3 — Keuze-elementen (3 punten)

  1. Voeg een tweede <fieldset> toe met <legend> "Voorkeuren".
  2. Voeg radiobuttons toe voor de keuze van een activiteit (minstens 3 opties):
    • bv. Wandeltocht | Fietsen | Sportspelen
  3. Voeg checkboxes toe voor extra opties (minstens 2):
    • bv. Vegetarische maaltijd | Allergieën melden
  4. Voeg een <select>-dropdown toe voor de klas (bv. 5ADB, 5B, 5C, 6ADB).

Deel 4 — Verzendknop en textarea (1 punt)

  1. Voeg een <textarea> toe met het label "Opmerkingen" (niet verplicht).
  2. Voeg een <button type="submit"> toe met de tekst "Inschrijven".
