H03 — GitHub Version Control 5ADB

Je repository clonen

ADB16De leerlingen gebruiken een versiebeheersysteem.
ADB13De leerlingen stellen een applicatie in werking.
🎯 Leerdoelen
  • Je kan de URL van een GitHub repository opzoeken via de groene Code-knop
  • Je kan een repository clonen met git clone <url>
  • Je kan de mappenstructuur van je repo bekijken in de terminal
  • Je kan git status lezen en begrijpen wat de uitvoer betekent

3.1 Wat is clonen?

Clonen betekent: een volledige kopie maken van een GitHub-repository naar je computer. Je downloadt niet alleen de bestanden, maar ook de volledige versiegeschiedenis.

Je doet dit één keer, aan het begin van het jaar. De leerkracht heeft al een persoonlijke repository voor jou aangemaakt. Jij kloont die naar je computer en werkt er van daaruit mee.

3.2 De repository-URL vinden

Je leerkracht stuurt je een link naar je persoonlijke repository. Die ziet er zo uit:

Bash
https://github.com/appschool-munsterbilzen/jan-peeters-5adb

Op die GitHub-pagina vind je de clone-URL via de groene Code-knop:

  1. Open de repository op GitHub
  2. Klik op de groene <> Code knop
  3. Zorg dat het tabblad HTTPS geselecteerd is
  4. Kopieer de URL (klik op het kopieer-icoontje)
🔒 HTTPS vs SSH

Er zijn twee manieren om te clonen: HTTPS en SSH. In 5ADB gebruiken we altijd HTTPS — dit is het eenvoudigst. Je wordt dan gevraagd om in te loggen met je GitHub gebruikersnaam en wachtwoord (of een Personal Access Token).

3.3 git clone uitvoeren

Open je terminal en navigeer naar de map waar je de repository wil plaatsen, bv. Documents:

Bash
cd Documents
git clone https://github.com/appschool-munsterbilzen/jan-peeters-5adb

Git toont tijdens het clonen iets als:

Bash
Cloning into 'jan-peeters-5adb'...
remote: Enumerating objects: 12, done.
remote: Counting objects: 100% (12/12), done.
Receiving objects: 100% (12/12), done.

Er staat nu een nieuwe map jan-peeters-5adb in je Documents. Navigeer daarin:

Bash
cd jan-peeters-5adb
⚠ Windows: Personal Access Token

GitHub aanvaardt geen wachtwoorden meer via HTTPS. Je hebt een Personal Access Token (PAT) nodig. Ga op GitHub naar Settings → Developer settings → Personal access tokens → Generate new token. Geef het scope repo en kopieer het token. Gebruik dit token als wachtwoord wanneer Git erom vraagt.

3.4 Mappenstructuur bekijken

Bekijk de inhoud van je repository:

Bash
ls

Je ziet de basisstructuur die de leerkracht al heeft aangemaakt:

Bash
app-ontwikkeling/
README.md

Navigeer dieper in de structuur:

Bash
cd app-ontwikkeling
ls

3.5 git status — de toestand bekijken

git status is een van de commando's die je het meest gaat gebruiken. Het toont de huidige toestand van je repository: welke bestanden zijn gewijzigd, welke zijn klaar om te committen.

Bash
git status

Vlak na het clonen, zonder wijzigingen, zie je:

Bash
On branch main
Your branch is up to date with 'origin/main'.

nothing to commit, working tree clean

Dit betekent: er zijn geen onopgeslagen wijzigingen, alles is gesynchroniseerd met GitHub. Later, als je bestanden aanpast, zal git status je vertellen welke bestanden gewijzigd zijn.

📌 Onthoud

Twijfel je over de toestand van je repository? Typ altijd eerst git status. Het is altijd veilig om dit commando te gebruiken — het verandert niets, het toont alleen.

Oefeningen

ADB16 Oefening 1 — Clone je persoonlijke repository

De leerkracht heeft je de URL van je persoonlijke repository gegeven. Voer de volgende stappen uit:

  1. Open Git Bash
  2. Navigeer naar je Documents-map: cd ~/Documents
  3. Clone je repository: git clone [url-van-leerkracht]
  4. Navigeer in de geclonede map: cd [naam-van-repo]
  5. Bekijk de inhoud: ls
  6. Controleer de status: git status
  7. Maak een screenshot van elke stap
ADB16 Oefening 2 — Repository verkennen

Verken de mappenstructuur van je repository via de terminal:

  1. Toon alle mappen en bestanden met ls
  2. Navigeer naar app-ontwikkeling/ en bekijk de inhoud
  3. Ga terug naar de root van de repository met cd .. of cd ~
  4. Open de repository ook in VS Code: typ code . in de terminal

Huistaak

📋 Huistaak — Clone en verken

Opdracht: Maak een README.md aan

  1. Deel 1 — 4 punten: Open README.md in je repository met VS Code. Voeg toe:
    • Jouw naam
    • Jouw klas (5ADB)
    • Schooljaar (2025-2026)
    • Een korte beschrijving van wat er in de repo staat
  2. Deel 2 — 3 punten: Sla het bestand op. Voer git status uit en maak een screenshot. Wat zie je nu in de uitvoer?
  3. Deel 3 — 3 punten: Leg in eigen woorden uit wat het verschil is tussen de map op je computer (lokale repo) en de map op GitHub (remote repo).

We leren in H04 hoe we het bestand vastleggen en uploaden. Bewaar je screenshot alvast.
