H04 — GitHub Version Control 5ADB

Add & Commit

ADB16De leerlingen gebruiken een versiebeheersysteem.
ADB07De leerlingen houden de administratie bij in functie van de opdracht.
🎯 Leerdoelen
  • Je kan het concept van de staging area uitleggen
  • Je kan bestanden toevoegen aan de staging area met git add
  • Je kan een commit aanmaken met git commit -m "boodschap"
  • Je kan een goede, beschrijvende commit-boodschap schrijven
  • Je kan de versiegeschiedenis bekijken met git log

4.1 De drie zones van Git

Git werkt met drie zones. Begrijp deze zones goed — ze zijn de kern van hoe Git werkt.

1. Working Directory (werkmap)

Alle bestanden op je computer. Hier werk je normaal mee. Wijzigingen die je hier maakt zijn nog niet bijgehouden door Git.

2. Staging Area (klaarzetzone)

Een soort "wachtkamer" voor bestanden die je wil vastleggen. Je beslist zelf welke bestanden er in komen met git add.

3. Repository (.git map)

De permanente versiegeschiedenis. Eenmaal je git commit uitvoert, is de snapshot voor altijd opgeslagen.

4.2 git add — bestanden klaarzetten

Nadat je bestanden hebt gewijzigd of aangemaakt, zet je ze klaar voor een commit:

Bash
# Eén specifiek bestand toevoegen
git add index.html

# Meerdere bestanden tegelijk
git add index.html style.css

# Alle gewijzigde bestanden in de huidige map
git add .

Controleer altijd eerst met git status welke bestanden gewijzigd zijn:

Bash
git status

Na een wijziging zie je de bestanden in rood (niet klaargezet):

Bash
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)

        modified:   app-ontwikkeling/H01/index.html

Na git add zie je ze in groen (klaargezet):

Bash
Changes to be committed:
  (use "git restore --staged <file>..." to unstage)

        modified:   app-ontwikkeling/H01/index.html

4.3 git commit — de snapshot vastleggen

Als alle gewenste bestanden klaargezet zijn, maak je de commit aan:

Bash
git commit -m "H01 oefening 1 afgewerkt"

De -m staat voor "message" (boodschap). Wat je tussen de aanhalingstekens schrijft, is de beschrijving van de commit. Git bevestigt de commit:

Bash
[main 3f4a8b2] H01 oefening 1 afgewerkt
 1 file changed, 42 insertions(+), 3 deletions(-)

4.4 Goede commit-boodschappen schrijven

Een goede commit-boodschap vertelt wat je gedaan hebt en waarom. De leerkracht leest deze boodschappen als hij je werk nakijkt.

❌ Slecht
  • "update"
  • "changes"
  • "fix"
  • "asdfgh"
  • "werk van vandaag"
✓ Goed
  • "H01 oefening 1 nav-structuur toegevoegd"
  • "H03 CSS grid layout afgewerkt"
  • "bug opgelost: menu toont niet op mobiel"
  • "README bijgewerkt met installatiestappen"
📌 Naamconventie voor schoolwerk

Gebruik dit formaat voor je commit-boodschappen: [H01/H02/...] [beschrijving]
Voorbeelden: "H04 oef1 flexbox nav", "H07 huistaak formulier afgewerkt", "toets1 verbeterd nav feedback"

4.5 git log — versiegeschiedenis bekijken

git log toont alle commits die je gemaakt hebt, van nieuwste naar oudste:

Bash
git log
Bash
commit 3f4a8b2c1d9e8f7a6b5c4d3e2f1a0b9c8d7e6f5
Author: Jan Peeters <jan@schoolmail.be>
Date:   Mon Sep 9 14:23:11 2025 +0200

    H01 oefening 1 nav-structuur toegevoegd

commit 1a2b3c4d5e6f7a8b9c0d1e2f3a4b5c6d7e8f9a0
Author: Jan Peeters <jan@schoolmail.be>
Date:   Mon Sep 9 09:15:03 2025 +0200

    README bijgewerkt met naam en klas

Druk Q om het log af te sluiten.

Voor een compacte weergave gebruik je:

Bash
git log --oneline

Oefeningen

ADB16 Oefening 1 — Je eerste commit

Maak je eerste eigen commit in je repository:

  1. Open je repository in VS Code: code .
  2. Maak een nieuw bestand app-ontwikkeling/H01/index.html aan
  3. Voeg een eenvoudige HTML-structuur toe (mag van vorige lessen zijn)
  4. Sla op
  5. Voer uit in Git Bash:
    Bash
    git status
git add app-ontwikkeling/H01/index.html
git status
git commit -m "H01 eerste index.html aangemaakt"
git log --oneline
  6. Maak screenshots van elke stap
ADB16 Oefening 2 — Meerdere commits

Maak twee extra commits om de workflow in te oefenen:

  1. Voeg een <h1> toe aan je index.html
  2. Commit: git commit -m "H01 titel toegevoegd"
  3. Voeg een <p> toe
  4. Commit: git commit -m "H01 inleiding toegevoegd"
  5. Bekijk je geschiedenis: git log --oneline

Je ziet nu drie commits in je log. Zo werkt versiebeheer!

Huistaak

📋 Huistaak — Commit-log

Opdracht: Maak 5 commits aan

  1. Deel 1 — 6 punten: Bouw een eenvoudige HTML-pagina (app-ontwikkeling/H01-oefening/index.html) stap voor stap op en maak na elke stap een commit met een duidelijke boodschap:
    • Stap 1: basisstructuur (<!DOCTYPE>, <html>, <head>, <body>)
    • Stap 2: titels toevoegen
    • Stap 3: een paragraaf toevoegen
    • Stap 4: een link toevoegen
    • Stap 5: een afbeelding toevoegen
  2. Deel 2 — 4 punten: Maak een screenshot van git log --oneline die alle 5 commits toont. Zijn de commit-boodschappen duidelijk en beschrijvend?
