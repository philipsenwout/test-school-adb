H01 — GitHub Version Control 5ADB

Wat is versiebeheer?

ADB16De leerlingen gebruiken een versiebeheersysteem.
ADB12De leerlingen documenteren een applicatie.
🎯 Leerdoelen
  • Je kan uitleggen waarom versiebeheer nuttig is aan de hand van een concreet voorbeeld
  • Je kan het verschil uitleggen tussen Git (lokale software) en GitHub (online platform)
  • Je kan de begrippen repository, commit en versiegeschiedenis definiëren
  • Je begrijpt hoe je jouw schoolwerk dit jaar organiseert via Git

1.1 Waarom versiebeheer?

Stel je voor: je bent een uur bezig met je HTML-pagina. Je beslist om het navigatiemenu helemaal te herwerken. Na een uur merk je dat je nieuwe versie er slechter uitziet dan de vorige. Wat doe je?

Als je geen versiebeheer gebruikt, is de originele versie weg. Tenzij je een kopie hebt gemaakt — maar dan heb je misschien index-v1.html, index-v2.html, index-nieuw.html, index-definitief.html, index-DEFINIITEF2.html staan. Herkenbaar?

⚠ Het klassieke probleem zonder versiebeheer

Bestanden met namen als opdracht-v3-definitief-ECHT-finaal.html zijn een teken dat je dringend versiebeheer nodig hebt.

Met versiebeheer bewaar je de volledige geschiedenis van je project. Elk moment waarop je een versie vastlegt heet een commit. Je kan altijd terugkeren naar een vroegere versie — zonder kopiebestanden.

Versiebeheer biedt nog meer voordelen:

  • Historiek: je ziet precies wat je wanneer hebt gewijzigd en waarom
  • Backup: je code staat online op GitHub — ook als je laptop crasht
  • Samenwerking: meerdere mensen kunnen aan hetzelfde project werken (6e jaar)
  • Feedback: je leerkracht kan commentaar geven op specifieke regels code

1.2 Git vs GitHub — niet hetzelfde!

Veel mensen verwarren Git en GitHub. Ze zijn nauw verbonden, maar niet hetzelfde:

● Git

  • Software die op je eigen computer staat
  • Houdt de versiegeschiedenis bij
  • Werkt ook zonder internet
  • Gratis en open-source

● GitHub

  • Online platform in de cloud
  • Sla je repository online op
  • Deel je code met anderen
  • Gratis basisaccount

Vergelijking: Git is als Microsoft Word (het programma op je computer), GitHub is als OneDrive (de online opslagplaats). Je schrijft lokaal met Word, en synchroniseert het naar OneDrive.

1.3 Begrippen die je moet kennen

In de volgende hoofdstukken kom je deze termen constant tegen. Leer ze nu goed:

Repository (repo)
Een map die je project bevat én de volledige versiegeschiedenis. Elke leerling heeft één persoonlijke repository voor dit schooljaar.
Commit
Een snapshot van je project op een bepaald moment. Je geeft elke commit een beschrijvende naam, bv. "H01 oefening 1 afgewerkt".
Clone
Een kopie van een online repository naar je lokale computer downloaden. Je doet dit één keer aan het begin van het jaar.
Push
Je lokale commits uploaden naar GitHub. Zo staat je werk online en kan de leerkracht het bekijken.
Pull
Wijzigingen van GitHub downloaden naar je lokale computer. Doe dit altijd als eerste wanneer je begint te werken.

1.4 Jouw jaarworkflow

Dit schooljaar gebruik je Git voor App-ontwikkeling. Elke leerling heeft een persoonlijke repository met een vaste mappenstructuur. Je werk lever je in door te pushen naar GitHub.

📁 Mappenstructuur van jouw repository
Bash
jouw-naam-5adb/
├── app-ontwikkeling/
│   ├── H01-html-basis/
│   │   └── index.html
│   ├── H02-html-tekst/
│   │   └── index.html
│   ├── huistaken/
│   │   ├── H01-huistaak/
│   │   └── H02-huistaak/
│   └── toetsen/
│       └── toets-1/
└── README.md

De leerkracht kloont of bekijkt jouw repository op GitHub. Hij/zij kan commentaar geven op specifieke regels code. Jij verwerkt die feedback, maakt een nieuwe commit en pusht opnieuw.

📌 Wat leer je dit jaar?

In 5ADB focussen we op de basisworkflow: clone → add → commit → push → pull. Branching en pull requests leer je in 6e jaar.

ADB16 Oefening 2 — Git vs GitHub matrix

Geef voor elke uitspraak aan of die over Git, GitHub, of beide gaat:

  1. "Ik sla mijn commits op in de cloud."
  2. "Ik voer git commit uit in de terminal."
  3. "De leerkracht kan mijn code online bekijken."
  4. "Het werkt ook zonder internetverbinding."
  5. "Ik maak er een gratis account aan."
  6. "Het houdt de versiegeschiedenis bij op mijn computer."
