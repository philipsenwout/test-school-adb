H05 — GitHub Version Control 5ADB

Push & Pull

ADB16De leerlingen gebruiken een versiebeheersysteem.
ADB14De leerlingen sturen een applicatie bij op basis van feedback.
🎯 Leerdoelen
  • Je kan lokale commits uploaden naar GitHub met git push
  • Je kan wijzigingen van GitHub downloaden met git pull
  • Je weet waarom je altijd eerst git pull doet voor je begint te werken
  • Je kan feedback van de leerkracht ophalen via git pull en verwerken

5.1 Lokaal vs remote

Je werkt altijd op twee plaatsen tegelijk:

  • Lokale repository: de map op jouw computer
  • Remote repository: de repo op GitHub (ook wel origin genoemd)

Na een commit bestaat de snapshot enkel lokaal. Met git push stuur je die naar GitHub. Met git pull haal je wijzigingen van GitHub naar je computer.

5.2 git push — uploaden naar GitHub

Na het committen stuur je je commits naar GitHub:

Bash
git push

Git geeft feedback:

Bash
Enumerating objects: 5, done.
Counting objects: 100% (5/5), done.
Writing objects: 100% (3/3), 892 bytes | 892.00 KiB/s, done.
To https://github.com/appschool-munsterbilzen/jan-peeters-5adb
   1a2b3c4..3f4a8b2  main -> main

Open je repository op GitHub in de browser. Je ziet de bestanden en commits verschijnen. De leerkracht kan jouw werk nu bekijken.

🔒 Authenticatie vereist

Bij de eerste git push vraagt GitHub om je gebruikersnaam en Personal Access Token (niet je GitHub-wachtwoord). Gebruik het token dat je aangemaakt hebt in H03.

5.3 git pull — ophalen van GitHub

git pull haalt de nieuwste commits van GitHub naar je lokale computer:

Bash
git pull
⚠ Doe dit altijd als eerste

Begin elke werksessie met git pull. Zo heb je altijd de meest recente versie — inclusief eventuele feedback van de leerkracht die online is gezet.

Als er niets te halen is, zie je:

Bash
Already up to date.

Als er wel nieuwe commits zijn, ziet de uitvoer er zo uit:

Bash
Updating 1a2b3c4..7d8e9f0
Fast-forward
 app-ontwikkeling/H01/index.html | 5 +++--
 1 file changed, 3 insertions(+), 2 deletions(-)

5.4 Veelgemaakte fouten

❌ Fout: "failed to push some refs"

Wat betekent dit? GitHub heeft commits die jij lokaal niet hebt. Je moet eerst pullen voor je kan pushen.

Oplossing:

Bash
git pull
git push
⚠ Je kan niet pushen zonder commits

Als je git push uitvoert zonder nieuwe commits, zie je "Everything up-to-date". Dat is normaal — eerst git add en git commit, dan pas git push.

5.5 Feedback van de leerkracht ophalen en verwerken

Je leerkracht bekijkt je gecommit werk op GitHub. Hij/zij kan:

  • Commentaar geven op specifieke regels in de GitHub-interface
  • Wijzigingen doorvoeren en een commit aanmaken op GitHub

Om die feedback naar je lokale computer te halen:

Bash
git pull

Je code is nu bijgewerkt met de commentaren. Open het bestand in VS Code en bekijk de wijzigingen. Verwerk de feedback, maak een nieuwe commit en push opnieuw:

Bash
git add .
git commit -m "H01 feedback leerkracht verwerkt"
git push

Oefeningen

ADB16 Oefening 1 — Je eerste push

Push de commits die je in H04 aanmaakte naar GitHub:

  1. Open Git Bash in je repository
  2. Controleer: git status
  3. Push: git push
  4. Open je repository op github.com in de browser
  5. Controleer of je bestanden en commits zichtbaar zijn
  6. Maak een screenshot van GitHub die je commits toont
ADB16 / ADB14 Oefening 2 — Pull-cyclus simuleren

Simuleer de dagelijkse startroutine:

  1. Open Git Bash
  2. Navigeer naar je repository
  3. Voer altijd als eerste uit: git pull
  4. Maak een kleine wijziging in een bestand
  5. Doorloop de volledige cyclus: git add . → git commit -m "..." → git push
  6. Controleer op GitHub dat de wijziging zichtbaar is
