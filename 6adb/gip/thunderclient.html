⚡ GIP-project — Software installatie

Thunder Client

Thunder Client is een VS Code-extensie waarmee je HTTP-requests kan sturen naar je eigen Node.js-server. Zo test je je API-routes snel en eenvoudig, zonder extra software te installeren.

💡 Waarvoor gebruik je Thunder Client?

Tijdens het GIP bouw je een Node.js-server met routes zoals POST /api/login of GET /api/items. Thunder Client laat je deze routes testen en debuggen terwijl je ze schrijft — zonder dat je eerst een volledige front-end nodig hebt.

Stap 1 — Installatie

  1. Open Visual Studio Code.
  2. Klik op het Extensions-icoontje in de linkerbalk, of gebruik de sneltoets: Ctrl Shift X
  3. Typ in de zoekbalk: Thunder Client
  4. Kies het resultaat van Ranga Vadhineni — herkenbaar aan het bliksemschicht-icoontje.
  5. Klik op Install.
  6. Na de installatie verschijnt een bliksemicoontje ⚡ in de linkerbalk van VS Code. Klik erop om Thunder Client te openen.

Stap 2 — Een route testen

  1. Start je Node.js-server: open een terminal in je Server/-map en voer npm start uit.
  2. Klik op New Request in het Thunder Client-paneel.
  3. Kies de HTTP-methode (GET, POST, PUT of DELETE) via het dropdown-menu links van de URL-balk.
  4. Vul de URL in van je route, bv. http://localhost:3000/api/items.
  5. Voeg indien nodig een Body toe via het tabblad Body → kies JSON.
  6. Klik op Send en bekijk het antwoord van je server in het rechterpaneel.

Stap 3 — GIP API’s testen

Hieronder vind je voor elke route uit het GIP-project hoe je die test in Thunder Client. Test ze in volgorde — beveiligde routes werken pas nadat je ingelogd bent.

⚠️ Sessie & cookies

Thunder Client bewaart je sessie-cookie automatisch zolang het venster open blijft. Je hoeft dus maar één keer in te loggen. Sluit je Thunder Client, dan ben je uitgelogd en moet je opnieuw starten met POST /api/login.

🌎 Publieke routes (geen login vereist)

Deze routes zijn toegankelijk voor iedereen, ook zonder sessie.

GET — alle activiteiten ophalen
Methode : GET
URL     : http://localhost:3000/api/activiteiten
Body    : (leeg)

Verwacht antwoord: 200 OK
[ { "id": 1, "titel": "Voetbal", ... }, ... ]

🔒 Authenticatie

Test registratie en login. Begin altijd met registreren of inloggen voordat je beveiligde routes test.

POST — registreren
Methode : POST
URL     : http://localhost:3000/api/register
Body    : JSON

{
  "username": "testuser",
  "password": "mijnWachtwoord123"
}

Verwacht antwoord: 201 Created
{ "message": "Gebruiker aangemaakt" }
POST — inloggen
Methode : POST
URL     : http://localhost:3000/api/login
Body    : JSON

{
  "username": "testuser",
  "password": "mijnWachtwoord123"
}

Verwacht antwoord: 200 OK
{ "message": "Ingelogd", "user": { "id": 1, "username": "testuser", "rol": "user" } }
GET — huidige gebruiker opvragen
Methode : GET
URL     : http://localhost:3000/api/mij
Body    : (leeg)

Verwacht antwoord: 200 OK  (als je ingelogd bent)
{ "id": 1, "username": "testuser", "rol": "user" }

Fout als niet ingelogd: 401 Unauthorized
POST — uitloggen
Methode : POST
URL     : http://localhost:3000/api/logout
Body    : (leeg)

Verwacht antwoord: 200 OK
{ "message": "Uitgelogd" }

👤 Gebruikersdashboard (login vereist)

Log eerst in als gewone gebruiker. Deze routes werken enkel met een actieve sessie en tonen alleen de eigen activiteiten.

GET — eigen activiteiten ophalen
Methode : GET
URL     : http://localhost:3000/api/mijn-activiteiten
Body    : (leeg)

Verwacht antwoord: 200 OK
[ { "id": 1, "naam": "Yoga", "datum": "2026-05-10", "user_id": 2, ... } ]
POST — nieuwe activiteit aanmaken
Methode : POST
URL     : http://localhost:3000/api/activiteiten
Body    : JSON

{
  "naam": "Yoga",
  "beschrijving": "Wekelijkse yogales",
  "datum": "2026-05-10",
  "starttijd": "09:00",
  "locatie": "Sportzaal",
  "max_deelnemers": 15
}

Verwacht antwoord: 201 Created
{ "id": 3, "bericht": "Activiteit aangemaakt" }
→ user_id wordt automatisch ingesteld op de ingelogde gebruiker
PUT — eigen activiteit bewerken
Methode : PUT
URL     : http://localhost:3000/api/activiteiten/3
Body    : JSON

{
  "naam": "Yoga (gevorderd)",
  "beschrijving": "Wekelijkse yogales voor gevorderden",
  "datum": "2026-05-10",
  "starttijd": "09:00",
  "locatie": "Sportzaal",
  "max_deelnemers": 10
}

Verwacht antwoord: 200 OK
{ "bericht": "Activiteit bijgewerkt" }

Fout als niet jouw activiteit: 403 Forbidden
DELETE — eigen activiteit verwijderen
Methode : DELETE
URL     : http://localhost:3000/api/activiteiten/3
Body    : (leeg)

→ vervang "3" door het id van jouw activiteit

Verwacht antwoord: 204 No Content
Fout als niet jouw activiteit: 403 Forbidden

Veelgemaakte fouten

⚠️ Connection refused

Je server staat niet aan. Open een terminal in je Server/-map en voer npm start uit. Controleer of je geen foutmelding ziet in de terminal.

⚠️ 404 Not Found

De route bestaat niet op je server. Controleer de URL (bv. /api/items en niet /api/item) en of de route correct gedefinieerd is in server.js.

⚠️ 401 Unauthorized

Je probeert een beveiligde route aan te spreken zonder actieve sessie. Log eerst in via POST /api/login. Thunder Client bewaart de cookie automatisch als je Cookies ingeschakeld laat.
