1. Python functies en modules recap

Alvorens Flask te bouwen, herhalen we hoe we Python-code netjes organiseren:

# sensor.py — apart module-bestand
import random

def lees_temperatuur():
    """Simuleer een temperatuursensor (in productie: echte sensor)."""
    return round(random.uniform(18.0, 26.0), 1)

def lees_luchtvochtigheid():
    return round(random.uniform(40.0, 80.0), 1)

# app.py — hoofdprogramma
from sensor import lees_temperatuur, lees_luchtvochtigheid

temp  = lees_temperatuur()
vocht = lees_luchtvochtigheid()
print(f"Temperatuur: {temp}°C, Luchtvochtigheid: {vocht}%")

2. Flask installatie op de Raspberry Pi

Flask is een lichtgewicht Python-webframework. We installeren het in een virtual environment (afgesloten Python-omgeving):

# Virtual environment aanmaken (eenmalig)
python3 -m venv venv

# Virtual environment activeren
source venv/bin/activate    # Linux/Mac
# (venv) verschijnt nu in de terminal

# Flask installeren
pip install flask

# Virtual environment deactiveren
deactivate
Waarom een virtual environment?

Een virtual environment isoleert de packages van een project van de rest van het systeem. Zo voorkom je conflicten tussen projecten die verschillende versies van hetzelfde package nodig hebben.

3. Flask basis-app

# app.py
from flask import Flask, jsonify, render_template
import random
import datetime

app = Flask(__name__)

# Route: HTML-pagina
@app.route("/")
def index():
    return render_template("index.html")

# Route: JSON API-endpoint
@app.route("/api/sensor")
def sensor_data():
    data = {
        "temperatuur":     round(random.uniform(18.0, 26.0), 1),
        "luchtvochtigheid": round(random.uniform(40.0, 80.0), 1),
        "tijdstip":        datetime.datetime.now().isoformat()
    }
    return jsonify(data)

# Route met parameter
@app.route("/api/sensor/<int:aantal>")
def meerdere_metingen(aantal):
    metingen = []
    for _ in range(min(aantal, 100)):   # max 100
        metingen.append({
            "temperatuur": round(random.uniform(18.0, 26.0), 1),
            "tijdstip":    datetime.datetime.now().isoformat()
        })
    return jsonify(metingen)

if __name__ == "__main__":
    # host="0.0.0.0" zorgt dat de server bereikbaar is vanop het netwerk
    app.run(host="0.0.0.0", port=5000, debug=False)

HTML-template aanmaken (templates/index.html):

<!DOCTYPE html>
<html>
<head><title>Raspberry Pi Dashboard</title></head>
<body>
  <h1>Sensordata</h1>
  <div id="data">Laden...</div>
  <script>
    async function update() {
      const resp = await fetch("/api/sensor");
      const d = await resp.json();
      document.getElementById("data").innerHTML =
        `Temperatuur: ${d.temperatuur}°C | Vochtigheid: ${d.luchtvochtigheid}%`;
    }
    update();
    setInterval(update, 5000);  // elke 5 seconden vernieuwen
  </script>
</body>
</html>
# Flask starten
source venv/bin/activate
python app.py

# Bezoek vanop een andere computer in hetzelfde netwerk:
# http://<IP-van-Pi>:5000

4. Cron jobs op Linux

Een cron job is een automatisch gepland script dat op vaste tijdstippen uitgevoerd wordt.

# Crontab bewerken
crontab -e

# Formaat: min uur dag-v-maand maand dag-v-week commando
# *  = elk/elke

# Elke minuut
* * * * * python3 /home/pi/sensor_log.py

# Elke dag om 08:30
30 8 * * * python3 /home/pi/dagrapport.py

# Elke maandag om 09:00
0 9 * * 1 python3 /home/pi/weekrapport.py

# Elke 15 minuten
*/15 * * * * python3 /home/pi/sensor_log.py

# Logs opslaan (>> = toevoegen aan bestand)
*/5 * * * * python3 /home/pi/sensor_log.py >> /home/pi/logs/sensor.log 2>&1
# sensor_log.py — script dat door cron uitgevoerd wordt
import datetime
import json

def log_meting():
    meting = {
        "tijdstip":    datetime.datetime.now().isoformat(),
        "temperatuur": 22.5   # in productie: echte sensor
    }
    with open("/home/pi/data/metingen.jsonl", "a") as f:
        f.write(json.dumps(meting) + "\n")

if __name__ == "__main__":
    log_meting()

5. systemd service voor Flask

Om Flask automatisch te starten bij het opstarten van de Pi, maken we een systemd service aan:

# Bestand aanmaken: /etc/systemd/system/flask-app.service
sudo nano /etc/systemd/system/flask-app.service
[Unit]
Description=Flask Sensor Dashboard
After=network.target

[Service]
Type=simple
User=pi
WorkingDirectory=/home/pi/flask-app
ExecStart=/home/pi/flask-app/venv/bin/python app.py
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
# Service activeren en starten
sudo systemctl daemon-reload
sudo systemctl enable flask-app    # automatisch starten bij boot
sudo systemctl start  flask-app    # nu starten

# Status controleren
sudo systemctl status flask-app

# Logs bekijken
sudo journalctl -u flask-app -f

Samenvatting

Samenvatting
  • Gebruik virtual environments (python3 -m venv venv) voor project-isolatie.
  • Flask: @app.route("/pad") definieert een route. jsonify() stuurt JSON. render_template() toont HTML.
  • Start Flask met host="0.0.0.0" zodat andere toestellen verbinding kunnen maken.
  • Cron jobs: plannen op basis van minuut/uur/dag/maand/weekdag.
  • systemd service: enable = automatisch starten bij boot; start = nu starten.
