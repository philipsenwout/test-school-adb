1. Flask met Jinja2 templates

In H02 maakten we een statische Flask-pagina. Nu gebruiken we Jinja2 templates om dynamische HTML te genereren vanuit Python-data:

# app.py
from flask import Flask, render_template, request, redirect, url_for

app = Flask(__name__)

metingen = []   # tijdelijke lijst — later vervangen door database

@app.route("/")
def index():
    return render_template("index.html", metingen=metingen)

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=5000, debug=False)

Template templates/index.html met Jinja2-syntax:

<!DOCTYPE html>
<html lang="nl">
<head>
  <title>Pi Monitor</title>
  <link rel="stylesheet" href="{{ url_for('static', filename='style.css') }}">
</head>
<body>
  <h1>Metingen</h1>

  <!-- Jinja2: lus over een lijst -->
  {% if metingen %}
    <table>
      <tr><th>Tijdstip</th><th>Temperatuur</th></tr>
      {% for m in metingen %}
        <tr>
          <td>{{ m.tijdstip }}</td>
          <td>{{ m.temperatuur }} °C</td>
        </tr>
      {% endfor %}
    </table>
  {% else %}
    <p>Nog geen metingen beschikbaar.</p>
  {% endif %}
</body>
</html>
Jinja2 cheatsheet
  • {{ variabele }} — waarde tonen
  • {% if … %} … {% endif %} — voorwaarde
  • {% for x in lijst %} … {% endfor %} — lus
  • {{ url_for('route_naam') }} — URL genereren

2. Formulieren verwerken (POST)

Met een HTML-formulier kan een gebruiker data naar de server sturen via een POST-request:

<!-- templates/index.html — formulier toevoegen -->
<form method="POST" action="/meting-toevoegen">
  <label>Temperatuur: <input type="number" step="0.1" name="temp" required></label>
  <button type="submit">Opslaan</button>
</form>
# app.py — route voor POST-verzoek
import datetime

@app.route("/meting-toevoegen", methods=["POST"])
def meting_toevoegen():
    temp = request.form.get("temp")
    if temp:
        metingen.append({
            "tijdstip":    datetime.datetime.now().strftime("%Y-%m-%d %H:%M"),
            "temperatuur": float(temp)
        })
    return redirect(url_for("index"))   # terug naar de hoofdpagina
GET vs POST
  • GET: data in de URL (?param=waarde) — voor opvragen van informatie
  • POST: data in de body van het verzoek — voor het wijzigen van data (formulieren, aanmaken, verwijderen)

3. SQLite database koppelen

SQLite is een lichtgewicht database die als een enkel bestand opgeslagen wordt — perfect voor de Raspberry Pi.

# database.py — database initialiseren
import sqlite3

DB_PATH = "metingen.db"

def init_db():
    """Maak de tabel aan als die nog niet bestaat."""
    conn = sqlite3.connect(DB_PATH)
    conn.execute("""
        CREATE TABLE IF NOT EXISTS metingen (
            id          INTEGER PRIMARY KEY AUTOINCREMENT,
            tijdstip    TEXT    NOT NULL,
            temperatuur REAL    NOT NULL
        )
    """)
    conn.commit()
    conn.close()

def voeg_meting_toe(tijdstip, temperatuur):
    conn = sqlite3.connect(DB_PATH)
    conn.execute(
        "INSERT INTO metingen (tijdstip, temperatuur) VALUES (?, ?)",
        (tijdstip, temperatuur)
    )
    conn.commit()
    conn.close()

def haal_metingen_op(limit=50):
    conn = sqlite3.connect(DB_PATH)
    cursor = conn.execute(
        "SELECT tijdstip, temperatuur FROM metingen ORDER BY id DESC LIMIT ?",
        (limit,)
    )
    rijen = cursor.fetchall()
    conn.close()
    return [{"tijdstip": r[0], "temperatuur": r[1]} for r in rijen]
# app.py — database gebruiken
from database import init_db, voeg_meting_toe, haal_metingen_op
import datetime

app = Flask(__name__)
init_db()   # eenmalig aanmaken bij opstart

@app.route("/")
def index():
    metingen = haal_metingen_op()
    return render_template("index.html", metingen=metingen)

@app.route("/meting-toevoegen", methods=["POST"])
def meting_toevoegen():
    temp = request.form.get("temp")
    if temp:
        voeg_meting_toe(
            tijdstip=datetime.datetime.now().strftime("%Y-%m-%d %H:%M"),
            temperatuur=float(temp)
        )
    return redirect(url_for("index"))

4. REST API bouwen

Een REST API laat toe dat andere applicaties (bv. een JavaScript-frontend of een ander apparaat) data opvragen en versturen via JSON.

# app.py — API-routes toevoegen
from flask import jsonify

# GET: alle metingen opvragen als JSON
@app.route("/api/metingen")
def api_metingen():
    metingen = haal_metingen_op()
    return jsonify(metingen)

# POST: meting toevoegen via API (JSON body)
@app.route("/api/metingen", methods=["POST"])
def api_meting_toevoegen():
    data = request.get_json()
    if not data or "temperatuur" not in data:
        return jsonify({"fout": "temperatuur veld ontbreekt"}), 400

    voeg_meting_toe(
        tijdstip=datetime.datetime.now().strftime("%Y-%m-%d %H:%M"),
        temperatuur=float(data["temperatuur"])
    )
    return jsonify({"status": "ok"}), 201

# DELETE: alle metingen wissen
@app.route("/api/metingen", methods=["DELETE"])
def api_metingen_wissen():
    conn = sqlite3.connect("metingen.db")
    conn.execute("DELETE FROM metingen")
    conn.commit()
    conn.close()
    return jsonify({"status": "gewist"})

De API testen met curl vanuit de terminal:

# Metingen opvragen
curl http://localhost:5000/api/metingen

# Meting toevoegen
curl -X POST http://localhost:5000/api/metingen \
     -H "Content-Type: application/json" \
     -d '{"temperatuur": 23.5}'

# Metingen wissen
curl -X DELETE http://localhost:5000/api/metingen
HTTP-methoden in REST
  • GET — lezen (geen wijziging)
  • POST — aanmaken
  • PUT/PATCH — bijwerken
  • DELETE — verwijderen

5. Projectstructuur

Een gestructureerd Flask-project ziet er zo uit:

pi-monitor/
├── app.py              ← Flask-applicatie
├── database.py         ← database-functies
├── requirements.txt    ← pip-pakketten (flask)
├── metingen.db         ← SQLite-bestand (automatisch aangemaakt)
├── templates/
│   └── index.html      ← Jinja2-template
└── static/
    └── style.css       ← CSS-stijlen
# requirements.txt
flask
# Installeren
pip install -r requirements.txt

# Starten
python app.py

Samenvatting

Samenvatting
  • Jinja2: gebruik {{ }} voor waarden, {% %} voor logica (if/for) in HTML-templates.
  • POST-formulieren: request.form.get("naam") leest het formulierveld; redirect(url_for(...)) stuurt terug.
  • SQLite: lichtgewicht bestandsdatabase, ideaal voor de Pi. Gebruik sqlite3.connect() en sluit altijd de verbinding.
  • REST API: jsonify() stuurt JSON. request.get_json() leest JSON uit een POST-body. HTTP-statuscodes: 200 OK, 201 Created, 400 Bad Request.
  • Structureer je project: app.py, database.py, templates/, static/.
