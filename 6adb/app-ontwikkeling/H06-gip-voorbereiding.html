1. Wat is het GIP?

Het GIP (Geïntegreerde Proef) is het sluitstuk van je opleiding. Je bouwt zelfstandig een volledige webapplicatie die alle kennis van App-ontwikkeling én Databeheer combineert:

LaagTechnologieGeleerd in
Front-endHTML, CSS, JavaScript5ADB + 6ADB H01–H02
Back-end APINode.js + Express6ADB H03, H05
DatabaseMySQL5ADB Databeheer + 6ADB DB

Het GIP duurt meerdere maanden. Je werkt zelfstandig, maar wordt begeleid door je leerkrachten.

3. Functionele analyse

Vóór je begint met coderen, maak je een functionele analyse. Dit document beschrijft wat de applicatie moet doen, voor je beslist hoe je het bouwt.

Onderdelen van de functionele analyse
  • Projectomschrijving: wat doet de applicatie? voor wie?
  • Gebruikersverhalen (user stories): "Als [gebruiker] wil ik [actie] zodat [doel]"
  • Schermen (wireframes): ruwe schetsen van de pagina's
  • Databaseschema (ERD): tabellen, kolommen, relaties
  • API-endpoints: lijst van alle routes met methode, URL en beschrijving
  • Planning: mijlpalen en deadlines

Voorbeeld van user stories:

Als bezoeker wil ik alle producten kunnen bekijken,
  zodat ik kan beslissen wat ik wil kopen.

Als beheerder wil ik nieuwe producten kunnen toevoegen,
  zodat het aanbod up-to-date blijft.

Als beheerder wil ik producten kunnen verwijderen,
  zodat verouderde items van de lijst verdwijnen.

4. Code-organisatie en best practices

// routes/producten.js — routes in een apart bestand
const express = require("express");
const router  = express.Router();
const pool    = require("../db");

router.get("/", async (req, res) => {
  const [rijen] = await pool.query("SELECT * FROM producten");
  res.json(rijen);
});

router.post("/", async (req, res) => {
  // ...
});

module.exports = router;

// index.js — routes koppelen
const productenRouter = require("./routes/producten");
app.use("/api/producten", productenRouter);

Tips voor nette code:

  • Gebruik zinvolle variabelenamen in het Nederlands of Engels (consistent)
  • Voeg commentaar toe dat het waarom uitlegt, niet het wat
  • Elke functie doet één ding (Single Responsibility Principle)
  • Gebruik .env-bestand voor wachtwoorden en gevoelige configuratie

Samenvatting

Samenvatting
  • Het GIP combineert front-end (HTML/CSS/JS), back-end (Node.js/Express) en database (MySQL).
  • Start altijd met een functionele analyse: user stories, ERD, wireframes, API-endpoints.
  • Organiseer je code in mappen: frontend/, backend/, database/, documentatie/.
  • Gebruik routes-bestanden in Express om de code overzichtelijk te houden.
  • Beveiliging: geen hardcoded wachtwoorden, geparametriseerde queries, validatie van input.
  • Controleer de checklist vóór je indient.
