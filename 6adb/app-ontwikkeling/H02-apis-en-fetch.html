API's ophalen met fetch

verdieping 6 u
[LD-APP-14] Externe API's ophalen en verwerken — verdieping

1. Wat is een REST API?

Een API (Application Programming Interface) is een interface waarmee twee programma's met elkaar kunnen communiceren. Een REST API werkt via HTTP en maakt gebruik van de standaard HTTP-methoden:

MethodeGebruikVoorbeeld
GETGegevens ophalenGET /api/producten
POSTNieuwe gegevens aanmakenPOST /api/producten
PUTGegevens volledig bijwerkenPUT /api/producten/5
PATCHGegevens gedeeltelijk bijwerkenPATCH /api/producten/5
DELETEGegevens verwijderenDELETE /api/producten/5

Gegevens worden uitgewisseld in JSON-formaat (JavaScript Object Notation).

2. HTTP Status codes

Elke HTTP-response bevat een statuscode die aangeeft hoe de aanvraag verlopen is:

CodeBetekenisSituatie
200OKAanvraag geslaagd (GET)
201CreatedNieuw object aangemaakt (POST)
204No ContentGeslaagd, geen inhoud terug (DELETE)
400Bad RequestFout in de aanvraag (ongeldige data)
401UnauthorizedNiet ingelogd / geen token
403ForbiddenWel ingelogd, maar geen toegang
404Not FoundResource bestaat niet
500Internal Server ErrorFout aan server-kant

3. fetch en Promises

fetch() is de ingebouwde browser-functie om HTTP-aanvragen te doen. Ze geeft een Promise terug: een object dat een toekomstige waarde vertegenwoordigt (pending → fulfilled of rejected).

// Basis fetch met .then()/.catch()
fetch("https://jsonplaceholder.typicode.com/posts/1")
  .then(response => {
    // response.ok is true als status 200-299
    if (!response.ok) {
      throw new Error("Netwerk-antwoord was niet ok: " + response.status);
    }
    return response.json(); // parse de JSON, geeft opnieuw een Promise
  })
  .then(data => {
    console.log("Titel:", data.title);
    console.log("Inhoud:", data.body);
  })
  .catch(fout => {
    console.error("Fout bij ophalen:", fout.message);
  });

4. async/await

De async/await-syntaxis maakt asynchrone code leesbaarder — het ziet er uit als synchrone code. Een async-functie geeft altijd een Promise terug. await pauzeert de uitvoering tot de Promise opgelost is.

// Equivalent met async/await
async function haalPostOp(id) {
  const response = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`);

  if (!response.ok) {
    throw new Error(`HTTP-fout: ${response.status}`);
  }

  const data = await response.json();
  return data;
}

// Aanroepen
async function init() {
  try {
    const post = await haalPostOp(1);
    console.log("Titel:", post.title);
  } catch (fout) {
    console.error("Fout:", fout.message);
  }
}

init();

5. Foutafhandeling bij fetch

Belangrijk — fetch gooit geen fout bij 404/500!

fetch() gooit enkel een fout als de verbinding volledig mislukt (bv. geen internet). Bij HTTP-codes zoals 404 of 500 is de Promise fulfilled maar response.ok is false. Je moet dit zelf controleren.

async function veiligOphalen(url) {
  try {
    const response = await fetch(url);

    // Controleer statuscode
    if (!response.ok) {
      if (response.status === 404) {
        throw new Error("Pagina niet gevonden (404)");
      } else if (response.status === 500) {
        throw new Error("Serverfout (500) — probeer later opnieuw");
      } else {
        throw new Error(`HTTP-fout: ${response.status}`);
      }
    }

    return await response.json();
  } catch (fout) {
    // Verbindingsfout (offline, CORS, ...)
    if (fout instanceof TypeError) {
      console.error("Verbindingsfout:", fout.message);
    } else {
      console.error("Fout:", fout.message);
    }
    return null;
  }
}

6. Resultaten in de DOM tonen

In de praktijk haal je data op via fetch en toon je die in de pagina. Hier is een volledig voorbeeld:

// HTML:
// <input id="zoek" placeholder="Gebruikersnaam">
// <button id="zoekBtn">Zoek</button>
// <div id="resultaat"></div>

const zoekBtn = document.getElementById("zoekBtn");
const resultaatDiv = document.getElementById("resultaat");

zoekBtn.addEventListener("click", async () => {
  const naam = document.getElementById("zoek").value.trim();
  if (!naam) return;

  resultaatDiv.innerHTML = "<p>Laden...</p>";

  try {
    const response = await fetch(`https://api.github.com/users/${naam}`);

    if (!response.ok) {
      throw new Error(response.status === 404
        ? "Gebruiker niet gevonden"
        : `Fout: ${response.status}`);
    }

    const gebruiker = await response.json();

    resultaatDiv.innerHTML = `
      <h3>${gebruiker.login}</h3>
      <p>Naam: ${gebruiker.name ?? "Onbekend"}</p>
      <p>Publieke repos: ${gebruiker.public_repos}</p>
    `;
  } catch (fout) {
    resultaatDiv.innerHTML = `<p class="fout">${fout.message}</p>`;
  }
});

Samenvatting

Samenvatting
  • Een REST API gebruikt HTTP-methoden (GET, POST, PUT, DELETE) en JSON-formaat.
  • HTTP-statuscodes: 200 (OK), 201 (aangemaakt), 404 (niet gevonden), 500 (serverfout).
  • fetch() geeft een Promise terug. Gebruik .then() of async/await.
  • Controleer altijd response.ok — fetch gooit zelf geen fout bij 404/500.
  • async/await maakt asynchrone code leesbaarder. Gebruik altijd try/catch errond.
  • Promise.all() laat toe meerdere requests tegelijk te doen.
