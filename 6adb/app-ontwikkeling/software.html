💻 App-ontwikkeling 6ADB

Software installatie

Installeer alle tools die je nodig hebt voordat je begint. In 6ADB gebruiken we naast VS Code ook Node.js voor server-side JavaScript. Volg de stappen in volgorde.

📌 Wat installeren we?
  • Visual Studio Code — onze code-editor (al geïnstalleerd vanuit 5ADB)
  • Extra VS Code extensions — specifieke plugins voor Node.js-ontwikkeling
  • Node.js & npm — JavaScript runtime voor de server + pakketbeheer
⚠ VS Code al geïnstalleerd?

Als je VS Code nog niet hebt, installeer het eerst via de 5ADB software-gids. Kom daarna terug om de extra tools voor 6ADB te installeren.

⚙

Node.js & npm

JavaScript runtime voor de server — Windows, macOS en Linux

Node.js laat toe om JavaScript buiten de browser uit te voeren — rechtstreeks op je computer of server. We gebruiken het in 6ADB vanaf H20 om een eigen back-end te bouwen met Express. npm (Node Package Manager) wordt automatisch meegeïnstalleerd en dient om pakketten zoals Express te installeren.

Installatiestappen

1
Ga naar de officiële website

Open je browser en surf naar nodejs.org. Je ziet twee versies: kies altijd de LTS-versie (Long Term Support) — die is stabiel en geschikt voor schoolgebruik.

2
Download het installatiebestand

Windows: download het .msi installatiebestand (Windows Installer).
macOS: download het .pkg bestand.
Linux: gebruik de package manager van je distributie.

3
Voer de installatie uit

Windows: dubbelklik op het .msi bestand en volg de wizard. Laat alle standaardinstellingen staan. Vink zeker aan: Automatically install the necessary tools.
macOS: dubbelklik op het .pkg bestand en volg de installatieprocedure.
De installatie voegt node en npm automatisch toe aan je systeem-PATH.

4
Controleer de installatie via de terminal

Open een terminal (Windows: Win+R → typ cmd of gebruik de terminal in VS Code) en voer de volgende twee commando's uit:

Bash
node --version
npm --version

Je ziet iets zoals v22.x.x en 10.x.x. Als je versienummers ziet, is de installatie gelukt.

✔ Geïntegreerde terminal in VS Code

In VS Code kan je de terminal rechtstreeks openen via Ctrl+` (Windows/Linux) of Cmd+` (macOS). Je hoeft nooit een apart venster te openen — typ al je Node.js-commando's direct in VS Code.

Extra VS Code Extensions

Aanvullend op de extensions uit 5ADB — specifiek voor Node.js & back-end

Installeer de extensions via Ctrl+Shift+X / Cmd+Shift+X in VS Code. Zoek op de naam of de extension-id en klik op Install.

Verplichte extensions (nieuw in 6ADB)

⛏ REST Client

Stuur HTTP-verzoeken (GET, POST, DELETE …) rechtstreeks vanuit VS Code om je eigen API te testen — zonder aparte tool.

Zoek op: humao.rest-client

🚀 ESLint

Waarschuwt je direct in VS Code bij veelgemaakte JavaScript-fouten en stijlproblemen. Handig voor clean code bij Node.js-projecten.

Zoek op: dbaeumer.vscode-eslint

📄 DotENV

Kleuring en syntax-highlighting voor .env-bestanden, waarin je API-sleutels en configuratievariabelen opslaat.

Zoek op: mikestead.dotenv

📊 Thunder Client

Lichtgewicht alternatief voor Postman — test GET/POST/DELETE/PATCH requests tegen je Express API vanuit VS Code zelf.

Zoek op: rangav.vscode-thunder-client

Je eerste Node.js-project opzetten

Verifieer dat alles werkt met een eenvoudig testproject

Doe dit eenmalig om te controleren dat Node.js en npm correct werken.

1
Maak een nieuwe map aan en open die in VS Code
Bash
mkdir mijn-node-test
cd mijn-node-test
code .
2
Initialiseer een npm-project

In de VS Code-terminal:

Bash
npm init -y

Dit maakt een package.json aan. Voeg daarna "type": "module" toe zodat je import/export kan gebruiken.

3
Installeer Express
Bash
npm install express

Je ziet een node_modules/ map verschijnen. Dit is normaal — zet deze nooit mee in een indiening.

4
Maak een testserver

Maak een bestand server.js aan met deze inhoud:

JavaScript
import express from 'express';
const app = express();

app.get('/', (req, res) => {
    res.send('Node.js werkt!');
});

app.listen(3000, () => {
    console.log('Server draait op poort 3000');
});

Start de server met node server.js en open http://localhost:3000 in je browser. Als je "Node.js werkt!" ziet, is alles correct ingesteld.
