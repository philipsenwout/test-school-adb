Node.js basics

verdieping 6 u
[LD-APP-15] Server-side programmeren met Node.js en Express — verdieping

1. Wat is Node.js?

Tot nu toe draaide al je JavaScript in de browser. Node.js laat toe om JavaScript uit te voeren op de server, buiten de browser. Dit opent de deur naar:

  • Webservers bouwen in JavaScript
  • Bestanden lezen en schrijven op de server
  • Verbinding maken met een database
  • Automatiseringsscripts schrijven
Definitie — Node.js

Node.js is een JavaScript-runtime gebouwd op de V8-engine (dezelfde als in Chrome). Het gebruikt een event-driven, non-blocking I/O-model, wat het efficiënt maakt voor netwerktoepassingen.

# Node.js versie controleren
node --version   # v20.x.x

# JavaScript-bestand uitvoeren
node mijnscript.js

2. npm: pakketbeheer

npm (Node Package Manager) is de pakketbeheerder van Node.js. Hiermee installeer je externe bibliotheken (packages).

# Nieuw project aanmaken (maakt package.json aan)
npm init -y

# Package installeren (wordt toegevoegd aan node_modules/)
npm install express

# Package installeren enkel voor development
npm install --save-dev nodemon

# Script uitvoeren uit package.json
npm run start

De package.json houdt bij welke packages je project gebruikt:

{
  "name": "mijn-api",
  "version": "1.0.0",
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js"
  },
  "dependencies": {
    "express": "^4.18.2"
  },
  "devDependencies": {
    "nodemon": "^3.0.1"
  }
}
Tip — nodemon

nodemon herstart de server automatisch bij elke bestandswijziging. Ideaal tijdens het ontwikkelen: je hoeft de server niet telkens handmatig te stoppen en te starten.

3. Express.js

Express.js is het meest gebruikte framework voor Node.js webservers. Het maakt het eenvoudig om routes en middleware te definiëren.

3.1 Eerste Express-server

// index.js
const express = require("express");
const app = express();
const POORT = 3000;

// Eenvoudige route
app.get("/", (req, res) => {
  res.send("Hallo wereld!");
});

// Server starten
app.listen(POORT, () => {
  console.log(`Server draait op http://localhost:${POORT}`);
});

Open http://localhost:3000 in je browser en je ziet "Hallo wereld!"

3.2 Routes: GET, POST, PUT, DELETE

// GET: gegevens ophalen
app.get("/producten", (req, res) => {
  res.json([
    { id: 1, naam: "Laptop", prijs: 999 },
    { id: 2, naam: "Muis",   prijs:  25 }
  ]);
});

// Route parameter (:id)
app.get("/producten/:id", (req, res) => {
  const id = parseInt(req.params.id);
  res.json({ id, naam: "Voorbeeld product" });
});

// Query parameter (?categorie=elektronica)
app.get("/zoek", (req, res) => {
  const categorie = req.query.categorie;
  res.json({ resultaten: `Zoekresultaten voor: ${categorie}` });
});

// POST: nieuw product aanmaken
app.post("/producten", (req, res) => {
  const nieuw = req.body; // vereist express.json() middleware!
  console.log("Ontvangen:", nieuw);
  res.status(201).json({ id: 3, ...nieuw });
});

// DELETE: product verwijderen
app.delete("/producten/:id", (req, res) => {
  const id = req.params.id;
  res.status(204).send(); // 204 No Content
});

3.4 Middleware

Middleware is code die uitgevoerd wordt vóór de route-handler. Typisch gebruik: JSON parseren, logging, authenticatie.

// express.json() middleware: parseert JSON in de request body
app.use(express.json());

// Eigen logging middleware
app.use((req, res, next) => {
  const tijdstip = new Date().toISOString();
  console.log(`[${tijdstip}] ${req.method} ${req.url}`);
  next(); // verplicht! anders stopt de verwerking
});

// CORS middleware (voor gebruik vanuit browser)
app.use((req, res, next) => {
  res.header("Access-Control-Allow-Origin", "*");
  res.header("Access-Control-Allow-Methods", "GET,POST,PUT,DELETE");
  res.header("Access-Control-Allow-Headers", "Content-Type");
  next();
});

4. Volledige REST API voor producten

const express = require("express");
const app = express();
app.use(express.json());

// In-memory opslag (later: echte database)
let producten = [
  { id: 1, naam: "Laptop",  prijs: 999 },
  { id: 2, naam: "Muis",    prijs:  25 },
  { id: 3, naam: "Monitor", prijs: 349 }
];
let volgendId = 4;

// GET alle producten
app.get("/api/producten", (req, res) => {
  res.json(producten);
});

// GET één product
app.get("/api/producten/:id", (req, res) => {
  const product = producten.find(p => p.id === parseInt(req.params.id));
  if (!product) return res.status(404).json({ fout: "Product niet gevonden" });
  res.json(product);
});

// POST nieuw product
app.post("/api/producten", (req, res) => {
  const { naam, prijs } = req.body;
  if (!naam || !prijs) {
    return res.status(400).json({ fout: "naam en prijs zijn verplicht" });
  }
  const nieuw = { id: volgendId++, naam, prijs };
  producten.push(nieuw);
  res.status(201).json(nieuw);
});

// DELETE product
app.delete("/api/producten/:id", (req, res) => {
  const index = producten.findIndex(p => p.id === parseInt(req.params.id));
  if (index === -1) return res.status(404).json({ fout: "Niet gevonden" });
  producten.splice(index, 1);
  res.status(204).send();
});

app.listen(3000, () => console.log("API draait op http://localhost:3000"));

Samenvatting

Samenvatting
  • Node.js laat toe JavaScript buiten de browser uit te voeren (server-side).
  • npm beheert packages. package.json houdt de afhankelijkheden bij.
  • Express.js: app.get/post/put/delete(route, handler) definieert routes.
  • req.params = URL-parameters, req.query = querystring, req.body = POST-body.
  • res.json() stuurt JSON, res.status(code) stelt de statuscode in.
  • Middleware (app.use()) wordt uitgevoerd vóór de route-handler. express.json() is verplicht om POST-body te lezen.

Huistaak H03

Breid de taken-API uit met validatie en extra functionaliteit:

  1. Voeg validatie toe: beschrijving mag niet leeg zijn en mag max. 200 tekens zijn.
  2. Voeg een query-parameter toe aan GET /api/taken?gedaan=true die enkel gedane taken teruggeeft.
  3. Voeg een route toe GET /api/taken/statistieken die het totaal aantal taken, gedane taken en openstaande taken teruggeeft.
  4. Installeer nodemon en configureer het als dev-script in package.json.
