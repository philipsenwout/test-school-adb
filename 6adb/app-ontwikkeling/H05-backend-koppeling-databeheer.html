Backend koppeling met Databeheer

verdieping 6 u
[LD-APP-17] Express API koppelen aan een MySQL-database — verdieping

1. Client-server architectuur

In een volledige webapplicatie werken drie lagen samen:

LaagTechnologieVerantwoordelijkheid
Front-end (client)HTML, CSS, JavaScript (browser)Wat de gebruiker ziet en interageert
Back-end (server)Node.js + ExpressBusinesslogica, API-endpoints, beveiliging
DatabaseMySQLGegevens opslaan en ophalen

De communicatie verloopt als volgt: Browser → HTTP-aanvraag → Express (Node.js) → SQL-query → MySQL → resultaat terug naar Express → JSON-antwoord → Browser.

2. MySQL instellen

We gebruiken de database uit Databeheer. Maak eerst een testdatabase aan:

-- SQL (MySQL Workbench of terminal)
CREATE DATABASE winkel_db;
USE winkel_db;

CREATE TABLE producten (
  id       INT AUTO_INCREMENT PRIMARY KEY,
  naam     VARCHAR(100) NOT NULL,
  prijs    DECIMAL(10,2) NOT NULL,
  voorraad INT DEFAULT 0
);

3. Express koppelen aan MySQL via mysql2

# Package installeren
npm install mysql2

3.1 Verbinding maken

// db.js — databaseverbinding
const mysql = require("mysql2/promise");

const pool = mysql.createPool({
  host:     "localhost",
  user:     "root",
  password: "jouw_wachtwoord",
  database: "winkel_db",
  waitForConnections: true,
  connectionLimit:    10
});

module.exports = pool;
Connection pool vs. enkele verbinding

Een pool beheert meerdere databaseverbindingen tegelijk. Wanneer meerdere gebruikers tegelijk je API aanvragen, wordt elke aanvraag bediend door een beschikbare verbinding uit de pool. Dit is efficiënter dan telkens een nieuwe verbinding openen.

3.2 GET route: data ophalen

// index.js
const express = require("express");
const pool    = require("./db");
const app     = express();
app.use(express.json());

// GET alle producten
app.get("/api/producten", async (req, res) => {
  try {
    const [rijen] = await pool.query("SELECT * FROM producten");
    res.json(rijen);
  } catch (fout) {
    console.error(fout);
    res.status(500).json({ fout: "Databasefout" });
  }
});

// GET één product op ID
app.get("/api/producten/:id", async (req, res) => {
  try {
    const [rijen] = await pool.query(
      "SELECT * FROM producten WHERE id = ?",
      [req.params.id]  // geparametriseerde query — voorkomt SQL-injectie
    );
    if (rijen.length === 0) {
      return res.status(404).json({ fout: "Product niet gevonden" });
    }
    res.json(rijen[0]);
  } catch (fout) {
    res.status(500).json({ fout: "Databasefout" });
  }
});

3.3 POST route: nieuw record invoegen

// POST nieuw product
app.post("/api/producten", async (req, res) => {
  const { naam, prijs, voorraad = 0 } = req.body;

  if (!naam || prijs === undefined) {
    return res.status(400).json({ fout: "naam en prijs zijn verplicht" });
  }

  try {
    const [resultaat] = await pool.query(
      "INSERT INTO producten (naam, prijs, voorraad) VALUES (?, ?, ?)",
      [naam, prijs, voorraad]
    );
    res.status(201).json({
      id: resultaat.insertId,
      naam, prijs, voorraad
    });
  } catch (fout) {
    res.status(500).json({ fout: "Kon product niet opslaan" });
  }
});

4. Foutafhandeling

Altijd geparametriseerde queries!

Gebruik nooit string-concatenatie in SQL-queries: "SELECT * FROM users WHERE naam = '" + naam + "'". Dit opent de deur voor SQL-injectie. Gebruik altijd vraagtekens als placeholders en geef de waarden als array mee: pool.query("... WHERE naam = ?", [naam]).

// Globale foutafhandeling (zet onderaan, na alle routes)
app.use((fout, req, res, next) => {
  console.error("Onbehandelde fout:", fout.stack);
  res.status(500).json({
    fout: "Er is een interne serverfout opgetreden",
    detail: process.env.NODE_ENV === "development" ? fout.message : undefined
  });
});

5. Volledige CRUD API

// PUT: product bijwerken
app.put("/api/producten/:id", async (req, res) => {
  const { naam, prijs, voorraad } = req.body;
  try {
    const [resultaat] = await pool.query(
      "UPDATE producten SET naam=?, prijs=?, voorraad=? WHERE id=?",
      [naam, prijs, voorraad, req.params.id]
    );
    if (resultaat.affectedRows === 0) {
      return res.status(404).json({ fout: "Niet gevonden" });
    }
    res.json({ id: parseInt(req.params.id), naam, prijs, voorraad });
  } catch (fout) {
    res.status(500).json({ fout: "Bijwerken mislukt" });
  }
});

// DELETE: product verwijderen
app.delete("/api/producten/:id", async (req, res) => {
  try {
    const [resultaat] = await pool.query(
      "DELETE FROM producten WHERE id = ?",
      [req.params.id]
    );
    if (resultaat.affectedRows === 0) {
      return res.status(404).json({ fout: "Niet gevonden" });
    }
    res.status(204).send();
  } catch (fout) {
    res.status(500).json({ fout: "Verwijderen mislukt" });
  }
});

app.listen(3000, () => console.log("API draait op poort 3000"));

Samenvatting

Samenvatting
  • Client (browser) → HTTP → Express (Node.js) → SQL → MySQL.
  • mysql2/promise: gebruik een pool voor meerdere verbindingen.
  • Geparametriseerde queries (?-placeholders) voorkomen SQL-injectie.
  • pool.query(sql, [params]) geeft [rijen, velden] terug via destructuring.
  • Controleer altijd affectedRows bij UPDATE/DELETE om 404 te geven als niets gevonden.
  • Gebruik altijd try/catch in async route-handlers.
