22.1 Waarom wachtwoorden hashen?

Als je het wachtwoord "geheim123" gewoon opslaat in de database en die database wordt gehackt, kan iedereen direct alle wachtwoorden lezen.

Hashing zet een wachtwoord om naar een onleesbare reeks tekens:

Bash
"geheim123"  →  bcrypt  →  "$2b$10$K8JepbwUKlV..."

Je kan nooit terugrekenen van de hash naar het origineel. Bij inloggen hash je het ingegeven wachtwoord opnieuw en vergelijk je de twee hashes.

22.3 Users-tabel aanmaken

Voer dit script uit in MySQL Workbench:

SQL
CREATE DATABASE IF NOT EXISTS sportclub_db;
USE sportclub_db;

CREATE TABLE IF NOT EXISTS users (
  id         INT AUTO_INCREMENT PRIMARY KEY,
  naam       VARCHAR(100) NOT NULL,
  email      VARCHAR(150) NOT NULL UNIQUE,
  wachtwoord VARCHAR(255) NOT NULL,
  rol        VARCHAR(20)  NOT NULL DEFAULT 'lid'
);
💡 Waarom VARCHAR(255) voor wachtwoord?

Een bcrypt-hash is altijd 60 tekens — maar met 255 heb je ruimte voor de toekomst.

22.5 Registratie — /api/register

JavaScript
// POST /api/register — nieuwe gebruiker aanmaken
app.post('/api/register', async (req, res) => {
    const { naam, email, wachtwoord } = req.body;

    if (!naam || !email || !wachtwoord) {
        return res.status(400).json({ fout: 'Alle velden zijn verplicht' });
    }
    if (wachtwoord.length < 8) {
        return res.status(400).json({ fout: 'Wachtwoord moet minstens 8 tekens zijn' });
    }

    // Controleer of e-mailadres al bestaat
    const [bestaande] = await pool.execute(
        'SELECT id FROM users WHERE email = ?', [email]
    );
    if (bestaande.length > 0) {
        return res.status(409).json({ fout: 'E-mailadres is al in gebruik' });
    }

    // Wachtwoord hashen
    const hash = await bcrypt.hash(wachtwoord, 10);

    // Gebruiker opslaan
    const [resultaat] = await pool.execute(
        'INSERT INTO users (naam, email, wachtwoord) VALUES (?, ?, ?)',
        [naam, email, hash]
    );

    res.status(201).json({ id: resultaat.insertId, bericht: 'Account aangemaakt' });
});

22.6 Login — /api/login

JavaScript
// POST /api/login — inloggen
app.post('/api/login', async (req, res) => {
    const { email, wachtwoord } = req.body;

    const [rijen] = await pool.execute(
        'SELECT * FROM users WHERE email = ?', [email]
    );
    if (rijen.length === 0) {
        return res.status(401).json({ fout: 'Ongeldig e-mailadres of wachtwoord' });
    }

    const gebruiker = rijen[0];
    const klopt = await bcrypt.compare(wachtwoord, gebruiker.wachtwoord);

    if (!klopt) {
        return res.status(401).json({ fout: 'Ongeldig e-mailadres of wachtwoord' });
    }

    // Sessie aanmaken
    req.session.gebruikerId = gebruiker.id;
    req.session.naam        = gebruiker.naam;
    req.session.rol         = gebruiker.rol;

    res.json({ bericht: 'Ingelogd', naam: gebruiker.naam, rol: gebruiker.rol });
});
💡 Zelfde foutmelding voor onbekend e-mail én fout wachtwoord

Zo weet een aanvaller niet welk deel klopt — e-mailadres of wachtwoord.

22.7 Uitloggen & /api/mij

JavaScript
// POST /api/logout — uitloggen
app.post('/api/logout', (req, res) => {
    req.session.destroy(() => {
        res.json({ bericht: 'Uitgelogd' });
    });
});

// GET /api/mij — wie ben ik?
app.get('/api/mij', (req, res) => {
    if (!req.session.gebruikerId) {
        return res.status(401).json({ fout: 'Niet ingelogd' });
    }
    res.json({ naam: req.session.naam, rol: req.session.rol });
});

22.8 Een route beveiligen

Voeg bovenaan elke beveiligde route een sessiecontrole toe — inline, geen aparte functie:

JavaScript
// GET /api/profiel — enkel voor ingelogde gebruikers
app.get('/api/profiel', async (req, res) => {
    if (!req.session.gebruikerId) {
        return res.status(401).json({ fout: 'Inloggen vereist' });
    }

    const [rijen] = await pool.execute(
        'SELECT id, naam, email, rol FROM users WHERE id = ?',
        [req.session.gebruikerId]
    );
    res.json(rijen[0]);
});

22.9 Login-formulier in de front-end

HTML
<!-- Client/login.html -->
<form id="login-form">
    <label>E-mail<input type="email" id="email" required></label>
    <label>Wachtwoord<input type="password" id="wachtwoord" required></label>
    <button type="submit">Inloggen</button>
</form>
<p id="foutmelding"></p>
JavaScript
// Client/login.js
document.getElementById('login-form').addEventListener('submit', async (e) => {
    e.preventDefault();

    const email      = document.getElementById('email').value;
    const wachtwoord = document.getElementById('wachtwoord').value;

    const response = await fetch('/api/login', {
        method:  'POST',
        headers: { 'Content-Type': 'application/json' },
        body:    JSON.stringify({ email, wachtwoord })
    });
    const data = await response.json();

    if (response.ok) {
        window.location.href = '/dashboard.html';
    } else {
        document.getElementById('foutmelding').textContent = data.fout;
    }
});

Dashboard beveiligen bij het laden:

JavaScript
// Client/dashboard.js
async function checkLogin() {
    const response = await fetch('/api/mij');
    if (!response.ok) {
        window.location.href = '/login.html';
        return;
    }
    const gebruiker = await response.json();
    document.getElementById('welkom').textContent = 'Welkom, ' + gebruiker.naam + '!';
}

checkLogin();

Oefeningen

ADB01 / ADB25 Oefening 1 — Registratieformulier

Maak een register.html met velden voor naam, e-mail en wachtwoord. Koppel het formulier aan POST /api/register.

  • Toon een foutmelding als het e-mailadres al in gebruik is
  • Stuur na succesvolle registratie door naar login.html
ADB09 / ADB25 Oefening 2 — Beschermde pagina

Voeg een dashboard.html toe met een logout-knop.

  • Bij het laden: controleer via GET /api/mij of de gebruiker ingelogd is
  • Zo niet → doorsturen naar login.html
  • De logout-knop roept POST /api/logout aan en stuurt daarna ook door naar login.html

Huistaak

📋 Huistaak — Login-systeem

Opdracht: volledige registratie & login

Deel 1 — Server (5 punten)

  1. POST /api/register — gebruiker aanmaken met gehashed wachtwoord
  2. POST /api/login — inloggen, sessie aanmaken
  3. POST /api/logout — sessie vernietigen
  4. GET /api/mij — naam en rol van ingelogde gebruiker
  5. GET /api/profiel — beveiligde route: geeft naam + e-mail terug (status 401 als niet ingelogd)

Deel 2 — Front-end (3 punten)

  1. register.html — registratieformulier met foutmelding
  2. login.html — loginformulier met foutmelding
  3. dashboard.html — toont naam van ingelogde gebruiker + logout-knop

Deel 3 — Indiening (2 punten)

  1. Lever de Server/-map in zonder node_modules/
  2. Voeg het .sql-bestand toe
