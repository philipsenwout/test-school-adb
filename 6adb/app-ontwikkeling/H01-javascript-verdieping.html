1. Array-methoden

In 5ADB leerde je werken met basisarray's. Nu verkennen we drie krachtige array-methoden die je code compacter en leesbaarder maken: filter(), map() en reduce(). Ze werken allemaal met een callback-functie die op elk element wordt toegepast.

Definitie — Higher-order functions

Een higher-order function is een functie die een andere functie als argument accepteert of als resultaat teruggeeft. filter, map en reduce zijn higher-order functions.

1.1 filter()

filter() maakt een nieuwe array met enkel de elementen waarvoor de callback-functie true teruggeeft. De originele array wordt niet gewijzigd.

const producten = [
  { naam: "Laptop",    prijs: 999 },
  { naam: "Muis",      prijs:  25 },
  { naam: "Monitor",   prijs: 349 },
  { naam: "Toetsenbord", prijs: 79 }
];

// Enkel producten goedkoper dan 100 euro
const goedkoop = producten.filter(p => p.prijs < 100);
console.log(goedkoop);
// [{naam:"Muis", prijs:25}, {naam:"Toetsenbord", prijs:79}]

// Producten waarvan de naam met "M" begint
const metM = producten.filter(p => p.naam.startsWith("M"));
console.log(metM);
// [{naam:"Muis", prijs:25}, {naam:"Monitor", prijs:349}]

1.2 map()

map() maakt een nieuwe array waarbij elk element getransformeerd wordt door de callback. De nieuwe array heeft altijd dezelfde lengte als de originele.

// Prijzen met 21% btw
const prijzenMetBtw = producten.map(p => ({
  naam: p.naam,
  prijs: (p.prijs * 1.21).toFixed(2)
}));
console.log(prijzenMetBtw);
// [{naam:"Laptop", prijs:"1208.79"}, ...]

// Enkel de namen als strings
const namen = producten.map(p => p.naam);
console.log(namen);
// ["Laptop", "Muis", "Monitor", "Toetsenbord"]

1.3 reduce()

reduce() verwerkt een array tot één enkele waarde (getal, object, string…). De callback krijgt twee argumenten: de accumulator (tussenresultaat) en het huidige element.

// Totale prijs berekenen
const totaal = producten.reduce((som, p) => som + p.prijs, 0);
console.log(totaal); // 1452

// Duurste product zoeken
const duurste = producten.reduce((max, p) => p.prijs > max.prijs ? p : max);
console.log(duurste); // {naam:"Laptop", prijs:999}

// Aantal dure producten tellen (filter + reduce gecombineerd)
const aantalDuur = producten
  .filter(p => p.prijs > 100)
  .reduce((tel) => tel + 1, 0);
console.log(aantalDuur); // 2
Tip — Chaining

Je kan filter(), map() en reduce() na elkaar schrijven (chaining). Elke methode geeft een nieuwe array terug waarop je de volgende methode toepast. Dit maakt complexe bewerkingen leesbaar in één vloeiende ketting.

2. Closures

Een closure is een functie die toegang heeft tot variabelen uit de omliggende scope, zelfs nadat die buitenste functie al uitgevoerd is. Dit is een fundamenteel concept in JavaScript.

// Een teller maken met een closure
function maakTeller(start = 0) {
  let waarde = start; // deze variabele "leeft" voort

  return {
    verhoog() { waarde++; },
    verlaag() { waarde--; },
    geefWaarde() { return waarde; }
  };
}

const teller = maakTeller(10);
teller.verhoog();
teller.verhoog();
teller.verlaag();
console.log(teller.geefWaarde()); // 11

// Elke teller is onafhankelijk
const teller2 = maakTeller(0);
teller2.verhoog();
console.log(teller2.geefWaarde()); // 1
console.log(teller.geefWaarde());  // nog steeds 11

Closures worden vaak gebruikt om privé-variabelen te simuleren: waarde is niet rechtstreeks bereikbaar van buitenaf, maar enkel via de teruggegeven methoden.

3. Foutafhandeling met try/catch/throw

In professionele code mag een fout het programma niet volledig doen crashen. We gebruiken try/catch om fouten op te vangen en afhandig te maken.

// Basis try/catch
function deelDoor(a, b) {
  if (b === 0) {
    throw new Error("Deling door nul is niet toegestaan!");
  }
  return a / b;
}

try {
  console.log(deelDoor(10, 2));   // 5
  console.log(deelDoor(10, 0));   // gooit een fout
  console.log("Dit wordt niet bereikt");
} catch (fout) {
  console.error("Fout opgetreden:", fout.message);
} finally {
  console.log("Dit wordt altijd uitgevoerd");
}

// Eigen foutklasse definiëren
class ValidatieFout extends Error {
  constructor(veld, bericht) {
    super(bericht);
    this.name = "ValidatieFout";
    this.veld = veld;
  }
}

function valideerLeeftijd(leeftijd) {
  if (typeof leeftijd !== "number") {
    throw new ValidatieFout("leeftijd", "Leeftijd moet een getal zijn");
  }
  if (leeftijd < 0 || leeftijd > 150) {
    throw new ValidatieFout("leeftijd", "Leeftijd moet tussen 0 en 150 zijn");
  }
  return true;
}

try {
  valideerLeeftijd("twintig");
} catch (fout) {
  if (fout instanceof ValidatieFout) {
    console.error(`Veld '${fout.veld}': ${fout.message}`);
  }
}

4. OOP met klassen

Object-georiënteerd programmeren (OOP) organiseert code in objecten die zowel data (properties) als gedrag (methoden) bevatten. JavaScript ondersteunt OOP via de class-syntaxis (sinds ES6).

4.1 class en constructor

class Boek {
  // constructor wordt aangeroepen bij 'new Boek(...)'
  constructor(titel, auteur, jaar) {
    this.titel  = titel;
    this.auteur = auteur;
    this.jaar   = jaar;
    this.gelezen = false;   // standaardwaarde
  }

  // Methode
  markeerGelezen() {
    this.gelezen = true;
    console.log(`"${this.titel}" is gelezen!`);
  }

  // Getter (property die berekend wordt)
  get leeftijd() {
    return new Date().getFullYear() - this.jaar;
  }

  // toString voor nette weergave
  toString() {
    return `${this.titel} (${this.auteur}, ${this.jaar})`;
  }
}

const boek1 = new Boek("De Ontdekking van de Hemel", "Harry Mulisch", 1992);
console.log(boek1.toString());   // De Ontdekking van de Hemel (Harry Mulisch, 1992)
console.log(boek1.leeftijd);     // aantal jaar oud
boek1.markeerGelezen();
console.log(boek1.gelezen);      // true

4.2 Overerving met extends en super

Een klasse kan een andere klasse uitbreiden met extends. De super()-aanroep roept de constructor van de bovenliggende klasse aan.

class EBoek extends Boek {
  constructor(titel, auteur, jaar, bestandsformaat) {
    super(titel, auteur, jaar); // roept Boek-constructor aan
    this.bestandsformaat = bestandsformaat;
  }

  // Extra methode
  download() {
    console.log(`"${this.titel}" downloaden als ${this.bestandsformaat}...`);
  }

  // toString overschrijven (override)
  toString() {
    return `[eBoek] ${super.toString()} — ${this.bestandsformaat}`;
  }
}

const eboek = new EBoek("Clean Code", "Robert C. Martin", 2008, "EPUB");
console.log(eboek.toString());   // [eBoek] Clean Code (Robert C. Martin, 2008) — EPUB
eboek.markeerGelezen();          // methode geërfd van Boek
eboek.download();                // eigen methode

// instanceof controleren
console.log(eboek instanceof EBoek);  // true
console.log(eboek instanceof Boek);   // ook true (overerving)

Samenvatting

Samenvatting
  • filter() geeft een nieuwe array met elementen waarvoor de callback true is.
  • map() geeft een nieuwe array waarbij elk element getransformeerd is.
  • reduce() verwerkt een array tot één waarde via een accumulator.
  • Een closure is een functie die toegang houdt tot variabelen uit de omliggende scope.
  • try/catch/finally laat toe om fouten op te vangen zonder het programma te crashen. throw gooit een fout.
  • OOP met class: properties in constructor, gedrag als methoden, overerving via extends en super().
