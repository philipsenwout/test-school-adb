2.1 Wat is een object?

In H01 leerde je arrays: lijsten van gelijksoortige waarden op volgorde. Een object is anders: het bundelt samenhangende eigenschappen onder één naam, elk met een eigen label.

JavaScript
// Array: een lijst van adressen
let adressen = ["Kerkstraat 1", "Marktplein 4", "Dorpslaan 12"];

// Object: één adres met al zijn eigenschappen
let adres = {
  straat: "Kerkstraat",
  nummer: 1,
  stad: "Munsterbilzen",
  postcode: 3740
};

Een object heeft key-value paren (sleutel-waarde-paren):

  • De key (sleutel) is de naam van de eigenschap: straat, nummer, ...
  • De value (waarde) is de inhoud: "Kerkstraat", 1, ...

Je maakt een object met accolades { }. Properties worden gescheiden door komma's.

JavaScript
let leerling = {
  voornaam: "Lena",
  familienaam: "Pieters",
  leerjaar: 6,
  isActief: true
};

2.2 Properties lezen: dot notation vs bracket notation

Er zijn twee manieren om een property op te vragen.

Dot notation (puntnotatie)

De meest gebruikte manier — schrijf de key direct na een punt:

JavaScript
let auto = {
  merk: "Toyota",
  model: "Yaris",
  bouwjaar: 2021,
  kleur: "grijs"
};

console.log(auto.merk);     // "Toyota"
console.log(auto.bouwjaar); // 2021
console.log(auto.kleur);    // "grijs"

Bracket notation (haakjesnotatie)

Gebruik vierkante haakjes met de key als string. Dit is nodig als de key in een variabele zit, of als de key spaties of speciale tekens bevat:

JavaScript
let auto = {
  merk: "Toyota",
  "max snelheid": 180
};

console.log(auto["merk"]);         // "Toyota"
console.log(auto["max snelheid"]); // 180 ← dot notation werkt hier niet

// Dynamische key via variabele
let eigenschap = "merk";
console.log(auto[eigenschap]); // "Toyota"
💡 Vuistregel

Gebruik dot notation wanneer je de key kent op voorhand. Gebruik bracket notation wanneer de key variabel is of speciale tekens bevat.

2.3 Properties toevoegen, wijzigen en verwijderen

Eigenschap toevoegen

JavaScript
let persoon = {
  naam: "Omar",
  leeftijd: 17
};

// Nieuwe property toevoegen
persoon.stad = "Hasselt";
persoon.hobby = "fotografie";

console.log(persoon);
// { naam: "Omar", leeftijd: 17, stad: "Hasselt", hobby: "fotografie" }

Eigenschap wijzigen

JavaScript
let persoon = {
  naam: "Omar",
  leeftijd: 17
};

persoon.leeftijd = 18; // verjaardag!
console.log(persoon.leeftijd); // 18

Eigenschap verwijderen

Gebruik het sleutelwoord delete:

JavaScript
let product = {
  naam: "Laptop",
  prijs: 899,
  kortingscode: "ZOMER10"
};

delete product.kortingscode;
console.log(product); // { naam: "Laptop", prijs: 899 }

2.4 Methodes in objecten

Een methode is een functie die als property in een object is opgeslagen. Zo geef je een object eigen gedrag:

JavaScript
let rekenmachine = {
  getal1: 10,
  getal2: 5,
  optellen: function() {
    return rekenmachine.getal1 + rekenmachine.getal2;
  },
  vermenigvuldigen: function() {
    return rekenmachine.getal1 * rekenmachine.getal2;
  }
};

console.log(rekenmachine.optellen());          // 15
console.log(rekenmachine.vermenigvuldigen());  // 50

Kortere notatie

Moderne JavaScript heeft een kortere schrijfwijze voor methodes:

JavaScript
let robot = {
  naam: "R2D2",
  kleur: "wit",
  groeten() {
    console.log("Beep boop! Ik ben " + robot.naam);
  },
  beschrijven() {
    console.log("Kleur: " + robot.kleur);
  }
};

robot.groeten();     // "Beep boop! Ik ben R2D2"
robot.beschrijven(); // "Kleur: wit"

2.5 Geneste objecten

Een property van een object kan zelf ook een object zijn. Dit noemen we een genest object:

JavaScript
let student = {
  naam: "Amber",
  leeftijd: 17,
  adres: {
    straat: "Dorpslaan",
    nummer: 42,
    stad: "Bilzen"
  },
  resultaten: {
    wiskunde: 78,
    informatica: 92,
    Nederlands: 85
  }
};

// Geneste property lezen
console.log(student.adres.stad);             // "Bilzen"
console.log(student.resultaten.informatica); // 92

// Geneste property wijzigen
student.adres.nummer = 43;
console.log(student.adres.nummer); // 43
⚠️ Opgelet

Je kan zo diep nesten als je wil, maar probeer het niet te overdrijven — meer dan 2-3 niveaus maakt code moeilijk leesbaar.

2.6 Array van objecten

In de praktijk combineer je bijna altijd arrays en objecten. Een array van objecten is dé standaard manier om meerdere gelijksoortige items op te slaan:

JavaScript
let leerlingen = [
  { naam: "Ali", punten: 85, geslaagd: true },
  { naam: "Bo",  punten: 52, geslaagd: false },
  { naam: "Cara", punten: 91, geslaagd: true },
  { naam: "Daan", punten: 68, geslaagd: true }
];

// Eerste leerling
console.log(leerlingen[0]);          // { naam: "Ali", ... }
console.log(leerlingen[0].naam);     // "Ali"
console.log(leerlingen[0].punten);   // 85

// Tweede leerling, geslaagd?
console.log(leerlingen[1].geslaagd); // false

// Doorheen alle leerlingen gaan met for...of
for (let leerling of leerlingen) {
  console.log(leerling.naam + ": " + leerling.punten + " punten");
}
// Ali: 85 punten
// Bo: 52 punten
// Cara: 91 punten
// Daan: 68 punten

Dit patroon gebruik je constant — of je nu producten in een webshop toont, recepten opslaat, of gebruikersprofielen beheert.

ADB09 Oefening 1 — Boekenbibliotheek

Schrijf een JavaScript-programma dat een kleine bibliotheek beheert.

  1. Maak één boekobject boek1 met de properties: titel, auteur, aantalPaginas, isGelezen (boolean)
  2. Voeg een methode beschrijf() toe die een volledige zin toont in de console
  3. Wijzig isGelezen naar true
  4. Maak een array bibliotheek met minimaal 3 boekobjecten
  5. Toon de titel en auteur van elk boek via een for...of-lus
  6. Bonus: Voeg een methode beoordeling() toe die op basis van aantalPaginas toont of het een "kort" (<200), "normaal" (200-400) of "dik" (>400) boek is
ADB09 / ADB10 Oefening 2 — Spelersprofielen

Maak een array spelers met 4 spelerobjecten. Elke speler heeft: naam, niveau (getal), levens, highscore en een genest object wapen met type en schade.

  1. Log de naam en het wapentype van de eerste speler
  2. Voeg aan de tweede speler een nieuwe property actief: false toe
  3. Verwijder de highscore property van de derde speler
  4. Maak een methode samenvatting() die een korte beschrijving van de speler teruggeeft als string
  5. Loop over alle spelers en log hun samenvatting
ADB09 Oefening 3 — Weerstation

Maak een object weerstation dat meetgegevens bijhoudt.

JavaScript
let weerstation = {
  locatie: "Munsterbilzen",
  metingen: {
    temperatuur: 18.5,
    luchtvochtigheid: 72,
    windsnelheid: 15
  },

  1. Log de temperatuur via bracket notation: gebruik een variabele met de sleutelnaam
  2. Voeg een methode omschrijving() toe die een leesbare zin genereert
  3. Voeg een nieuwe meting luchtdruk: 1013 toe aan het geneste object
