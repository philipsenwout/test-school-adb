Branching

🎯 Leerdoelen

  • Je kan een nieuwe branch aanmaken met git switch -c

  • Je kan een branch-strategie toepassen voor je GIP-project

2.1 Wat is een branch?

Tot nu toe werkte je altijd op de main-branch — de hoofdlijn van je project. Een branch is een aparte werklijn die je aanmaakt om een nieuwe functionaliteit te bouwen zonder de hoofdcode te verstoren.

Vergelijking: denk aan een boek dat je herwerkt. Je maakt een kopie van het manuscript, past die kopie aan, en als je tevreden bent, voeg je de wijzigingen samen met het origineel. Als het mislukt, staat het origineel nog intact.

2.2 Branch aanmaken en wisselen

Huidige branch bekijken

Bash
git branch

De actieve branch heeft een * voor de naam. Zonder andere branches zie je alleen * main.

Nieuwe branch aanmaken en er naartoe wisselen

Bash
# Modern commando (aanbevolen)
git switch -c feature/contactformulier

# Ouder alternatief (ook geldig)
git checkout -b feature/contactformulier

Naamconventie voor branches:

  • feature/naam-van-functie — voor nieuwe functies
  • fix/beschrijving-van-bug — voor bugfixes
  • gip/module-naam — voor GIP-werk

Wisselen tussen branches

Bash
git switch main                      # terug naar main
git switch feature/contactformulier  # naar je feature branch
⚠ Commit voor je wisselt

Zorg altijd dat je gewijzigde bestanden committed zijn voor je van branch wisselt. Ongecommitte wijzigingen kunnen verloren gaan of meegenomen worden naar een andere branch.

2.3 Werken op een branch

Eens je op de nieuwe branch zit, werk je precies zoals altijd: add, commit, push. Het verschil: je commits komen op de feature-branch, niet op main.

Bash
# Controleer op welke branch je zit
git branch
# Uitvoer: * feature/contactformulier
#            main

# Werk normaal
git add .
git commit -m "contactformulier: HTML-structuur toegevoegd"

# Push de branch naar GitHub
git push -u origin feature/contactformulier

De eerste keer push je met -u origin branchnaam om de link met GitHub in te stellen. Daarna volstaat git push.

2.4 Branch samenvoegen (merge)

Als je functionaliteit klaar is, voeg je de branch samen met main:

Bash
# 1. Ga terug naar main
git switch main

# 2. Zorg dat main up-to-date is
git pull

# 3. Merge de feature-branch
git merge feature/contactformulier

# 4. Push het resultaat
git push

# 5. Optioneel: verwijder de branch (opgeruimd!)
git branch -d feature/contactformulier
📌 Fast-forward merge

Als er geen conflicterende wijzigingen zijn, doet Git een fast-forward merge: de commits van de feature-branch worden direct op main gezet. Geen conflicten, clean history.

2.5 Branch-strategie voor het GIP

Voor het GIP gebruik je de volgende branch-structuur:

Bash
main                    ← stabiele, werkende versie
├── feature/homepage    ← werken aan de startpagina
├── feature/database    ← werken aan de backend
├── feature/login       ← login-functionaliteit
└── fix/nav-bug         ← bugfix

Regel: main is altijd werkend. Je merge nooit kapotte code naar main.

Oefeningen

ADB16 Oefening 1 — Eerste branch aanmaken

Maak je eerste feature-branch aan en werk ermee:

  1. git branch — op welke branch zit je?
  2. Maak een nieuwe branch: git switch -c feature/oefening-pagina
  3. Maak een bestand github-advanced/H02/oefening.html aan
  4. Commit op de feature-branch
  5. Push: git push -u origin feature/oefening-pagina
  6. Ga terug naar main: git switch main
  7. Merge: git merge feature/oefening-pagina
  8. Push main naar GitHub
ADB16 / SV07.07 Oefening 2 — Branch-overzicht op GitHub

Bekijk je branches op GitHub:

  1. Open je repository op GitHub
  2. Klik op het dropdown-menu met de branchnaam (standaard "main")
  3. Zie je de feature-branch verschijnen?
  4. Klik op de feature-branch en bekijk de commits
  5. Maak nog een tweede branch aan: feature/stijl, voeg een CSS-bestand toe, commit en push

Huistaak

📋 Huistaak — Feature branch workflow

Opdracht: Bouw een mini-webpagina via branches

  1. Deel 1 — 4 punten: Maak een branch feature/mini-website aan. Bouw hierop een eenvoudige index.html met:
    • HTML-structuur
    • Een navigatiebalk
    • Een sectie met inhoud
  2. Deel 2 — 3 punten: Commit elke stap apart met duidelijke boodschappen. Push de branch naar GitHub.
  3. Deel 3 — 3 punten: Merge de branch terug naar main en push. Maak een screenshot van de commit-geschiedenis op GitHub die de merge toont.
