1.1 Snelle herhaling 5ADB

Voer de volgende commando's blindelings uit — dit is je basisroutine voor heel het jaar:

Bash
# Begin van elke les
git pull

# Werk vastleggen
git status
git add .
git commit -m "beschrijvende boodschap"
git push

# Historiek bekijken
git log --oneline
⚠ Rust op je kennis

Als je de commando's uit 5ADB nog niet vlot kan uitvoeren, neem dan even de tijd om H04 t.e.m. H06 van 5ADB opnieuw door te nemen voor je verdergaat.

1.2 De GitHub-interface van a tot z

In 5ADB gebruikte je GitHub voornamelijk als opslagplaats. In 6ADB ga je actief met de interface werken. Leer de volgende secties kennen:

📄 Code

Overzicht van alle bestanden. Rechts staan de laatste commits per bestand.

🚬 Issues

Takenlijst en bugtracking. Je kan issues aanmaken, toewijzen en sluiten. Ideaal voor projectwerk.

🤹 Pull Requests

Een verzoek om een branch samen te voegen. Hier geeft de leerkracht code review.

🕐 Commits

Volledige geschiedenis van alle commits. Klik op een commit om precies te zien wat er gewijzigd is.

🌿 Branches

Overzicht van alle branches. In 6ADB maak je zelf branches aan voor nieuwe functies.

1.3 Issues — taakopvolging

Een Issue is een taak, bug of vraag die je wil bijhouden in je project. In de professionele wereld worden issues gebruikt om werk te plannen en op te volgen.

Issue aanmaken

  1. Ga naar je repository op GitHub
  2. Klik op het tabblad Issues
  3. Klik op New issue
  4. Geef een duidelijke titel en beschrijving
  5. Wijs het toe aan jezelf (Assignees)
  6. Klik op Submit new issue

Issue koppelen aan een commit

Je kan een issue sluiten vanuit een commit door het issue-nummer te vermelden in de commit-boodschap:

Bash
# Issue #3 afsluiten via commit
git commit -m "H01 nav-menu afgewerkt, closes #3"

GitHub sluit issue #3 automatisch zodra deze commit gepusht is.

Oefeningen

ADB16 Oefening 1 — Herhalingsoefening basisworkflow

Doorloop de volledige 5ADB-workflow zonder te spieken:

  1. Clone je 6ADB-repository (of gebruik je 5ADB-repo)
  2. Pull de laatste versie
  3. Maak een map github-advanced/H01/ aan met een README.md
  4. Schrijf in dat bestand wat je al weet over Git (in Markdown)
  5. Commit met een goede boodschap
  6. Push naar GitHub
ADB16 / ADB07 Oefening 2 — GitHub interface verkennen

Beantwoord de volgende vragen door ze op te zoeken in jouw repository op GitHub:

  1. Hoeveel commits staan er in jouw repository?
  2. Wanneer was de allereerste commit? Wat was de boodschap?
  3. Klik op een specifieke commit — welke bestanden zijn er gewijzigd?
  4. Maak een Issue aan met als titel "H01 GitHub Advanced afwerken"
  5. Wijs het issue toe aan jezelf
