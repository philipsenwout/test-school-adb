H06 — GitHub Version Control 6ADB

De volledige workflow

ADB16De leerlingen gebruiken een versiebeheersysteem.
ADB07De leerlingen houden de administratie bij in functie van de opdracht.
ADB12De leerlingen documenteren een applicatie.
🎯 Leerdoelen
  • Je kan de dagelijkse Git-workflow correct en zelfstandig uitvoeren
  • Je kent de mappenstructuur van jouw persoonlijke repository
  • Je kan taken en toetsen correct inleveren via GitHub
  • Je begrijpt hoe de leerkracht je werk opvolgt en beoordeelt

6.1 De dagelijkse Git-flow

Dit is de routine die je elke les volgt. Leer deze stappen van buiten — ze worden tweede natuur.

1
Start met pullen
Bash
git pull

Haal altijd de laatste versie op. Zo werk je nooit op verouderde code.

2
Werk aan je project

Open VS Code, schrijf je HTML/CSS/JS, werk aan je oefening of huistaak.

3
Kijk wat er gewijzigd is
Bash
git status
4
Zet bestanden klaar
Bash
git add .
5
Maak een commit
Bash
git commit -m "H01 oef1 nav-structuur afgewerkt"
6
Push naar GitHub
Bash
git push

Je werk staat nu online. Doe dit aan het einde van elke les, niet alleen thuis.

6.2 Jouw mappenstructuur

Jouw persoonlijke repository heeft een vaste structuur. Respecteer deze structuur altijd — de leerkracht gebruikt ze om je werk te vinden en beoordelen.

Bash
jouw-naam-6adb/
├── README.md                    ← beschrijving van je repo
├── app-ontwikkeling/
│   ├── H01-html-basis/
│   │   └── index.html
│   ├── H02-html-tekst/
│   │   └── index.html
│   ├── H03-semantische-html/
│   │   └── index.html
│   │   ...
│   ├── huistaken/
│   │   ├── H01-huistaak/
│   │   │   └── index.html
│   │   ├── H02-huistaak/
│   │   │   └── index.html
│   │   └── ...
│   └── toetsen/
│       ├── toets-1/
│       │   └── index.html
│       └── ...
└── .gitignore                   ← bestanden die Git negeert
⚠ Gebruik exacte mapnamen

Gebruik de mapnamen zoals ze in het schema staan. De leerkracht zoekt automatisch op basis van deze namen. Kies geen andere namen of schrijfwijzen.

6.3 Taken en toetsen inleveren

Huistaken inleveren

  1. Maak je werk in de juiste map: app-ontwikkeling/huistaken/H01-huistaak/
  2. Sla op
  3. git add .
  4. git commit -m "H01 huistaak ingediend"
  5. git push

Toetsen inleveren

Bij toetsen geldt precies hetzelfde, maar met een duidelijkere commit-boodschap:

Bash
git commit -m "toets-1 ingediend - eindversie"
🚫 Niet pushen = niet ingediend

Alleen git push maakt je indiening zichtbaar voor de leerkracht. git commit alleen is niet genoeg. Zorg altijd dat je pusht voor de deadline.

6.4 Hoe de leerkracht je werk bekijkt

De leerkracht:

  1. Opent jouw repository op GitHub
  2. Bekijkt de bestanden en de commit-geschiedenis
  3. Kan commentaar schrijven op specifieke regels code (via Pull Request reviews of inline comments)
  4. Kan een verbeterde versie pushen als correctie

Als de leerkracht iets heeft toegevoegd, doe je git pull om die wijzigingen te halen. Verwerk daarna de feedback en push opnieuw.

📖 GitHub — commentaar lezen

Als je op GitHub een bestand opent, zie je soms gele markeringen of commentaarballonnen. Die zijn van de leerkracht. Lees ze, pas je code aan en commit + push de verbeterde versie.

6.5 Git cheatsheet — alle commando's op een rij

Bash
# ── EÉNMALIG (begin van het jaar) ──────────────────────────
git clone https://github.com/...    # repository downloaden

# ── DAGELIJKS (begin van elke les) ─────────────────────────
git pull                            # laatste versie ophalen

# ── WERK VASTLEGGEN ─────────────────────────────────────────
git status                          # wat is er gewijzigd?
git add .                           # alles klaarzetten
git add bestand.html                # één bestand klaarzetten
git commit -m "omschrijving"        # snapshot vastleggen
git push                            # uploaden naar GitHub

# ── HISTORIEK ───────────────────────────────────────────────
git log                             # alle commits
git log --oneline                   # compacte weergave

# ── CONFIGURATIE ────────────────────────────────────────────
git config --global user.name "Naam"
git config --global user.email "mail@school.be"
git config --list                   # alle instellingen

Oefeningen

ADB16 / ADB07 Oefening 1 — Volledig workflow-oefening

Doorloop de volledige workflow 3 keer achter elkaar:

  1. git pull
  2. Maak een kleine wijziging in een HTML-bestand
  3. git status — beschrijf wat je ziet
  4. git add .
  5. git commit -m "..." — schrijf een goede boodschap
  6. git push
  7. Herhaal 3x
  8. Controleer op GitHub: 3 nieuwe commits zichtbaar?
ADB16 / ADB12 Oefening 2 — README aanvullen

Maak een professionele README.md voor je repository:

  1. Open README.md in VS Code
  2. Voeg toe (in Markdown-formaat):
    • Jouw naam en klas
    • Een overzicht van de vakken in deze repo
    • Hoe de mappenstructuur werkt
  3. Commit en push
  4. Bekijk hoe de README eruitziet op GitHub (het wordt automatisch getoond op de hoofdpagina)

Huistaak

📋 Huistaak — Jaarworkflow instellen

Opdracht: Maak de volledige mappenstructuur aan

  1. Deel 1 — 5 punten: Zorg dat de volgende mappen bestaan in je repository (elke map heeft een leeg .gitkeep bestand zodat Git ze bijhoudt):
    • app-ontwikkeling/H01-html-basis/
    • app-ontwikkeling/huistaken/
    • app-ontwikkeling/toetsen/
  2. Deel 2 — 3 punten: Commit elke stap apart met een duidelijke boodschap. Push alles naar GitHub.
  3. Deel 3 — 2 punten: Stuur de link naar je GitHub-repository naar de leerkracht. De leerkracht controleert de mappenstructuur en de commits.

Tip: maak een leeg bestand aan met touch .gitkeep in de terminal, of maak een leeg bestand aan in VS Code.
