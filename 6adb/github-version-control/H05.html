Geavanceerd GitHub

.gitignore — bestanden uitsluiten

Sommige bestanden wil je niet in je repository opslaan. Denk aan:

  • node_modules/ — duizenden kleine bestanden van npm
  • .env — wachtwoorden en API-sleutels
  • .DS_Store — macOS systeembestanden
  • *.log — logbestanden

Maak een .gitignore bestand aan in de root van je repository:

Bash
touch .gitignore

Voeg patronen toe — elk patroon staat op een aparte regel:

Bash
# .gitignore
node_modules/
.env
.DS_Store
*.log
dist/

Commit daarna het .gitignore bestand zelf:

Bash
git add .gitignore
git commit -m ".gitignore toegevoegd"
git push
💡 GitHub .gitignore templates

Bij het aanmaken van een repository op GitHub kan je een sjabloon kiezen (bv. Node of Python). GitHub genereert dan automatisch een passende .gitignore.

git stash — wijzigingen tijdelijk opzijleggen

Soms wil je snel van branch wisselen, maar heb je onafgemaakte wijzigingen die je nog niet wil committen. git stash bewaart die tijdelijk:

Bash
git stash            # wijzigingen opzijleggen
git switch main      # van branch wisselen
git switch feature/x # terug naar je branch
git stash pop        # wijzigingen terugzetten

Tags & releases

Een tag is een label dat je aan een specifieke commit geeft. Handig om versies aan te duiden:

Bash
# Annotated tag aanmaken
git tag -a v1.0 -m "GIP versie 1.0 — eindpresentatie"

# Tag pushen naar GitHub
git push origin v1.0

# Alle tags bekijken
git tag

Op GitHub kan je een Release aanmaken op basis van een tag: ga naar Releases → Create a new release. Dit maakt je project downloadbaar als zip.

GitHub Pages — je project online publiceren

GitHub Pages publiceert je statische website (HTML/CSS/JS) gratis online op een .github.io-adres. Perfect voor je GIP-frontend.

  1. Ga naar je repository op GitHub
  2. Klik op Settings
  3. Scroll naar de sectie Pages
  4. Kies bij Source: Deploy from a branch
  5. Kies main branch en / (root)
  6. Klik op Save

Na een minuutje is je website beschikbaar op:

Bash
https://gebruikersnaam.github.io/repository-naam/
⚠ Alleen statische sites

GitHub Pages werkt alleen met statische bestanden (HTML, CSS, JS). Server-side code (Node.js, Python Flask) werkt niet op GitHub Pages — daarvoor heb je een aparte hosting nodig.

Oefeningen

ADB16 / ADB13 Oefening 1 — .gitignore instellen

Maak een .gitignore voor je GIP-repository:

  1. Maak een .gitignore bestand aan in de root
  2. Voeg toe: .DS_Store, *.log, node_modules/
  3. Maak een bestand test.log aan — wordt het getrackt door Git?
  4. Controleer met git status
  5. Commit en push het .gitignore bestand
ADB13 Oefening 2 — GIP publiceren via GitHub Pages

Publiceer je GIP-frontend via GitHub Pages:

  1. Zorg dat je index.html in de root van je repository staat
  2. Ga naar Settings → Pages
  3. Activeer GitHub Pages via de main branch
  4. Wacht 1-2 minuten en bezoek je .github.io URL
  5. Test op een smartphone: werkt het responsive?

Huistaak

📋 Huistaak — GIP v1.0 releasen

Opdracht: Maak een officiële release van je GIP-frontend

  1. Deel 1 — 4 punten: Zorg dat je GIP-frontend klaar is op main. Maak een tag aan: git tag -a v1.0 -m "GIP frontend v1.0" en push: git push origin v1.0
  2. Deel 2 — 3 punten: Maak een GitHub Release op basis van de v1.0 tag. Schrijf een release-beschrijving met:
    • Wat zit er in v1.0?
    • Wat is nog niet af voor de volgende versie?
  3. Deel 3 — 3 punten: Activeer GitHub Pages en stuur de publieke URL naar de leerkracht. De website moet werken op een smartphone.
