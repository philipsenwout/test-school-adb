Merge conflicten oplossen

ADB16De leerlingen gebruiken een versiebeheersysteem. [verdieping]
ADB15De leerlingen onderhouden een applicatie en lossen problemen op.
🎯 Leerdoelen
  • Je kan uitleggen wanneer een merge conflict ontstaat
  • Je kan de conflict markers (<<<<<<<, =======, >>>>>>>) begrijpen en lezen
  • Je kan een merge conflict manueel oplossen in VS Code
  • Je kan een conflict-oplossing committen en pushen
  • Je kent strategieën om conflicten te voorkomen

4.1 Wat is een merge conflict?

Een merge conflict treedt op als twee branches hetzelfde bestand op dezelfde plek hebben gewijzigd. Git weet niet welke versie de correcte is en vraagt jou om te beslissen.

Typische situatie:

  1. Je werkt op feature/login en past index.html aan
  2. Tegelijk wordt index.html ook op main gewijzigd (bv. door de leerkracht)
  3. Als je feature/login probeert te mergen naar main: conflict!
📌 Niet panikeren

Een merge conflict is geen fout — het is Git dat om jouw beslissing vraagt. Met de juiste aanpak is het in enkele minuten opgelost.

4.2 Conflict markers lezen

Als Git een conflict detecteert, voegt het markers in het bestand:

HTML
<<<<<<< HEAD
<h1>Welkom op onze website</h1>
=======
<h1>Welkom bij Appschool Munsterbilzen</h1>
>>>>>>> feature/login
  • <<<<<<< HEAD — begin van de huidige versie (main)
  • ======= — scheiding tussen de twee versies
  • >>>>>>> feature/login — versie van de andere branch

4.3 Conflict oplossen in VS Code

VS Code detecteert automatisch conflict markers en toont knoppen om te kiezen:

  • Accept Current Change — bewaar de HEAD-versie
  • Accept Incoming Change — bewaar de branch-versie
  • Accept Both Changes — bewaar beide versies
  • Compare Changes — bekijk de twee versies naast elkaar

Je kan ook manueel de markers verwijderen en de gewenste code bewaren:

HTML
<!-- Verwijder de markers en kies de correcte versie -->
<h1>Welkom bij Appschool Munsterbilzen</h1>

Na het oplossen van alle conflicten:

Bash
git status                    # toont welke bestanden opgelost zijn
git add index.html            # mark als opgelost
git commit -m "merge conflict opgelost: h1 titel"
git push

4.4 Conflicten voorkomen

💡 Best practices
  • Pull altijd eerst voor je begint te werken: git pull
  • Kleine, frequente commits zijn makkelijker te mergen dan grote
  • Communiceer wie aan welk bestand werkt
  • Branches aanpassen aan functionaliteiten, niet aan personen
  • Merge regelmatig van main naar je feature-branch: git merge main

Oefeningen

ADB16 / ADB15 Oefening 1 — Conflict simuleren en oplossen

Maak bewust een merge conflict aan en los het op:

  1. Maak een bestand conflict-oefening.html op main
  2. Commit en push
  3. Maak een branch feature/conflict-test
  4. Wijzig regel 5 op de feature-branch, commit
  5. Ga terug naar main, wijzig dezelfde regel 5, commit
  6. Probeer de feature-branch te mergen: git merge feature/conflict-test
  7. Bekijk de conflict-markers in VS Code
  8. Los op, commit, push
