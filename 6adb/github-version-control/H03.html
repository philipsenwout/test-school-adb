Pull Requests & Code Review

🎯 Leerdoelen
  • Je kan uitleggen wat een Pull Request (PR) is en wanneer je er één aanmaakt
  • Je kan een PR aanmaken op GitHub vanuit een feature-branch
  • Je kan code review ontvangen en begrijpen
  • Je kan feedback verwerken in een bestaande PR
  • Je kan een PR mergen en sluiten

3.1 Wat is een Pull Request?

Een Pull Request (PR) is een formeel verzoek om jouw feature-branch samen te voegen met een andere branch (meestal main). Het is de professionele manier om code te reviewen voor ze in de productieversie belandt.

In het onderwijs werkt het zo:

  1. Je werkt op een feature-branch en pusht naar GitHub
  2. Je maakt een Pull Request aan op GitHub
  3. De leerkracht beoordeelt de code, voegt commentaar toe
  4. Jij verwerkt de feedback en pusht opnieuw
  5. De PR wordt goedgekeurd en gemerged

3.2 Pull Request aanmaken

  1. Push je feature-branch naar GitHub: git push -u origin feature/naam
  2. Ga naar je repository op GitHub
  3. GitHub toont een gele banner: "feature/naam had recent pushes" — klik op Compare & pull request
  4. Of ga via het tabblad Pull Requests → New pull request

De PR beschrijven

Titel

Kort en beschrijvend. Bv. "H01 contactformulier met validatie"

Description

Wat heb je gemaakt? Wat werkt er? Zijn er nog openstaande vragen voor de leerkracht?

Reviewers

Voeg je leerkracht toe als reviewer. Hij/zij krijgt een melding.

3.3 Code review ontvangen

De leerkracht kan op GitHub:

  • Inline commentaar geven op specifieke regels — klik op het + teken naast een regel
  • Een algemene review schrijven: Approve, Comment of Request changes

Je ontvangt een e-mail als er nieuwe commentaren zijn. Bekijk ze op de GitHub PR-pagina.

3.4 Feedback verwerken

Als de leerkracht wijzigingen vraagt:

Bash
# Zorg dat je op de feature-branch zit
git switch feature/naam

# Haal eventuele nieuwe commentaar-commits op
git pull

# Pas je code aan op basis van de feedback
# ... wijzigingen maken in VS Code ...

# Commit en push
git add .
git commit -m "feedback leerkracht verwerkt: validatie toegevoegd"
git push

De PR wordt automatisch bijgewerkt met de nieuwe commits. De leerkracht ziet de wijzigingen onmiddellijk.

3.5 PR mergen en sluiten

Als de leerkracht de PR goedkeurt (Approved), kan je mergen:

  1. Klik op Merge pull request op GitHub
  2. Klik op Confirm merge
  3. Optioneel: klik op Delete branch (opgeruimd!)

Daarna lokaal bijwerken:

Bash
git switch main
git pull
git branch -d feature/naam   # lokale branch opruimen

Oefeningen

ADB16 / ADB14 Oefening 1 — Volledige PR-cyclus

Doorloop een complete PR-cyclus:

  1. Maak een branch feature/pr-oefening
  2. Maak een HTML-bestand met een formulier
  3. Commit en push de branch
  4. Open een Pull Request op GitHub, voeg je leerkracht toe als reviewer
  5. Wacht op feedback (of vraag een medeleerling)
  6. Verwerk minstens één wijziging en push opnieuw
  7. Merge de PR
ADB28 Oefening 2 — PR-beschrijving schrijven

Schrijf een professionele PR-beschrijving voor je oefening van H02 (branching). Zorg voor:

  • Een duidelijke titel
  • Wat je gemaakt hebt (2-3 zinnen)
  • Minstens één vraag of aandachtspunt voor de leerkracht
  • Eventuele gekende beperkingen of openstaande punten

Huistaak

📋 Huistaak — GIP-voorbereiding via PR

Opdracht: Maak een PR voor je GIP-startsectie

  1. Deel 1 — 5 punten: Maak een branch gip/startpagina. Bouw hierop de basisstructuur van je GIP-startpagina (HTML + CSS). Commit in minstens 3 stappen.
  2. Deel 2 — 3 punten: Open een PR op GitHub met een volledige beschrijving. Voeg de leerkracht toe als reviewer.
  3. Deel 3 — 2 punten: Verwerk eventuele feedback en push de verbeterde versie. Merge de PR als de leerkracht akkoord is.
