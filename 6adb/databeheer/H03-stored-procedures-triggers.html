Stored Procedures en Triggers

verdieping 4 u
[LD-DB-11] Stored procedures en triggers schrijven en gebruiken — verdieping

1. Stored Procedures

Een stored procedure is een benoemde reeks SQL-statements die je kunt opslaan in de database en later aanroepen via CALL. Voordelen:

  • Complexe logica eenmalig schrijven en hergebruiken
  • Minder netwerkcommunicatie (minder round-trips)
  • Beveiliging: gebruikers kunnen een procedure aanroepen zonder directe tabelrechten

1.1 Aanmaken

-- DELIMITER wijzigen zodat MySQL ; binnenin de procedure niet als einde ziet
DELIMITER $$

CREATE PROCEDURE haal_bestellingen_op_per_klant(
  IN p_klant_id INT
)
BEGIN
  SELECT
    b.bestelling_id,
    b.besteldatum,
    SUM(br.aantal * p.prijs) AS totaal
  FROM bestellingen b
  JOIN bestelling_regels br ON b.bestelling_id = br.bestelling_id
  JOIN producten p           ON br.product_id   = p.product_id
  WHERE b.klant_id = p_klant_id
  GROUP BY b.bestelling_id, b.besteldatum
  ORDER BY b.besteldatum DESC;
END$$

DELIMITER ;

1.2 Parameters: IN, OUT en INOUT

TypeRichtingGebruik
INInputWaarde doorgeven aan de procedure (kan niet gewijzigd worden)
OUTOutputVariabele vullen vanuit de procedure
INOUTBeideZowel invoer als uitvoer
 
-- Procedure met OUT parameter
DELIMITER $$

CREATE PROCEDURE tel_bestellingen(
  IN  p_klant_id INT,
  OUT p_aantal   INT
)
BEGIN
  SELECT COUNT(*) INTO p_aantal
  FROM bestellingen
  WHERE klant_id = p_klant_id;
END$$

DELIMITER ;

1.3 Aanroepen en beheren

-- Procedure aanroepen (IN parameter)
CALL haal_bestellingen_op_per_klant(3);

-- Procedure aanroepen met OUT parameter
CALL tel_bestellingen(3, @resultaat);
SELECT @resultaat AS aantal_bestellingen;

-- Alle procedures bekijken
SHOW PROCEDURE STATUS WHERE Db = 'winkel_db';

-- Procedure verwijderen
DROP PROCEDURE IF EXISTS haal_bestellingen_op_per_klant;

2. Triggers

Een trigger is SQL-code die automatisch uitgevoerd wordt bij een INSERT, UPDATE of DELETE. Je gebruikt triggers voor:

  • Auditlogs bijhouden (wie heeft wat gewijzigd?)
  • Automatisch berekende kolommen bijwerken
  • Businessregels afdwingen
  • Voorraad automatisch bijwerken bij bestellingen
TimingEventWanneer?
BEFOREINSERT / UPDATE / DELETEVóór de actie — kan de waarden nog wijzigen of actie annuleren
AFTERINSERT / UPDATE / DELETENa de actie — voor logging of gevolgen

2.1 AFTER INSERT — auditlog

-- Auditlog-tabel
CREATE TABLE audit_log (
  log_id      INT AUTO_INCREMENT PRIMARY KEY,
  tabel_naam  VARCHAR(50),
  actie       VARCHAR(20),
  record_id   INT,
  tijdstip    DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Trigger: log elke nieuwe bestelling
DELIMITER $$

CREATE TRIGGER na_bestelling_insert
AFTER INSERT ON bestellingen
FOR EACH ROW
BEGIN
  INSERT INTO audit_log (tabel_naam, actie, record_id)
  VALUES ('bestellingen', 'INSERT', NEW.bestelling_id);
END$$

DELIMITER ;

-- Test de trigger
INSERT INTO bestellingen (klant_id, besteldatum) VALUES (1, CURDATE());
SELECT * FROM audit_log;
NEW en OLD

Binnen een trigger kan je de waarden vóór en na de actie raadplegen:

  • NEW.kolom — de nieuwe waarde (beschikbaar bij INSERT en UPDATE)
  • OLD.kolom — de oude waarde (beschikbaar bij UPDATE en DELETE)

2.2 BEFORE UPDATE — validatie en logging

-- Log prijswijzigingen van producten
CREATE TABLE prijs_log (
  log_id         INT AUTO_INCREMENT PRIMARY KEY,
  product_id     INT,
  oude_prijs     DECIMAL(10,2),
  nieuwe_prijs   DECIMAL(10,2),
  gewijzigd_op   DATETIME DEFAULT CURRENT_TIMESTAMP
);

DELIMITER $$

CREATE TRIGGER voor_prijs_update
BEFORE UPDATE ON producten
FOR EACH ROW
BEGIN
  -- Enkel loggen als prijs verandert
  IF OLD.prijs != NEW.prijs THEN
    INSERT INTO prijs_log (product_id, oude_prijs, nieuwe_prijs)
    VALUES (OLD.product_id, OLD.prijs, NEW.prijs);
  END IF;

  -- Validatie: prijs mag niet negatief zijn
  IF NEW.prijs < 0 THEN
    SIGNAL SQLSTATE '45000'
      SET MESSAGE_TEXT = 'Prijs mag niet negatief zijn';
  END IF;
END$$

DELIMITER ;
-- Trigger testen
UPDATE producten SET prijs = 1099 WHERE product_id = 1;
SELECT * FROM prijs_log;

-- Trigger beheren
SHOW TRIGGERS FROM winkel_db;
DROP TRIGGER IF EXISTS na_bestelling_insert;

Samenvatting

Samenvatting
  • Stored procedures worden opgeslagen in de DB en aangeroepen via CALL.
  • Parameters: IN (invoer), OUT (uitvoer), INOUT (beide).
  • Gebruik DELIMITER $$ bij het aanmaken van procedures en triggers.
  • Triggers draaien automatisch bij INSERT/UPDATE/DELETE op een tabel.
  • NEW.kolom = nieuwe waarde; OLD.kolom = oude waarde.
  • SIGNAL SQLSTATE '45000' stopt een actie en gooit een fout (validatie).
