H06 — Netwerkmonitoring

verdieping 4 u

Leerdoelen

[LD-NET-08] Netwerk monitoren en beheren verdieping

Na dit hoofdstuk kan je:

  • uitleggen waarom netwerkmonitoring essentieel is voor een beheerder
  • diagnostische tools (ping, traceroute, nslookup) correct gebruiken
  • het principe van SNMP en syslog beschrijven
  • het UniFi-dashboard gebruiken voor live monitoring van apparaten en verkeer
  • een netwerk troubleshooten aan de hand van monitoringdata

1. Theorie

1.1 Waarom monitoren?

Als beheerder ben je verantwoordelijk voor de beschikbaarheid en prestaties van het netwerk. Monitoring laat je toe:

  • Proactief problemen te ontdekken vóórdat gebruikers klagen
  • Historische data te raadplegen: wanneer begon het probleem?
  • Capaciteitsplanning: wanneer moet ik uitbreiden?
  • Beveiligingsincidenten te detecteren: ongewoon verkeer, inlogpogingen

1.2 Diagnostische tools

ping — test bereikbaarheid en vertraging (RTT):

$ ping 8.8.8.8
PING 8.8.8.8 (8.8.8.8): 56 data bytes
64 bytes from 8.8.8.8: icmp_seq=0 ttl=117 time=12.5 ms
64 bytes from 8.8.8.8: icmp_seq=1 ttl=117 time=11.8 ms

-- time = Round-Trip Time in milliseconden
-- Request timeout = host niet bereikbaar of ICMP geblokkeerd

traceroute / tracert — toont het pad van pakketten:

$ traceroute 8.8.8.8
1  192.168.1.1 (router)    1.2 ms
2  10.50.0.1   (ISP)       8.3 ms
3  172.20.0.1  (transit)  11.1 ms
4  8.8.8.8     (Google)   12.5 ms

nslookup / dig — DNS-opzoekingen:

$ nslookup google.com
Server:  192.168.1.1 (je lokale DNS)
Address: 192.168.1.1

Non-authoritative answer:
Name:    google.com
Address: 142.250.74.174

netstat / ss — actieve verbindingen bekijken:

$ ss -tuln
-- Toont alle luisterende (LISTEN) poorten op je systeem

1.3 SNMP — Simple Network Management Protocol

📚 Definitie — SNMP

SNMP is een protocol voor het beheren en monitoren van netwerkapparaten (routers, switches, servers). Een SNMP-agent op elk apparaat levert meetwaarden (uptime, bandbreedte, foutentellers) aan een centrale SNMP-manager.

VersieBeveiligingGebruik
SNMPv1Geen encryptie, community stringVerouderd, niet aanbevolen
SNMPv2cCommunity string (plaintext)Nog veel gebruikt, basaal
SNMPv3Encryptie + authenticatieAanbevolen voor productie

1.4 Syslog — gecentraliseerde logboeken

📚 Definitie — Syslog

Syslog is een standaard voor het doorsturen van logberichten van netwerkapparaten naar een centrale logserver. Je kunt alle fouten, waarschuwingen en informatieve berichten van switches, routers en servers op één plek bekijken.

Syslog-niveaus (van kritisch naar informatief):

0 Emergency  - Systeem onbruikbaar
1 Alert      - Onmiddellijke actie vereist
2 Critical   - Kritieke fouten
3 Error      - Foutmeldingen
4 Warning    - Waarschuwingen
5 Notice     - Normale maar significante gebeurtenissen
6 Info       - Informatieve berichten
7 Debug      - Debuginformatie (veel output)

1.5 UniFi-dashboard voor monitoring

Het UniFi Network-dashboard biedt realtime inzicht in je netwerk:

  • Overzicht: verbonden apparaten, actieve clients, uptime per apparaat
  • Clients: IP-adres, MAC-adres, VLAN, datagebruik per client
  • Statistics: bandbreedte per poort, per VLAN, over tijd
  • Alerts: configureerbare meldingen (apparaat offline, onbekend apparaat, hoog dataverbruik)
  • Events: logboek van alle netwerkgebeurtenissen
💡 Tip — UniFi-monitoring in de praktijk

Activeer e-mailmeldingen in UniFi (Settings → Notifications) zodat je automatisch een melding krijgt als een apparaat offline gaat. Zo hoef je niet constant in het dashboard te kijken.

2. Oefeningen

Oefening 6.1 Diagnostische tools

Test je kennis van ping, traceroute, nslookup en SNMP.

Vraag 1
Wat meet de RTT bij het commando ping 8.8.8.8?
Vraag 2
Welk commando toont het pad (alle tussenliggende routers/hops) dat pakketten afleggen naar 8.8.8.8?
Vraag 3
Welke SNMP-versie wordt aanbevolen voor productieomgevingen?
Vraag 4
Wat is het syslog-niveau voor het meest kritische type bericht ("Systeem onbruikbaar")? Geef het nummer.
Oefening 6.2 UniFi-dashboard en troubleshooting

Test je kennis van het UniFi-dashboard en troubleshooting-aanpak.

Vraag 1
Welke informatie biedt het UniFi-dashboard? (Selecteer alle juiste antwoorden)
Vraag 2
Een gebruiker meldt dat hij de schoolwebserver niet kan bereiken. Wat is de eerste stap in troubleshooting?
Vraag 3
Waarvoor gebruik je het UniFi Events-logboek?

3. Samenvatting

📋 Samenvatting
  • Monitoring: proactief problemen detecteren, historische data, capaciteitsplanning, beveiligingsincidenten.
  • Tools: ping (bereikbaarheid/RTT), traceroute (pad), nslookup (DNS), ss (poorten).
  • SNMP: protocol om netwerkapparaten te monitoren. Gebruik SNMPv3 voor beveiliging.
  • Syslog: gecentraliseerde logboeken. Niveaus 0 (emergency) t/m 7 (debug).
  • UniFi-dashboard: realtime overzicht van clients, verkeer, statistieken en alerts.
