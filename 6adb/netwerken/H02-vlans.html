2. Theorie

2.1 Wat is een VLAN en waarom?

Definitie — VLAN

Een VLAN (Virtual Local Area Network) is een logische segmentatie van een fysiek netwerk. Apparaten in hetzelfde VLAN communiceren alsof ze op hetzelfde fysieke netwerk zitten, ongeacht hun fysieke locatie. Communicatie tussen VLANs vereist een router of Layer 3-switch.

Zonder VLAN: alle apparaten op dezelfde switch zitten in hetzelfde broadcast-domein. Broadcasts (ARP, DHCP-discover) bereiken alle apparaten.

Met VLAN: elk VLAN is een apart broadcast-domein. Een DHCP Discover van VLAN 10 bereikt nooit VLAN 20.

Voordelen van VLANs:

VoordeelUitleg
BeveiligingAfdelingen gescheiden — HR-server niet bereikbaar vanuit gast-wifi
BeheerLogische segmentatie onafhankelijk van fysieke locatie
Minder broadcast-verkeerKleinere broadcast-domeinen = minder overhead
FlexibiliteitApparaat verplaatsen = VLAN-configuratie wijzigen, geen bekabeling
QoSPrioriteit geven aan VLAN-verkeer (bv. VoIP-VLAN hoge prioriteit)

2.2 Poorttypen: access en trunk

2.2.1 Access-poort (untagged)

Definitie — Access-poort

Een access-poort is geconfigureerd voor één VLAN. Apparaten aangesloten op een access-poort zijn "onbewust" van het VLAN — ze sturen gewone Ethernet-frames zonder VLAN-tag.

Gebruik: aansluiting van eindapparaten (pc, printer, IP-telefoon, access point) die slechts tot één VLAN behoren.

SW1(config)# interface GigabitEthernet0/1
SW1(config-if)# switchport mode access
SW1(config-if)# switchport access vlan 10

2.2.2 Trunk-poort (tagged)

Definitie — Trunk-poort

Een trunk-poort draagt verkeer van meerdere VLANs tegelijk. Elk frame wordt voorzien van een 802.1Q-tag met het VLAN-ID zodat de ontvangende switch weet bij welk VLAN het frame hoort.

Gebruik: verbindingen tussen switches onderling, of switch–router bij router-on-a-stick.

SW1(config)# interface GigabitEthernet0/24
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport trunk allowed vlan 10,20,30
SW1(config-if)# switchport trunk native vlan 1

Native VLAN: frames op de native VLAN worden niet getagged op de trunkpoort. Standaard is VLAN 1 de native VLAN. Best practice: verander de native VLAN naar een ongebruikt VLAN om VLAN-hopping aanvallen te voorkomen.

2.3 802.1Q tagging

Definitie — 802.1Q

IEEE 802.1Q is de standaard voor VLAN-tagging op Ethernet-netwerken. Er wordt een 4-byte tag ingevoegd in het Ethernet-frame na het bron-MAC-adres.

802.1Q tag-structuur:

+--------+--------+--------+--------+
| TPID   | PCP    | DEI    | VID    |
| 2 bytes| 3 bits | 1 bit  | 12 bits|
+--------+--------+--------+--------+
  • TPID (Tag Protocol Identifier): altijd 0x8100
  • PCP (Priority Code Point): QoS-prioriteit (0–7)
  • DEI (Drop Eligible Indicator): bij congestie mogen deze frames vallen
  • VID (VLAN Identifier): VLAN-ID 1–4094 (0 en 4095 gereserveerd)

2.5 Inter-VLAN routing

Apparaten in verschillende VLANs kunnen niet rechtstreeks communiceren — ze hebben een router (of Layer 3-switch) nodig.

2.5.1 Router-on-a-stick

Definitie — Router-on-a-stick

Router-on-a-stick is een configuratie waarbij één fysieke router-interface via een trunk-verbinding meerdere VLANs bedient door gebruik te maken van subinterfaces, elk geconfigureerd met 802.1Q encapsulatie.

! Router subinterfaces configureren
R1(config)# interface GigabitEthernet0/0
R1(config-if)# no shutdown

R1(config)# interface GigabitEthernet0/0.10
R1(config-subif)# encapsulation dot1Q 10
R1(config-subif)# ip address 192.168.10.1 255.255.255.0

R1(config)# interface GigabitEthernet0/0.20
R1(config-subif)# encapsulation dot1Q 20
R1(config-subif)# ip address 192.168.20.1 255.255.255.0

R1(config)# interface GigabitEthernet0/0.30
R1(config-subif)# encapsulation dot1Q 30
R1(config-subif)# ip address 192.168.30.1 255.255.255.0

Nadeel: alle inter-VLAN-traffic gaat door de router → bottleneck bij veel verkeer.

2.5.2 Layer 3-switch (SVI)

Definitie — Layer 3-switch

Een Layer 3-switch combineert switching (laag 2) en routing (laag 3). Inter-VLAN routing gebeurt intern op de switch via SVIs (Switched Virtual Interfaces) — veel sneller dan via een externe router.

! Layer 3-switch configureren voor inter-VLAN routing
L3SW(config)# ip routing

L3SW(config)# interface vlan 10
L3SW(config-if)# ip address 192.168.10.1 255.255.255.0
L3SW(config-if)# no shutdown

L3SW(config)# interface vlan 20
L3SW(config-if)# ip address 192.168.20.1 255.255.255.0
L3SW(config-if)# no shutdown
MethodeSnelheidKostprijsSchaalbaar
Router-on-a-stickBeperkt (router-bottleneck)GoedkoperNiet bij veel VLANs
Layer 3-switch (SVI)Hoog (hardware routing)Duurder switchJa

Voorbeelden

Voorbeeld 1 — School VLAN-ontwerp

Een school heeft drie VLANs:

  • VLAN 10 (LEERLINGEN): 192.168.10.0/24 — 180 leerlingen-pc's
  • VLAN 20 (LEERKRACHTEN): 192.168.20.0/24 — 30 leerkrachten-laptops
  • VLAN 30 (SERVERS): 192.168.30.0/24 — 5 servers

Trunkpoort naar de Layer 3-switch die inter-VLAN routing doet. Servers in VLAN 30 bereikbaar vanuit beide VLANs via SVI's.

Voorbeeld 2 — VLAN-hopping aanval voorkomen

Bescherming tegen een aanvaller die VLAN-hopping probeert:

  • Schakel ongebruikte poorten uit: shutdown
  • Zet poorten op switchport mode access (nooit dynamic)
  • Verander native VLAN: switchport trunk native vlan 999 (ongebruikt VLAN)
  • VLAN 1 nooit gebruiken voor productieverkeer

Samenvatting

Samenvatting
  • VLANs segmenteren een netwerk logisch in gescheiden broadcast-domeinen — beter beheer, beveiliging en minder broadcast-overhead.
  • Access-poorten: één VLAN, geen tag. Trunk-poorten: meerdere VLANs, 802.1Q-tagged.
  • Native VLAN = ongetagged op trunk. Verander de native VLAN om VLAN-hopping te voorkomen.
  • Inter-VLAN routing: router-on-a-stick (subinterfaces + trunk) of Layer 3-switch (SVI + ip routing).
  • Layer 3-switch is sneller en schaalbaarder voor inter-VLAN routing, maar duurder.
