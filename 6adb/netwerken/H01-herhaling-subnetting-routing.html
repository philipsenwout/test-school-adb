1. Theorie

1.1 Herhaling: VLSM op grotere schaal

In het 5e jaar leerden we subnetting en VLSM op één /24-blok. In professionele omgevingen werk je met grotere adresblokken en complexere scenario's.

Tip — VLSM-vuistregel

Werk altijd van groot naar klein. Begin met het grootste subnet en werk naar beneden. Sla dit als mentale standaardprocedure op — het vermijdt adresverspilling en overlapping.

Herhaling stappenplan:

  1. Sorteer de subnetten van groot naar klein (op basis van benodigde hosts)
  2. Bepaal per subnet het kleinste /x dat genoeg hosts biedt (2y - 2 ≥ hosts)
  3. Wijs subnetten toe zonder overlap, beginnend bij het basisadres

1.2 Routing verdiept: hoe routers beslissen

Een router bekijkt elk inkomend pakket en zoekt de best matching route via het principe van longest prefix match:

Definitie — Longest prefix match

Wanneer meerdere routes in de routeringstabel overeenkomen met de bestemming, kiest de router de route met het langste (meest specifieke) prefix. /28 wint van /24 wint van /0.

Voorbeeld:

Routeringstabel:
  S    10.0.0.0/8 via 192.168.1.1
  S    10.5.0.0/16 via 192.168.2.1
  S    10.5.10.0/24 via 192.168.3.1
  S*   0.0.0.0/0 via 10.255.255.1

Pakket naar 10.5.10.50:
- Matcht /8  → kandidaat
- Matcht /16 → betere kandidaat
- Matcht /24 → beste match (langste prefix) ✓

Pakket naar 172.20.5.1:
- Geen specifieke match → default route 0.0.0.0/0 ✓

1.3 Administratieve distance

Wanneer een router meerdere bronnen heeft voor dezelfde route (bv. statisch EN via OSPF), gebruikt hij de administrative distance om te kiezen:

RoutebronAdministrative Distance
Connected (direct verbonden)0
Static route1
OSPF110
RIP120
External OSPF170

Lagere waarde = meer vertrouwen.

1.4 Dynamische routeringprotocollen — concept

Bij statische routing configureer je elke route handmatig. Bij grote netwerken met tientallen routers is dit onhaalbaar. Dynamische routeringprotocollen laten routers hun routes automatisch uitwisselen.

Definitie — Routeringprotocol

Een routeringprotocol is een set regels waarmee routers informatie uitwisselen over bereikbare netwerken, zodat ze hun routeringstabellen automatisch opbouwen en aanpassen.

CategorieVoorbeeldenGebruik
IGP (Interior Gateway Protocol)OSPF, EIGRP, RIPRouting binnen één organisatie (autonomous system)
EGP (Exterior Gateway Protocol)BGPRouting tussen organisaties / op het internet

1.4.1 OSPF — Open Shortest Path First

Definitie — OSPF

OSPF (Open Shortest Path First) is een link-state routeringprotocol. Elke router bouwt een volledige topologiekaart van het netwerk en berekent de kortste paden via het Dijkstra-algoritme.

OSPF-kenmerken:

  • Open standaard (geen vendor lock-in)
  • Snel convergerend (reageert snel op topologiewijzigingen)
  • Gebruikt cost als metriek (gebaseerd op bandbreedte: cost = 108 / bandbreedte in bps)
  • Werkt met areas voor schaalbaarheid (Area 0 = backbone)
Tip

OSPF is de industriestandaard voor enterprise-netwerken. In de praktijk als beheerder kom je OSPF bijna zeker tegen. Je hoeft het niet volledig te configureren in het 6e jaar, maar de concepten (neighbor, LSA, SPF, area) zijn essentieel.

OSPF-states (vereenvoudigd):

Down → Init → Two-way → Exstart → Exchange → Loading → Full

OSPF minimale configuratie:

R1(config)# router ospf 1
R1(config-router)# network 192.168.1.0 0.0.0.255 area 0
R1(config-router)# network 192.168.2.0 0.0.0.255 area 0

(Wildcard mask = inverse van subnetmasker: 255.255.255.0 → wildcard = 0.0.0.255)

1.5 Routeringstabel diagnostiek

R1# show ip route
! Codes: C = Connected, S = Static, O = OSPF, R = RIP

R1# show ip ospf neighbor
! Toont OSPF-buren en hun status

R1# debug ip routing
! Toont real-time routeringswijzigingen (voorzichtig gebruiken — veel output!)

Voorbeelden

Voorbeeld 1 — Routeringstabel met OSPF

Routeringstabel van R1 in een netwerk met drie routers (R1, R2, R3) die OSPF gebruiken:

C    192.168.1.0/24 is directly connected, Gi0/0
C    192.168.2.0/24 is directly connected, Gi0/1
O    192.168.3.0/24 [110/2] via 192.168.2.2, 00:01:05, Gi0/1
O    192.168.4.0/24 [110/3] via 192.168.2.2, 00:01:05, Gi0/1
S*   0.0.0.0/0 [1/0] via 10.0.0.1

Interpretatie: [110/2]: administrative distance 110 (OSPF), cost 2. OSPF heeft automatisch de routes naar 192.168.3.0 en 192.168.4.0 geleerd via R2.

Voorbeeld 2 — Longest prefix match in de praktijk

Router ontvangt pakket naar 10.5.10.200: kandidaten: /8, /16, /24 → /24 wint (meest specifiek).

Pakket naar 10.8.0.1: kandidaten: /8 → enige match.

Samenvatting

Samenvatting
  • Longest prefix match: de router kiest altijd de meest specifieke (langste) route.
  • Administrative distance bepaalt welke routebron betrouwbaarder is: connected (0) > static (1) > OSPF (110) > RIP (120).
  • Dynamische routeringprotocollen (OSPF, EIGRP, BGP) bouwen automatisch routeringstabellen op — essentieel bij grote netwerken.
  • OSPF is een open-standaard link-state protocol dat de kortste paden berekent via Dijkstra. Gebruikt cost als metriek.
  • In productie-omgevingen is OSPF de standaard voor enterprise-routering.
