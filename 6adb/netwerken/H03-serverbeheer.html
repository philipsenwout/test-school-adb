3. Theorie

3.1 Serverrollen en diensten

Een server is een computer die diensten aanbiedt aan clients. In een bedrijfsomgeving zijn meerdere serverrollen gebruikelijk:

RolDienstWindows ServerLinux
Domain ControllerActive Directory, authenticatieAD DS-rolSamba
DNS-serverNaamresolutieDNS Server-rolBIND9, Unbound
DHCP-serverIP-adresverdelingDHCP Server-rolISC-DHCP, dnsmasq
FileserverGedeelde bestandenFile Services-rolSamba, NFS
WebserverHTTP/HTTPSIISApache, Nginx
MonitoringNetwerk- en systeembewakingSCCMZabbix, Nagios, Prometheus

3.2 Active Directory (AD)

Definitie — Active Directory

Active Directory (AD) is de directoryservice van Microsoft die gebruikersaccounts, computers, groepen en policies in een domein centraliseert. Het laat toe om vanuit één centrale locatie alle gebruikers en hun toegangsrechten te beheren.

ComponentOmschrijving
DomainGroepering van objecten (gebruikers, computers, groepen) die één beheereenheid vormen
Domain Controller (DC)Server die de AD-database beheert en authenticatie uitvoert
Organizational Unit (OU)Container voor het logisch indelen van objecten (bv. per afdeling)
Group Policy Object (GPO)Instellingen die automatisch toegepast worden op gebruikers/computers
Forest/TreeHiërarchie van meerdere domeinen

3.2.1 Gebruikersbeheer in AD (PowerShell)

# Nieuwe gebruiker aanmaken
New-ADUser -Name "Jan Peeters" `
           -SamAccountName "jpeeters" `
           -UserPrincipalName "jpeeters@school.be" `
           -Path "OU=Leerlingen,DC=school,DC=be" `
           -AccountPassword (ConvertTo-SecureString "Wachtwoord@123" -AsPlainText -Force) `
           -Enabled $true

# Gebruiker toevoegen aan groep
Add-ADGroupMember -Identity "Groep-IT" -Members "jpeeters"

# Alle gebruikers in een OU oplijsten
Get-ADUser -Filter * -SearchBase "OU=Leerlingen,DC=school,DC=be" | Select Name, SamAccountName

3.3 DNS-serverrol (Windows Server)

Installatie via Server Manager → Add Roles and Features → DNS Server.

TypeBeschrijving
Primary ZoneGezaghebbende zone, bewerkbaar
Secondary ZoneKopie van primary zone (read-only, voor redundantie)
Stub ZoneBevat enkel NS-records van een andere zone
Forward Lookup ZoneNaam → IP
Reverse Lookup ZoneIP → Naam (PTR-records)

DNS-recordtypes:

RecordGebruikVoorbeeld
ANaam → IPv4www.school.be → 192.168.1.10
AAAANaam → IPv6www.school.be → 2001:db8::1
CNAMEAlias voor een andere naamftp.school.be → www.school.be
MXMailserver voor het domeinschool.be → mail.school.be
PTRReverse lookup (IP → naam)192.168.1.10 → www.school.be

3.4 DHCP-serverrol (Windows Server)

Installatie via Server Manager → Add Roles and Features → DHCP Server.

DHCP-scope configureren: Tools → DHCP → rechtsklik op IPv4 → New Scope.

InstellingVoorbeeld
Adresbereik192.168.10.100 – 192.168.10.200
Subnetmasker255.255.255.0
Default Gateway192.168.10.1
DNS-servers192.168.10.5, 8.8.8.8
Leaseduurtijd8 uur (schoolnetwerk)
Tip — DHCP-reservaties

Voor servers, printers en access points gebruik je DHCP-reservaties: het apparaat krijgt altijd hetzelfde IP-adres op basis van zijn MAC-adres. Dit combineert het gemak van DHCP met de voorspelbaarheid van een statisch IP.

3.5 Linux-serverbeheer (CLI)

Essentiële commando's voor een Linux-server:

# Systeemupdates
sudo apt update && sudo apt upgrade -y

# Pakketten installeren (Debian/Ubuntu)
sudo apt install nginx          # Nginx webserver
sudo apt install openssh-server # SSH-server

# Services beheren
sudo systemctl start nginx      # Starten
sudo systemctl stop nginx       # Stoppen
sudo systemctl enable nginx     # Automatisch starten bij boot
sudo systemctl status nginx     # Status bekijken

# Netwerkinfo
ip addr show                    # IP-adressen
ip route show                   # Routeringstabel
ss -tulnp                       # Open poorten en services

# Logs bekijken
sudo journalctl -u nginx        # Logs van een service
sudo tail -f /var/log/syslog    # Systeemlog live volgen

3.6 Bestandspermissies

Linux permissies

# Permissies bekijken
ls -la /var/www/html

# Uitvoer:
# -rw-r--r-- 1 root www-data 1234 jan 15 10:00 index.html
#  ^ ^ ^ ^
#  | | | +-- Other: r-- (lezen)
#  | | +---- Group: r-- (lezen)
#  | +------ Owner: rw- (lezen + schrijven)
#  +-------- Bestandstype: - = bestand, d = directory

# Permissies wijzigen (chmod)
sudo chmod 644 index.html       # rw-r--r--
sudo chmod 755 /var/www/html    # rwxr-xr-x (directory)

# Eigenaar wijzigen (chown)
sudo chown www-data:www-data index.html

Samenvatting

Samenvatting
  • Windows Server biedt serverrollen: Active Directory, DNS, DHCP, Fileserver, IIS.
  • Active Directory centraliseert gebruikers, groepen, computers en policies (GPOs) in een domein.
  • DNS-records: A (naam→IP), CNAME (alias), MX (mail), PTR (reverse).
  • DHCP-reservaties bieden het gemak van DHCP met de voorspelbaarheid van statische adressen.
  • Linux-servers worden beheerd via CLI: systemctl, apt, ip, chmod, chown.
  • Linux-permissies: read(4), write(2), execute(1) — voor owner, group, other.

Huistaak H03

Installeer een Ubuntu Server (of gebruik een Raspberry Pi) en stel een volledig geconfigureerde webserver in:

  1. Installeer Nginx en zorg dat de service automatisch start bij boot.
  2. Maak een eenvoudige HTML-pagina in /var/www/html/index.html met je naam en klas.
  3. Stel de bestandspermissies correct in (owner = www-data, permissies 644).
  4. Noteer je IP-adres en toon de pagina via browser vanop een andere computer in hetzelfde netwerk.
